本页是 Aleph Gateway 公开的每个 JSON-RPC 方法的紧凑快速参考。有关详细的请求/响应示例，请点击链接查看各个方法页面。

## 传输

所有方法都通过到 Gateway 服务器的 **WebSocket** 连接调用（默认 `ws://127.0.0.1:18790`）。协议是 JSON-RPC 2.0——每个请求都需要 `jsonrpc`、`method` 和 `id` 字段。通知（无 `id`）用于服务器到客户端的事件。

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "agent.run",
  "params": { "message": "Hello" }
}
```

## 认证

当 Gateway 配置中启用 `require_auth` 时，新 WebSocket 连接上的**第一条消息**必须是 `connect`。认证成功之前，所有后续请求都会被拒绝。

| 方法 | 描述 | 参数 |
|--------|-------------|------------|
| `connect` | 认证 WebSocket 连接 | `token` (string), `client_id` (string, optional) |

## 错误代码

标准 JSON-RPC 2.0 错误代码加上 Aleph 扩展：

| 代码 | 常量 | 描述 |
|------|----------|-------------|
| `-32700` | `PARSE_ERROR` | 无效的 JSON |
| `-32600` | `INVALID_REQUEST` | 格式错误的 JSON-RPC 请求 |
| `-32601` | `METHOD_NOT_FOUND` | 方法不存在 |
| `-32602` | `INVALID_PARAMS` | 无效的方法参数 |
| `-32603` | `INTERNAL_ERROR` | 内部服务器错误 |
| `-32000` | `AUTH_REQUIRED` | 需要认证（先发送 `connect`） |
| `-32001` | `AUTH_FAILED` | 认证凭据被拒绝 |
| `-32002` | `PERMISSION_DENIED` | 权限不足 |
| `-32003` | `RATE_LIMITED` | 请求过多 |
| `-32004` | `RESOURCE_NOT_FOUND` | 请求的资源未找到 |
| `-32005` | `TIMEOUT_ERROR` | 操作超时 |
| `-32006` | `TOOL_ERROR` | 工具执行失败 |

错误响应格式：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32601,
    "message": "Method not found: agent.nonexistent"
  }
}
```

## 事件主题

事件作为 JSON-RPC 通知（无 `id` 字段）交付给订阅的客户端。通过 `events.subscribe` 订阅。

| 主题模式 | 描述 |
|---------------|-------------|
| `stream.start` | Agent 响应流式传输开始 |
| `stream.chunk` | Agent 的文本块 |
| `stream.tool_start` | 工具调用开始 |
| `stream.tool_end` | 工具执行完成 |
| `stream.thinking` | 推理步骤（CoT） |
| `stream.end` | 流式传输完成 |
| `agent.started` | Agent 运行开始 |
| `agent.completed` | Agent 运行完成 |
| `agent.error` | Agent 运行失败 |
| `agent.cancelled` | Agent 运行取消 |
| `session.created` | 新会话创建 |
| `session.deleted` | 会话删除 |
| `session.compacted` | 会话压缩 |
| `config.changed` | 配置修改 |
| `config.reloaded` | 从文件重新加载配置 |
| `cron.triggered` | Cron 作业触发 |
| `cron.completed` | Cron 作业完成 |
| `cron.failed` | Cron 作业失败 |
| `exec.approval_requested` | 命令需要审批 |
| `exec.approval_resolved` | 审批决策已做出 |

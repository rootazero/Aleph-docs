Aleph 通过**接口**（也称为频道）与用户通信——可插拔的适配器，将消息平台桥接到 Gateway 的统一消息总线。单个 Aleph 实例可以同时为 Telegram、Discord、iMessage、WebChat 和 CLI 提供服务，所有这些都共享同一个 Agent 大脑、记忆和工具生态系统。

## 架构

每个接口遵循相同的数据流：

```
平台 API ──> 接口适配器 ──> InboundMessage ──> Gateway 路由器
                                                              │
                                                         会话密钥
                                                              │
                                                          执行器
                                                               │
平台 API <── 接口适配器 <── OutboundMessage <── 响应
```

Gateway 的 **ChannelRegistry** 管理所有活动接口的生命周期。当 Aleph 启动时，它读取频道配置，通过 **ChannelFactory** 实例化每个启用的接口，并调用 `start()`。从那时起，入站消息通过 `mpsc` 频道流向 Gateway，出站响应通过同一个适配器调度回去。

## Channel Trait

所有接口都实现 `Channel` trait，无论底层平台如何，都提供统一的 API：

```rust
#[async_trait]
pub trait Channel: Send + Sync {
    /// 频道元数据（id、名称、类型、状态、能力）
    fn info(&self) -> &ChannelInfo;

    /// 启动频道（连接、认证、开始轮询/监听）
    async fn start(&mut self) -> ChannelResult<()>;

    /// 停止频道（断开连接、清理）
    async fn stop(&mut self) -> ChannelResult<()>;

    /// 通过此频道发送消息
    async fn send(&self, message: OutboundMessage) -> ChannelResult<SendResult>;

    /// 获取入站消息接收器
    fn inbound_receiver(&self) -> Option<mpsc::Receiver<InboundMessage>>;

    /// 发送输入中指示器
    async fn send_typing(&self, conversation_id: &ConversationId) -> ChannelResult<()>;

    /// 对消息做出反应
    async fn react(&self, message_id: &MessageId, reaction: &str) -> ChannelResult<()>;

    /// 编辑先前发送的消息
    async fn edit(&self, message_id: &MessageId, new_text: &str) -> ChannelResult<()>;

    /// 删除消息
    async fn delete(&self, message_id: &MessageId) -> ChannelResult<()>;
}
```

每个接口还有一个 **ChannelFactory**，用于在运行时从 JSON/TOML 配置创建实例，当配置文件更改时支持热重载。

## 频道能力

并非每个平台都支持每个功能。每个频道声明一个 `ChannelCapabilities` 结构体，以便 Agent 可以调整其行为：

```rust
pub struct ChannelCapabilities {
    pub attachments: bool,
    pub images: bool,
    pub audio: bool,
    pub video: bool,
    pub reactions: bool,
    pub replies: bool,
    pub editing: bool,
    pub deletion: bool,
    pub typing_indicator: bool,
    pub read_receipts: bool,
    pub rich_text: bool,
    pub max_message_length: usize,
    pub max_attachment_size: u64,
}
```

## InteractionManifest

除了原始能力之外，Aleph 还使用 **InteractionManifest** 系统来通知 AI 当前频道可以做什么。每个频道属于一个 `InteractionParadigm`：

| 范式 | 频道 | 能力 |
|----------|----------|-------------|
| **CLI** | 终端 | 富文本、代码高亮、流式传输 |
| **WebRich** | WebChat、桌面应用 | 完整交互式 UI、canvas、mermaid 图表、流式传输 |
| **Messaging** | Telegram、Discord、iMessage | 富文本、内联图片 |
| **Background** | Cron 作业、webhooks | 仅静默回复 |
| **Embedded** | 最小 UI 上下文 | 无特殊能力 |

清单告诉 AI 它是否可以使用 Mermaid 图表、交互式按钮、canvas 绘图或流式传输——因此它会根据用户屏幕上实际渲染的内容定制其输出格式。

## 会话密钥解析

当消息到达任何接口时，Gateway 会解析一个 **SessionKey**，确定要加载哪个对话历史记录。会话密钥编码完整上下文：

| 会话类型 | 密钥格式 | 示例 |
|-------------|-----------|---------|
| **Main** | `agent:{id}:main` | `agent:main:main` |
| **DM (按对话方)** | `agent:{id}:dm:{peer}` | `agent:main:dm:user123` |
| **DM (按频道-对话方)** | `agent:{id}:{channel}:dm:{peer}` | `agent:main:telegram:dm:user123` |
| **群组** | `agent:{id}:{channel}:group:{peer}` | `agent:main:discord:group:guild456` |
| **任务** | `agent:{id}:{type}:{task}` | `agent:main:cron:daily-summary` |
| **临时** | `agent:{id}:ephemeral:{uuid}` | `agent:main:ephemeral:a1b2c3` |

**DmScope** 策略控制如何隔离直接消息：

- **`Main`** — 所有 DM 共享主会话（跨用户的对话合并）
- **`PerPeer`** — 每个用户获得自己的会话，跨频道共享（默认）
- **`PerChannelPeer`** — 每个频道上的每个用户获得单独的会话

<Callout type="info">
**身份链接**允许你跨频道映射同一个人。如果 Telegram 上的用户 `123` 和 Discord 上的用户 `456` 都是 "john"，他们的 DM 会话可以在 `agent:main:dm:john` 下统一。
</Callout>

## 路由绑定

你可以使用路由绑定将特定频道、用户或公会绑定到不同的 Agent。解析遵循优先级链：

1. **Peer** — 特定用户/聊天匹配（最高优先级）
2. **Guild** — Discord 公会或 Slack 工作区匹配
3. **Team** — 团队级匹配
4. **Account** — 特定账户匹配
5. **Channel** — 频道类型匹配（例如，所有 Telegram 到一个 Agent）
6. **Default** — 回退到默认 Agent

```toml
# 将所有 Telegram 消息路由到 "personal" Agent
[[bindings]]
agent_id = "personal"
[bindings.match_rule]
channel = "telegram"
account_id = "*"

# 将特定 Discord 公会路由到 "work" Agent
[[bindings]]
agent_id = "work"
[bindings.match_rule]
channel = "discord"
guild_id = "123456789"
```

## 功能编译

Aleph 默认编译所有生产功能。接口通过配置启用（而非编译时功能）。只需在 `~/.aleph/config.toml` 中启用你需要的频道。

iMessage 接口仅在 macOS 上有条件编译（`#[cfg(target_os = "macos")]`）。

## 频道配置

所有频道都通过 Aleph 配置文件（`~/.aleph/config.toml`）进行配置。文件监视器检测到更改，并在 600 毫秒内通过热重载应用：

```toml
[channels.telegram]
enabled = true
token = "${TELEGRAM_BOT_TOKEN}"
allowed_chats = []

[channels.discord]
enabled = true
token = "${DISCORD_BOT_TOKEN}"
allowed_guilds = [123456789]

[channels.webchat]
enabled = true
port = 18790
cors_origins = ["http://localhost:*"]
```

配置支持 `${ENV_VAR}` 扩展用于密钥，因此令牌永远不需要以明文存储。

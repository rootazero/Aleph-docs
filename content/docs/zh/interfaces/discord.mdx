Discord 接口使用 [serenity](https://github.com/serenity-rs/serenity) 框架将 Aleph 连接到 Discord。它支持公会（服务器）频道、直接消息、文件附件、嵌入、反应、Markdown 格式和斜杠命令。

## 能力

| 功能 | 状态 |
|---------|--------|
| 文本消息 | 支持 |
| Markdown | 支持 |
| 文件附件 | 支持（发送和接收） |
| 图片 | 支持（发送和接收） |
| 音频/视频 | 支持（发送和接收） |
| 消息反应 | 支持 |
| 回复线程 | 支持 |
| 消息编辑 | 支持（带频道上下文） |
| 消息删除 | 支持（带频道上下文） |
| 输入中指示器 | 支持 |
| 嵌入 | 支持（出站） |
| 斜杠命令 | 可配置 |
| 最大消息长度 | 2,000 字符 |
| 最大附件大小 | 25 MB（Nitro 为 100 MB） |

## 前置条件

<Steps>
<Step>

### 创建 Discord 应用

1. 前往 [Discord Developer Portal](https://discord.com/developers/applications)
2. 点击 **New Application** 并命名（例如 "Aleph"）
3. 在左侧边栏导航到 **Bot** 部分
4. 点击 **Add Bot** 并确认
5. 在机器人的令牌部分下，点击 **Copy** 获取你的机器人令牌

</Step>
<Step>

### 配置 Bot Intents

仍在 Developer Portal 中，在 **Bot** 部分下：

1. 向下滚动到 **Privileged Gateway Intents**
2. 启用 **Message Content Intent**（需要读取消息文本）
3. 如果你需要成员相关功能，启用 **Server Members Intent**
4. 保存更改

<Callout type="warn">
**Message Content Intent** 是一个特权 intent。没有它，机器人无法在公会频道中读取消息文本。Discord 要求 100+ 服务器的机器人进行验证。
</Callout>

</Step>
<Step>

### 生成邀请 URL

导航到 **OAuth2 > URL Generator**：

1. 在 **Scopes** 下，选择 `bot` 和 `applications.commands`
2. 在 **Bot Permissions** 下，选择：
   - **Send Messages**（必需）
   - **View Channel**（必需）
   - **Read Message History**（必需）
   - **Embed Links**（推荐）
   - **Attach Files**（推荐）
   - **Add Reactions**（推荐）
   - **Use Slash Commands**（可选）
   - **Manage Messages**（可选，用于清理）
3. 复制生成的 URL 并在浏览器中打开以邀请机器人到你的服务器

</Step>
<Step>

### 启用 Discord 功能

使用 `discord` 功能标志构建 Aleph：

```bash
cargo build --release --features discord
# 或用于所有接口：
cargo build --release --features all-interfaces
```

</Step>
</Steps>

## 配置

### 最小配置

```toml
[[channels]]
id = "discord"
channel_type = "discord"
enabled = true

[channels.config]
bot_token = "MTIzNDU2Nzg5MDEyMzQ1Njc4OQ.ABcDeF.1234567890abcdefghijklmnop"
```

### 完整配置参考

```toml
[[channels]]
id = "discord"
channel_type = "discord"
enabled = true

[channels.config]
# 来自 Discord Developer Portal 的机器人令牌（必需）
# 支持 ${ENV_VAR} 扩展
bot_token = "${DISCORD_BOT_TOKEN}"

# 用于斜杠命令注册的应用 ID（可选）
application_id = 123456789012345678

# 公会（服务器）白名单——空表示允许所有公会
allowed_guilds = [123456789012345678]

# 公会内频道白名单——空表示允许所有频道
allowed_channels = [987654321098765432]

# 允许直接消息（默认：true）
dm_allowed = true

# 文本命令前缀（默认："!"）
command_prefix = "!"

# 仅在公会中被提及时响应（默认：true）
respond_to_mentions = true

# 注册并使用斜杠命令（默认：true）
slash_commands_enabled = true

# 处理时发送输入中指示器（默认：true）
send_typing = true

# Gateway intents 配置
[channels.config.intents]
guild_messages = true      # 接收公会消息（默认：true）
direct_messages = true     # 接收 DM（默认：true）
message_content = true     # 读取消息内容（默认：true，特权）
guild_members = false      # 跟踪公会成员事件（默认：false，特权）
```

<Callout type="warn">
Discord 机器人令牌是长 Base64 编码字符串（通常 59+ 字符）。Aleph 在配置加载时验证令牌长度。
</Callout>

### 环境变量模式

```bash
export DISCORD_BOT_TOKEN="MTIzNDU2..."
export DISCORD_APPLICATION_ID="123456789012345678"  # 可选
```

## 权限审计

Aleph 包含内置的权限审计系统，检查机器人在每个公会中的权限。权限分为三个级别：

### 必需权限

这些对基本操作至关重要：

| 权限 | 标志 | 用途 |
|-----------|------|---------|
| Send Messages | `0x800` | 发送响应 |
| View Channel | `0x400` | 查看机器人所在的频道 |
| Read Message History | `0x40000` | 读取对话上下文 |

### 推荐权限

这些启用更丰富的交互：

| 权限 | 标志 | 用途 |
|-----------|------|---------|
| Embed Links | `0x4000` | 富消息嵌入 |
| Attach Files | `0x8000` | 发送文件附件 |
| Add Reactions | `0x40` | 对消息做出反应 |

### 可选权限

| 权限 | 标志 | 用途 |
|-----------|------|---------|
| Manage Messages | `0x2000` | 删除/置顶消息 |
| Use Slash Commands | `0x80000000` | 注册斜杠命令 |

审计产生一个交通灯评估：

- **绿色（健康）** — 所有必需和推荐权限已授予
- **黄色（降级）** — 必需权限存在，缺少一些推荐权限
- **红色（严重）** — 缺少必需权限；机器人无法运行

你可以通过 `interface.status` RPC 方法检查权限状态。

## 消息路由

### 公会频道消息

在公会频道中，机器人在以下情况下处理消息：

1. 消息**不是来自机器人**（防止自循环）
2. 公会在 `allowed_guilds` 中（或列表为空）
3. 频道在 `allowed_channels` 中（或列表为空）
4. 机器人被**提及**，或消息以 `command_prefix` 开头，或 `respond_to_mentions` 为 `false`

当 `respond_to_mentions` 为 `true`（默认）时，机器人仅响应 `@mention` 它或使用命令前缀的消息。这防止机器人在繁忙频道中响应每条消息。

提及文本在路由到 Agent 之前自动从消息中剥离。

### 直接消息

当 `dm_allowed` 为 `true` 时处理 DM。不需要前缀或提及——所有 DM 文本直接转发给 Agent。

### 对话 ID

| 上下文 | 对话 ID |
|---------|----------------|
| 公会频道 | 频道 ID（例如 `987654321098765432`） |
| 直接消息 | `dm:{user_id}`（例如 `dm:123456789012345678`） |

## 会话路由

每个 Discord 对话映射到会话密钥：

| 上下文 | 会话密钥 |
|---------|------------|
| 与用户 DM | `agent:main:dm:{user_id}` 或 `agent:main:discord:dm:{user_id}` |
| 公会频道 | `agent:main:discord:group:{channel_id}` |

## 附件

### 接收附件

Discord 附件以完整元数据提取：

```rust
Attachment {
    id: "attachment_id",
    mime_type: "image/png",      // 来自 content_type 或默认值
    filename: Some("image.png"), // 原始文件名
    size: Some(12345),           // 字节大小
    url: Some("https://cdn.discordapp.com/..."), // CDN URL
}
```

### 发送附件

出站附件作为 Discord 文件附件发送。目前支持内存字节数据：

```rust
let msg = OutboundMessage::text("channel_id", "Here is the file")
    .with_attachment(Attachment {
        data: Some(file_bytes),
        filename: Some("output.txt".to_string()),
        ..Default::default()
    });
```

## 回复线程

当 Agent 响应特定消息时，使用 Discord 的回复引用：

- 入站消息如果用户回复了先前消息，则包含 `reply_to`
- 出站消息可以设置 `reply_to` 以创建回复链

这在 Discord 的 UI 中创建视觉回复指示器。

## 斜杠命令

当 `slash_commands_enabled` 为 `true` 且 `application_id` 已设置时，Aleph 可以向 Discord 注册斜杠命令。这些在用户输入 `/` 时出现在命令选择器中：

- `/ask` — 向 Aleph 发送消息
- `/status` — 检查 Aleph 的状态
- `/help` — 显示可用命令

<Callout type="info">
斜杠命令注册需要 `applications.commands` OAuth2 范围。全局命令可能需要长达一个小时才能传播；公会特定命令是即时的。
</Callout>

## 错误处理

| 错误 | 行为 |
|-------|----------|
| **无效的机器人令牌** | 客户端构建失败；频道进入 `Error` 状态 |
| **缺少 intents** | 机器人连接但无法读取消息；检查 Developer Portal 设置 |
| **公会不在白名单中** | 来自该公会的消息被静默忽略 |
| **频道不在白名单中** | 来自该频道的消息被静默忽略 |
| **机器人未被提及** | 当 `respond_to_mentions` 为 `true` 时跳过消息 |
| **速率限制** | Serenity 自动处理 Discord 速率限制 |

## Gateway Intents

Discord 要求机器人声明它们想要接收哪些事件。Aleph 的 intent 配置直接映射到 Discord Gateway Intents：

```toml
[channels.config.intents]
guild_messages = true      # GUILD_MESSAGES intent
direct_messages = true     # DIRECT_MESSAGES intent
message_content = true     # MESSAGE_CONTENT intent (privileged)
guild_members = false      # GUILD_MEMBERS intent (privileged)
```

<Callout type="warn">
**特权 intents**（`message_content` 和 `guild_members`）必须在使用前在 Discord Developer Portal 中启用。如果你的机器人在 100+ 公会中，Discord 要求在授予这些 intents 之前进行机器人验证。
</Callout>

## 故障排除

| 问题 | 解决方案 |
|---------|----------|
| 机器人在线但不响应 | 在 Developer Portal 中启用 **Message Content Intent** |
| 机器人忽略频道中的消息 | 检查 `allowed_guilds` 和 `allowed_channels`；验证机器人具有 View Channel 权限 |
| 机器人响应每条消息 | 设置 `respond_to_mentions = true` 或使用 `command_prefix` |
| "Failed to create Discord client" | 验证机器人令牌有效且足够长（59+ 字符） |
| 斜杠命令不出现 | 确保 `application_id` 已设置且机器人被邀请时带有 `applications.commands` 范围 |
| 权限审计显示红色 | 使用审计的 `fix_suggestions` 在服务器设置中授予缺少的权限 |
| 附件发送失败 | 检查机器人在目标频道中是否具有 **Attach Files** 权限 |

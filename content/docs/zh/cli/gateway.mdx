`aleph-server` 二进制文件管理 Aleph Gateway 生命周期。本页涵盖与启动、停止、监控和查询服务器相关的所有命令。

## 启动 Gateway

### 前台模式

```bash
aleph-server
# 或显式：
aleph-server start
```

服务器在配置的绑定地址和端口（默认 `127.0.0.1:18790`）上开始监听。日志打印到 stdout。

### 自定义绑定地址和端口

```bash
aleph-server --bind 0.0.0.0 --port 9000
```

| 标志 | 默认值 | 说明 |
|------|---------|-------------|
| `--bind <ADDR>` | `127.0.0.1` | 要绑定的网络接口。使用 `0.0.0.0` 表示所有接口。 |
| `--port <PORT>` | `18790` | WebSocket 监听端口 |
| `--force` | `false` | 即使端口似乎正在使用也启动 |
| `--max-connections <N>` | `1000` | 最大并发 WebSocket 连接数 |

### 使用配置文件

```bash
aleph-server -c ~/.aleph/config.toml
```

配置文件为 TOML 格式，可以指定代理、网关和控制平面设置：

```toml
[agent.main]
provider = "anthropic"
model = "claude-sonnet-4-20250514"

[gateway]
bind = "127.0.0.1"
port = 18790

[control_plane]
port = 18790
```

### 使用 WebChat UI

如果您有构建为静态文件的 WebChat 前端，可以 alongside Gateway 提供它：

```bash
aleph-server --webchat-dir ./webchat/dist --webchat-port 8080
```

| 标志 | 说明 |
|------|-------------|
| `--webchat-dir <PATH>` | 包含静态 HTML/JS/CSS 文件的目录 |
| `--webchat-port <PORT>` | WebChat 服务器的 HTTP 端口（默认与 `--port` 相同） |

## 守护进程模式

使用 `--daemon` (`-d`) 标志将 Gateway 作为后台进程运行。这对于长期运行部署很有用。

### 启动守护进程

```bash
aleph-server -d --log-file ~/.aleph/gateway.log
```

```
$ aleph-server -d --log-file ~/.aleph/gateway.log
# (进程分叉到后台，父进程立即退出)
```

| 标志 | 默认值 | 说明 |
|------|---------|-------------|
| `--daemon` / `-d` | `false` | 分叉到后台 |
| `--pid-file <PATH>` | `~/.aleph/gateway.pid` | PID 文件的位置 |
| `--log-file <PATH>` | — | 将 stdout/stderr 重定向到此文件。如果省略，输出转到 `/dev/null`。 |

守护进程将其 PID 写入 PID 文件。所有后续的 `stop` 和 `status` 命令都使用此文件来定位进程。

### 停止守护进程

```bash
aleph-server stop
```

预期输出：

```
Sending SIGTERM to gateway process (PID 54321)
Gateway stopped successfully
```

`stop` 命令发送 `SIGTERM` 并等待最多 5 秒以进行优雅关闭。如果进程未退出，则升级到 `SIGKILL`。

### 检查状态

```bash
aleph-server status
```

可能的输出：

```
Gateway is running (PID 54321)
```

```
Gateway is not running (stale PID file for PID 54321)
```

```
Gateway is not running (no PID file)
```

对于机器可读的输出：

```bash
aleph-server status --json
```

```json
{
  "running": true,
  "pid": 54321
}
```

## 健康检查

从客户端 CLI，验证 Gateway 是否可达：

```bash
aleph health
```

```
Server Status: ok
Timestamp: 2026-02-25T10:30:00Z
```

这会通过 WebSocket 发送 `health` JSON-RPC 调用。如果服务器不可达，您将看到连接错误：

```
Error: Connection error: Connection refused (os error 61)
```

要检查远程服务器：

```bash
aleph -s ws://192.168.1.50:18790 health
```

  ✓ anthropic

`info` 命令查询 `health` 和 `providers.list` RPC 方法。

## Gateway RPC 工具

`aleph-server gateway call` 命令允许您在正在运行的 Gateway 上调用任意 JSON-RPC 方法。这对于调试和脚本编写很有用。

```bash
aleph-server gateway call <METHOD> [OPTIONS]
```

| 标志 | 简写 | 默认值 | 说明 |
|------|-------|---------|-------------|
| `--params <JSON>` | `-p` | — | 随 RPC 调用传递的 JSON 参数 |
| `--url <URL>` | — | `ws://127.0.0.1:18790` | Gateway WebSocket URL |
| `--timeout <MS>` | — | `30000` | 请求超时（毫秒） |

### 示例

```bash
# 通过原始 RPC 进行健康检查
aleph-server gateway call health

# 获取完整配置为 JSON
aleph-server gateway call config.get

# 调用带有参数的方法
aleph-server gateway call config.get -p '{"path": "general.language"}'

# 使用自定义超时调用
aleph-server gateway call agent.run \
  -p '{"input": "Hello"}' \
  --timeout 60000
```

## 系统服务集成

### macOS (launchd)

创建 `~/Library/LaunchAgents/com.aleph.server.plist`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.aleph.server</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/aleph-server</string>
        <string>--daemon</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
</dict>
</plist>
```

```bash
launchctl load ~/Library/LaunchAgents/com.aleph.server.plist
launchctl start com.aleph.server
```

### Linux (systemd)

创建 `/etc/systemd/system/aleph-server.service`：

```ini
[Unit]
Description=Aleph AI Server
After=network.target

[Service]
Type=simple
User=aleph
ExecStart=/usr/local/bin/aleph-server
Restart=on-failure
Environment="ANTHROPIC_API_KEY=your-api-key"

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now aleph-server
sudo systemctl status aleph-server
```

## 故障排除

### 端口已被占用

```bash
# 查找使用该端口的进程
lsof -i :18790

# 终止它，然后重试
kill <PID>
aleph-server start
```

或者使用 `--force` 尝试无论如何绑定，并使用 `--port` 选择不同的端口：

```bash
aleph-server --port 18790
```

### 陈旧的 PID 文件

如果 `aleph-server status` 报告陈旧的 PID 文件，则前一个进程在未清理的情况下退出。运行 `stop` 以清除它：

```bash
aleph-server stop
# "Gateway is not running (stale PID file)"
# PID 文件自动移除
```

### 来自客户端的连接被拒绝

确保服务器正在运行且 URL 匹配：

```bash
# 检查服务器状态
aleph-server status

# 验证端口
aleph -s ws://127.0.0.1:18790 health
```

如果服务器绑定到 `127.0.0.1` 而您从另一台机器连接，请使用 `--bind 0.0.0.0` 重新启动。

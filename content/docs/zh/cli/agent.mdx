`aleph` 客户端 CLI 提供多种方式与 Aleph 代理交互：交互式聊天、一次性问题和会话管理。

## 交互式聊天

启动与代理的持久对话：

```bash
aleph chat
```

或者简单地运行不带子命令的 `aleph`（chat 是默认操作）。

### 选项

| 标志 | 简写 | 默认值 | 说明 |
|------|-------|---------|-------------|
| `--session <KEY>` | `-s` | 自动生成的 UUID | 要恢复或创建的会话密钥 |

### 启动会话

```bash
$ aleph chat
=== Aleph Chat ===
Session: chat-a1b2c3d4-e5f6-7890-abcd-ef1234567890
Type 'exit' or 'quit' to end the session.
Type '/help' for commands.

You: Hello, what can you do?

Aleph: I can help you with a wide range of tasks...
```

要恢复之前的会话：

```bash
aleph chat -s my-project
```

### 会话内命令

在交互式聊天中，以下斜杠命令可用：

| 命令 | 说明 |
|---------|-------------|
| `/help` | 显示可用的会话内命令 |
| `/session` | 显示当前会话密钥 |
| `/clear` | 清除当前会话的对话历史 |
| `exit` 或 `quit` | 结束聊天会话并断开连接 |

### 流式响应

响应实时流式传输。当代理使用工具时，您将看到内联状态指示器：

```
You: What is the current system load?

Aleph:
  [Using: system_info] ✓
The current system load average is 1.23...
```

对勾 (`✓`) 表示工具执行成功；叉号 (`✗`) 表示失败。

### 交互式问题

代理可能会要求您在工具执行期间做出选择：

```
[Question: Allow file write to /tmp/report.txt?]
  1. Yes
  2. No
```

## 单条消息 (`ask`)

发送一次性消息并在 stdout 上接收响应。适用于脚本和管道。

```bash
aleph ask "Summarize the contents of my last meeting note"
```

### 选项

| 标志 | 简写 | 默认值 | 说明 |
|------|-------|---------|-------------|
| `--session <KEY>` | `-s` | `"default"` | 用于上下文连续性的会话密钥 |

### 输出行为

- 响应正文打印到 **stdout**。
- 工具使用通知和元数据打印到 **stderr**。
- 如果设置了 `ALEPH_VERBOSE`，推理跟踪也会打印到 stderr。

```bash
# 正常用法
$ aleph ask "What is 2 + 2?"
The answer is 4.

# 带有详细推理
$ ALEPH_VERBOSE=1 aleph ask "What is 2 + 2?"
  [Thinking: Simple arithmetic...]
The answer is 4.

# 带有工具使用
$ aleph ask "What time is it in Tokyo?"
  [Tool: world_clock]
It is currently 19:30 JST in Tokyo.

(1 tools used)
```

### 脚本编写

因为主要响应转到 stdout 而元数据转到 stderr，所以您可以干净地管道输出：

```bash
# 将响应管道到文件
aleph ask "Generate a haiku about Rust" > haiku.txt

# 在管道中使用
aleph ask "List 5 random words" | tr ',' '\n'

# 抑制工具噪音
aleph ask "Search for recent news" 2>/dev/null
```

## 会话管理

会话在服务器上存储对话历史。使用 `aleph session` 来管理它们。

### 列出会话

```bash
aleph session list
```

```
=== Sessions ===

• chat-a1b2c3d4 (my-project) - 42 messages
• default - 3 messages
• research-session - 18 messages

Total: 3 sessions
```

### 创建会话

```bash
aleph session new
aleph session new --name "my-project"
```

```
✓ Session created: ses-7f8e9d0c
```

| 标志 | 简写 | 说明 |
|------|-------|-------------|
| `--name <NAME>` | `-n` | 会话的可选人类可读名称 |

### 删除会话

```bash
aleph session delete <KEY>
```

```bash
$ aleph session delete chat-a1b2c3d4
✓ Session deleted: chat-a1b2c3d4
```

这将永久移除会话及其所有消息历史。

## 工具发现

列出服务器上注册的所有工具：

```bash
aleph tools
```

```
=== Available Commands ===

[builtin]
  • system_info - Get system information
  • file_read - Read file contents
  • file_write - Write content to a file

[mcp]
  • web_search - Search the web
  • calculator - Evaluate math expressions

Total: 5 commands
```

### 按类别过滤

```bash
aleph tools --category mcp
```

| 标志 | 简写 | 说明 |
|------|-------|-------------|
| `--category <FILTER>` | `-c` | 按源类型、密钥或命令类型过滤工具 |

## 认证

### 连接并保存令牌

在使用认证命令之前，连接到服务器：

```bash
aleph connect
```

```
Authenticating as 'aleph-cli'...
✓ Connected successfully!

Auth token: eyJhbGciOiJIUzI1N...
Token saved to config file.
```

| 标志 | 简写 | 默认值 | 说明 |
|------|-------|---------|-------------|
| `--device <NAME>` | `-d` | `aleph-cli` | 此客户端实例的设备名称 |

令牌保存到 `~/.config/aleph-cli/config.toml` 并自动用于未来的请求。

## 访客管理

管理对您的 Aleph 实例的访客访问。访客通过邀请令牌接收有限的工具权限。

### 创建邀请

```bash
aleph guests invite --name "Mom" --tools "translate,summarize"
```

```
Guest invitation created

  Guest ID: 6ba7b810-9dad-11d1-80b4-00c04fd430c8
  Token:    inv_abc123def456
  URL:      https://aleph.local/guest/inv_abc123def456

Share this token with the guest to activate their session.
```

| 标志 | 简写 | 默认值 | 说明 |
|------|-------|---------|-------------|
| `--name <NAME>` | `-n` | required | 访客显示名称 |
| `--tools <LIST>` | `-t` | required | 逗号分隔的工具名称，或 `"*"` 表示所有 |
| `--expires-days <N>` | — | — | 邀请过期天数 |
| `--format <FMT>` | `-f` | `text` | 输出格式：`text` 或 `json` |

### 列出待处理邀请

```bash
aleph guests list
```

### 撤销邀请

```bash
aleph guests revoke <GUEST_ID> [--force]
```

### 查看访客活动

```bash
aleph guests info <GUEST_ID>
```

```
=== Activity Logs for Guest: 6ba7b810-... ===

[2026-02-25 14:00:00] tool_call - translate("Hello" → "Hola")
[2026-02-25 14:01:30] tool_call - summarize(article.txt)

Total: 2 log entries
```

# 更新日志

Aleph 遵循里程碑驱动的开发过程。由于项目尚未达到 1.0，变更按月份和里程碑而非语义版本标签组织。

<Callout type="info">
Aleph 正在积极开发中。里程碑之间可能发生破坏性变更。API 和架构正在稳定但尚未冻结。
</Callout>

---

## 2026 年 2 月

### 桌面桥接重构

桌面集成层的完整重新设计，从原生 Swift macOS 应用迁移到跨平台 Tauri 桥接架构。

**以服务器为中心的构建架构**

- 引入 `BridgeSupervisor` — Rust 服务器现在生成并管理 Tauri 桥接作为子进程
- 实现 `DesktopBridgeManager` 用于服务器端桥接生命周期管理
- 添加基于 Unix 域套接字 (UDS) 的 JSON-RPC 协议用于服务器到桥接通信
- 桥接握手与能力协商 (`aleph.handshake`、`system.ping`)

**浏览器自动化**

- 新的 `BrowserTool`，通过 `chromiumoxide` 使用 Chrome DevTools Protocol (CDP) 传输
- 跨平台 Chromium 发现（macOS、Linux、Windows）
- ARIA 快照提取与 `ref_id` 解析用于语义元素定位
- 完整动作集：点击、输入、填充、滚动、悬停，支持 ARIA ref 定位
- 完整浏览器工作流的集成测试

**视觉与审批系统**

- `VisionTool`，支持可插拔提供商：Claude vision 和平台原生 OCR
- `ApprovalPolicy` trait 和配置用于控制敏感工具操作
- 审批检查集成到 `BrowserTool` 和 `DesktopTool`

**桌面能力（第 3 阶段）**

- 实现 12 个桥接方法：`click`、`type_text`、`key_combo`、`launch_app`、`window_list`、`focus_window`、`canvas_overlay`、`tray.update_status`、`ocr`、`ax_tree`、`scroll`、`drag`
- Windows 支持：通过 WinRT 进行输入模拟、OCR、UI 自动化可访问性树
- macOS：Vision 框架 OCR、可访问性 API 检查

**破坏性变更**

- 移除 UniFFI Swift 绑定 — 由 UDS IPC 协议替代
- 弃用并移除 macOS Swift 应用 — Tauri 桥接是唯一的桌面前端
- 移除 macOS 特定的 CI 工作流和构建脚本

### Cron 系统重新设计

- 用状态机调度引擎替换基于窗口的调度
- 添加带 `DeliveryTarget` trait 的交付管道（支持 webhook）
- 模板系统、资源管理和作业链
- 状态机、弹性、交付和链字段的架构迁移

### 记忆系统增强

- 通过向量相似度进行存储时去重
- 带滚动保留的 JSONL 备份服务
- `ScoringPipeline` 接入 `HybridRetrieval` 搜索流
- `NoiseFilter` 集成到摄取管道
- `MemoryConfig` 中添加自适应检索和备份设置

## 2026 年 1 月下旬

### POE (原则-操作-评估) 架构

用于结构化、验证的 Agent 任务执行的新执行框架。

- 核心类型：原则、操作、评估标准
- 基于熵的预算管理器用于资源分配
- 硬验证规则和通过 LLM 裁判的语义验证
- 结合基于规则和语义检查的复合验证器
- `POEManager` 用于编排 POE 生命周期
- 带 `AgentLoopWorker` 集成的工作器抽象
- Gateway RPC 处理程序：`poe.run`、`poe.status`、`poe.cancel`、`poe.list`
- 经验结晶输入技能进化系统

### Agent-动作交互

- `QuestionKind` 类型用于结构化用户交互（确认、选择、输入）
- `UserAnswer` 类型用于结构化响应
- Agent 循环中的 `AskUserRich` 决策变体
- `LoopCallback` trait 中的 `on_user_question` 回调
- 通过 `inquire` crate 的 CLI 实现

### 技能进化系统

- `EvolutionTracker` 用于记录工具执行模式
- `SolidificationDetector` 用于检测重复的成功模式
- `SkillGenerator` 用于自动创建 `SKILL.md`
- `GitCommitter` 用于自动提交进化的技能
- `SkillCompiler`（第 10 阶段）用于将技能编译为可执行形式

### 规范驱动开发

- `SpecWriter` 用于自动化需求分析
- `TestWriter` 用于从规范生成测试
- `LlmJudge` 用于根据规范评估生成的代码
- `SpecDrivenWorkflow` 编排器将所有内容联系在一起

### 弹性任务执行

- `ResilientTask` trait 用于定义容错任务
- `ResilientExecutor`，带重试逻辑、指数退避和回退策略
- 与 `PodcastTask` 示例的 Cron 集成

## 2026 年 1 月中旬

### rig-core 迁移

从自定义 AI 提供商层到 `rig-core` 框架的重大内部迁移。

- 将 `rig-core` 从 0.6 升级到 0.28
- 完整的 V1 到 V2 核心迁移（移除 `AetherCore`，`AetherV2Core` 现在是唯一实现）

- `RigAgentManager`，带工具调用 Agent 的处理方法
- `MemoryStore`，带 SQLite 和用于向量搜索的 `fastembed`
- `SearchTool`（Tavily API）和 `WebFetchTool`（HTML 提取）作为内置工具

## 2025 年 12 月前

### 基础

初始开发期建立了核心 Rust 服务器、Gateway WebSocket 协议、Agent 循环、会话管理和原始 macOS Swift 应用。关键里程碑包括：

- 带异步运行时（Tokio）的 Rust 核心
- 基于 WebSocket 的 Gateway JSON-RPC 2.0 协议
- 多提供商 AI 路由（Claude、GPT、Gemini、Ollama）
- Telegram 和 Discord 频道适配器
- 三级命令执行安全模型
- MCP（Model Context Protocol）工具集成
- SQLite 支持的对话持久化
- 通过 TOML 配置，支持环境变量覆盖

在 5 分钟内让 Aleph 运行起来。

## 选项 1：桌面应用（最快）

下载适合你平台的最新版本：

| 平台 | 下载 |
|------|------|
| **macOS** | `.dmg` 从 [GitHub Releases](https://github.com/rootazero/Aleph/releases) |
| **Linux** | `.deb` 或 `.AppImage` 从 [GitHub Releases](https://github.com/rootazero/Aleph/releases) |
| **Windows** | `.msi` 从 [GitHub Releases](https://github.com/rootazero/Aleph/releases) |

1. 安装并启动应用
2. 完成设置向导（API 密钥、模型选择）
3. 开始聊天

桌面应用将核心服务器作为后台守护进程捆绑。壳停留在你的系统托盘中；关闭窗口**不会**停止服务器。

**全局热键：** `CmdOrCtrl+Shift+A` 从任何地方召唤窗口。

### 连接到远程核心

桌面壳可以通过 Tailnet 或 VPN 连接到远程 `aleph-server`：

```bash
# 在远程机器上，确保网关可访问
aleph gateway run --bind 0.0.0.0 --port 18790
```

然后将本地壳指向远程来源（高级——参见 [配置](./configuration)）。

## 选项 2：从源代码（开发）

### 1. 构建并运行

```bash
# 克隆仓库
git clone https://github.com/rootazero/Aleph.git
cd Aleph

# 以开发模式运行（自动重建 WASM）
just dev

# 或手动构建并运行
cargo run --bin aleph-server
```

服务器默认在 `ws://127.0.0.1:18790` 启动。

### 2. 配置

在 `~/.aleph/config.toml` 创建配置文件：

```bash
mkdir -p ~/.aleph
cat > ~/.aleph/config.toml << 'EOF'
[agent]
model = "anthropic/claude-sonnet-4"

[provider.claude]
protocol = "anthropic"
api_key = "${ANTHROPIC_API_KEY}"

[gateway]
port = 18790
bind = "127.0.0.1"
EOF
```

设置你的 API 密钥：

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

### 3. 通过 WebChat 连接

在 `http://localhost:18790` 打开 WebChat 界面（或在 `gateway.port` 中配置的端口）。

WebChat 是一个基于 WASM 的 UI，连接到 Gateway WebSocket。

### 4. 连接消息通道（可选）

#### Telegram

添加到 `~/.aleph/config.toml`：

```toml
[channel.telegram]
enabled = true
token = "YOUR_BOT_TOKEN"
```

重启服务器以接收新通道。

#### Discord

```toml
[channel.discord]
enabled = true
token = "YOUR_BOT_TOKEN"
```

### 5. 验证

检查服务器是否正在运行：

```bash
# 服务器日志显示 "Gateway listening on ws://127.0.0.1:18790"
tail -f ~/.aleph/logs/aleph.log
```

测试 WebSocket 连接：

```bash
curl http://localhost:18790/health
```

## 架构一览

> **一核多端，壳核分离；网关互通，权限控制。**

- **一个核心**（`aleph-server`）驱动所有界面
- **壳**（桌面应用）= 纯 OS 集成；**核心** = 所有业务逻辑
- 所有终端使用相同的认证 WebSocket 协议
- 桌面壳可以连接本地或远程核心

---
title: "设置向导"
description: "通过 Aleph 的交互式入门向导完成配置"
---

设置向导是配置 Aleph 并使其准备就绪的最快方式。它以友好的分步流程引导你完成每一个基本设置——从选择 AI 提供商到连接消息应用。

## 启动向导

从命令行运行入门向导：

```bash
aleph wizard
```

向导会在你的终端中打开一个交互式会话。如果你正在构建客户端界面，也可以通过网关 RPC 启动它（参见 [向导 RPC 方法](/docs/gateway/methods/wizard)）。

### 向导变体

Aleph 提供三种向导流程。默认的 `aleph wizard` 命令运行完整的入门流程。

| 命令 | 流程 | 描述 |
|------|------|------|
| `aleph wizard` | `onboarding` | 完整的 10 阶段设置 —— 推荐给首次使用的用户 |
| `aleph wizard --quick` | `quick-setup` | 最小化配置 —— 只需一个 API 密钥即可就绪 |
| `aleph wizard --provider` | `provider-setup` | 添加或重新配置单个 AI 提供商 |

## 10 个阶段

以下是完整入门向导每个阶段的内容。

### 阶段 1 —— 欢迎

向导会向你问好并解释将要配置的内容：

- AI 提供商和模型
- 可选的用于故障转移的辅助模型
- AI 回复的思考级别
- 消息应用集成

无需输入 —— 按 **Enter** 继续。

### 阶段 2 —— 提供商选择

选择你的主要 AI 提供商。使用方向键高亮选项，然后按 **Enter** 选择。

| 提供商 | 说明 |
|--------|------|
| **Anthropic (Claude)** | 推荐 —— 最适合编程和推理 |
| **OpenAI (GPT)** | GPT-4o 和 o3 系列 |
| **Google (Gemini)** | Gemini Pro 和 Ultra |
| **Ollama (本地)** | 在你自己的硬件上运行模型 —— 无需 API 密钥 |
| **OpenRouter** | 通过单个密钥访问多个提供商 |

### 阶段 3 —— 凭据

如果你选择了云提供商，向导会要求你提供 API 密钥：

```
Enter your anthropic API key:
> sk-ant-••••••••
```

输入是隐藏的 —— 你的密钥永远不会显示在屏幕上。Aleph 将其安全地存储在本地保险库 `~/.aleph/` 中。

如果你选择了 **Ollama**，向导会跳过此步骤，并提醒你确保 Ollama 已在本地安装并运行。

### 阶段 4 —— 主模型

为你选择的提供商选择默认模型。选项因提供商而异。例如，使用 Anthropic：

| 模型 | 描述 |
|------|------|
| Claude Opus 4.5 | 最强大，最适合复杂任务 |
| Claude Sonnet 4 | 快速且功能强大（推荐） |
| Claude 3.5 Haiku | 最快，适合简单任务 |

### 阶段 5 —— 辅助模型（可选）

向导会询问你是否要配置故障转移模型：

```
Would you like to configure a secondary model for failover? (y/N)
```

如果你回答 **是**，你需要重复提供商、凭据和模型步骤来配置辅助提供商。当主模型不可用时，Aleph 会自动回退到辅助模型。

如果你回答 **否**（默认），向导会继续下一步。

### 阶段 6 —— 思考级别

选择 Aleph 在回复前的推理深度：

| 级别 | 描述 |
|------|------|
| Off | 不进行扩展思考 |
| Minimal | 简短推理 |
| Low | 快速分析 |
| **Medium** | 平衡 —— 适合大多数任务（推荐） |
| High | 针对复杂问题的深入分析 |
| XHigh | 最大深度 —— 较慢但全面 |

你可以随时通过 `aleph chat --thinking high` 按会话更改此设置。

### 阶段 7 —— 消息应用

选择要连接的消息平台。这是可选的 —— 你可以不按任何选项直接按 **Enter** 跳过。

| 应用 | 状态 |
|------|------|
| Telegram | 可用 |
| Discord | 可用 |
| Slack | 即将推出 |
| iMessage | 仅 macOS |

如果你选择了一个或多个应用，向导**不会**在此处收集机器人令牌。你可以在向导完成后通过 `~/.aleph/config.toml` 或 `aleph config set` 配置这些令牌。

### 阶段 8 —— 回顾

向导会显示你配置的所有内容的摘要：

```
Configuration Summary:

  Primary Provider: anthropic
  Primary Model:    claude-sonnet-4-20250514
  Secondary:        None
  Thinking Level:   medium
  Messaging Apps:   telegram
```

在继续之前仔细查看摘要。

### 阶段 9 —— 完成

```
Apply this configuration? (Y/n)
```

如果你确认，向导会验证你的 API 密钥并将配置写入 `~/.aleph/config.toml`。进度指示器会显示状态：

```
  Validating API keys...
  Saving configuration...
  Configuration saved
```

如果你拒绝，向导会退出而不保存任何内容。

### 阶段 10 —— 完成

```
Aleph is ready! Run 'aleph chat' to start.
```

完成了。使用 `aleph chat` 开始聊天，或使用 `aleph gateway run` 启动网关。

## 重新运行向导

你可以随时再次运行向导：

```bash
# 完整的入门流程 —— 会覆盖现有配置
aleph wizard

# 添加新提供商而不更改其他设置
aleph wizard --provider
```

再次运行向导会替换你完成的任何阶段的值。你跳过的设置（例如，拒绝辅助模型）会从当前配置中保持不变。

## 快速设置

如果你只想使用推荐默认值（Anthropic Claude Sonnet 4，中等思考级别）快速开始，请使用快速设置：

```bash
aleph wizard --quick
```

快速设置只询问你的 Anthropic API 密钥，并自动配置其他所有内容。大约需要 10 秒钟。

## 提示

- **API 密钥错误？** 重新运行 `aleph wizard --provider` 来更新凭据，而无需重新配置其他所有内容。
- **稍后更改思考级别：** 使用 `aleph config set agents.defaults.thinking high` 或在单个命令中传递 `--thinking`。
- **消息令牌：** 在向导完成后，使用 `aleph config set channels.telegram.token "YOUR_TOKEN"` 设置机器人令牌，然后使用 `aleph channels start telegram` 启动频道。
- **完全跳过向导：** 高级用户可以直接编辑 `~/.aleph/config.toml` —— 参见 [配置](./configuration) 获取完整参考。

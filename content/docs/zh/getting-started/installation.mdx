本指南涵盖了在你的系统上安装 Aleph。

## 前置条件

- **Rust 1.92+** - 从源代码构建需要（参见 `Cargo.toml` 中的 `rust-version`）
- **just** - 命令运行器（`cargo install just`）
- **npm** - 用于构建 WASM 面板 UI
- **wasm-bindgen** - WASM JS 绑定（`cargo install wasm-bindgen-cli`）
- **macOS 14+**、**Linux** 或 **Windows** - 主要支持的平台
- **SQLite** - 通过 `rusqlite` 捆绑（无需单独安装）

### 平台特定要求

**macOS：**
- Xcode 命令行工具
- Swift 工具链（用于桌面桥接）

**Linux：**
- `build-essential` 或等效工具
- `pkg-config`
- `libssl-dev`

**Windows：**
- Visual Studio 生成工具
- `windows-sys` 功能由 Cargo 自动处理

## 安装方法

### 桌面应用（推荐）

运行 Aleph 最简单的方式是使用原生桌面应用。它同时捆绑了核心服务器和 Tauri 壳——无需终端。

| 平台 | 下载 | 说明 |
|------|------|------|
| **macOS** | `.dmg` 从 [Releases](https://github.com/rootazero/Aleph/releases) | 通用二进制文件（Intel 和 Apple Silicon） |
| **Linux** | `.deb` / `.AppImage` 从 [Releases](https://github.com/rootazero/Aleph/releases) | Debian、Ubuntu、Fedora |
| **Windows** | `.msi` 从 [Releases](https://github.com/rootazero/Aleph/releases) | Windows 10/11 |

桌面应用：
- 将 `aleph-server` 作为后台守护进程嵌入
- 在原生 webview 中托管 Leptos Panel
- 提供系统托盘、OS 通知、全局热键（`CmdOrCtrl+Shift+A`）
- 通过 GitHub Releases 静默自动更新
- 支持开机自启

### 使用 Just（开发）

用于开发或无头服务器部署：

```bash
# 验证所有构建依赖已安装
just deps

# 以开发模式运行服务器（重建 WASM）
just dev

# 构建发布二进制文件
just build

# 运行测试
just test
```

### 从源代码

```bash
# 克隆仓库
git clone https://github.com/rootazero/Aleph.git
cd Aleph

# 验证依赖
just deps

# 以开发模式运行
just dev

# 或构建发布二进制文件
just build
```

发布二进制文件将位于 `target/release/aleph-server`。

### 桌面壳（开发）

```bash
# 以开发模式运行桌面壳（重建守护进程 + 壳）
just shell-dev

# 构建平台安装程序（.app/.dmg、.msi、.deb）
just shell-build
```

### 验证构建

```bash
# 快速编译检查
just check

# 运行核心测试
just test

# 检查所有 crate，包括桌面端
just check-desktop
```

## 目录结构

Aleph 在 `~/.aleph/` 创建以下目录：

| 路径 | 用途 |
|------|---------|
| `~/.aleph/` | 主配置目录 |
| `~/.aleph/aleph.toml` | 主配置文件 |
| `~/.aleph/logs/` | 服务器日志 |
| `~/.aleph/skills/` | 用户安装的 skill |
| `~/.aleph/plugins/` | 扩展 |

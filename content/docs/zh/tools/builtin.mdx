## 概述

Aleph 附带了一套全面的内置工具，涵盖文件操作、代码执行、网页交互、内存管理、浏览器自动化等。所有内置工具都实现 `AlephTool` trait，并使用静态分发以获得最大性能。

内置工具使用 builder 方法在 `AlephToolServer` 上注册：

```rust
let server = AlephToolServer::new()
    .with_bash()
    .with_file_ops()
    .with_search()
    .with_web_fetch()
    .with_youtube()
    .with_code_exec()
    .with_browser()
    .with_desktop()
    .with_memory_search(database)
    .with_atomic_ops(workspace_root);
```

## 代码执行

### bash

执行 bash/shell 命令。一个便捷的包装器，将命令路由到代码执行引擎，设置 `language=shell`。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `cmd` | string | 是 | 要执行的 bash 命令 |
| `working_dir` | string | 否 | 工作目录（默认为临时目录） |
| `timeout` | integer | 否 | 超时（秒）（默认：60） |

```json
{ "cmd": "find . -name '*.rs' | wc -l", "working_dir": "/project", "timeout": 30 }
```

<Callout type="warn">
危险命令如 `sudo`、`rm -rf /` 和类似的破坏性操作会被安全过滤器阻止。
</Callout>

### code_exec

在沙盒环境中执行多语言代码片段。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `language` | string | 是 | 语言：`shell`, `python`, `javascript` |
| `code` | string | 是 | 要执行的代码 |
| `working_dir` | string | 否 | 工作目录 |
| `timeout` | integer | 否 | 超时（秒）（默认：60） |

```json
{ "language": "python", "code": "import math\nprint(math.pi)" }
```

## 文件操作

### file_ops

一个统一的文件系统工具，通过 `action` 参数支持多种操作。

#### 支持的操作

| 操作 | 描述 | 必需参数 |
|------|------|----------|
| `list` | 列出目录内容 | `path` |
| `read` | 读取文件内容 | `path`, 可选 `encoding` |
| `write` | 将内容写入文件 | `path`, `content` |
| `edit` | 对文件应用有针对性的编辑 | `path`, `edits` |
| `move` | 移动/重命名文件 | `source`, `destination` |
| `copy` | 复制文件 | `source`, `destination` |
| `delete` | 删除文件或目录 | `path` |
| `mkdir` | 创建目录 | `path` |
| `search` | 按模式搜索文件 | `path`, `pattern` |

#### 参数

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `action` | string | 是 | 要执行的操作（见上文） |
| `path` | string | 因操作而异 | 文件或目录路径 |
| `content` | string | 因操作而异 | 写入操作的内容 |
| `source` | string | 因操作而异 | 移动/复制的源路径 |
| `destination` | string | 因操作而异 | 移动/复制的目标路径 |
| `encoding` | string | 否 | 文件编码（默认：`utf-8`） |
| `pattern` | string | 因操作而异 | 搜索模式（glob 或 regex） |
| `recursive` | boolean | 否 | 递归列出/搜索 |

```json
{ "action": "read", "path": "/etc/hostname" }
```

```json
{ "action": "write", "path": "/tmp/output.txt", "content": "Hello, Aleph!" }
```

```json
{ "action": "search", "path": "/project/src", "pattern": "*.rs", "recursive": true }
```

### atomic_ops

由 Atomic Engine 提供支持的原子搜索、替换和移动操作。设计用于跨工作空间的精确代码修改。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `action` | string | 是 | `search`, `replace`, 或 `move` |
| `path` | string | 是 | 文件或目录路径 |
| `pattern` | string | 因操作而异 | 搜索模式 |
| `replacement` | string | 因操作而异 | 替换文本 |
| `source` | string | 因操作而异 | 移动的源路径 |
| `destination` | string | 因操作而异 | 移动的目标路径 |

## 网页和搜索

### web_fetch

从网页 URL 获取和提取内容，并自动进行 HTML 到文本的转换。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `url` | string | 是 | 要获取的 URL |
| `method` | string | 否 | HTTP 方法（默认：`GET`） |
| `headers` | object | 否 | 自定义 HTTP 头 |

```json
{ "url": "https://api.github.com/repos/rust-lang/rust", "headers": { "Accept": "application/json" } }
```

### search

使用 SearXNG 或其他配置的搜索引擎搜索网页。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `query` | string | 是 | 搜索查询 |
| `engine` | string | 否 | 要使用的搜索引擎 |

```json
{ "query": "Rust async trait patterns 2025" }
```

### youtube

从 YouTube URL 提取视频信息和字幕。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `url` | string | 是 | YouTube 视频 URL |

```json
{ "url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ" }
```

## 浏览器自动化

### browser

通过 Chrome DevTools Protocol (CDP) 控制 Chromium 浏览器。支持启动浏览器、导航页面、与元素交互、捕获屏幕截图和获取无障碍快照。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `action` | string | 是 | 浏览器操作（见下文） |
| `tab_id` | string | 因操作而异 | 目标标签页标识符 |
| `url` | string | 因操作而异 | 导航 URL |
| `ref_id` | string | 因操作而异 | ARIA 元素 ref ID |
| `selector` | string | 因操作而异 | CSS 选择器（后备） |
| `text` | string | 因操作而异 | type/fill 操作的文本 |
| `js` | string | 因操作而异 | 要评估的 JavaScript 代码 |
| `direction` | string | 因操作而异 | 滚动方向 |
| `headless` | boolean | 否 | 以无头模式启动 |
| `full_page` | boolean | 否 | 整页屏幕截图 |

#### 操作

| 操作 | 描述 | 必需 |
|------|------|------|
| `start` | 启动浏览器实例 | 可选：`headless` |
| `stop` | 关闭浏览器 | - |
| `open_tab` | 打开新标签页 | `url` |
| `close_tab` | 关闭标签页 | `tab_id` |
| `list_tabs` | 列出所有打开的标签页 | - |
| `navigate` | 导航到 URL | `tab_id`, `url` |
| `click` | 点击元素 | `tab_id`, `ref_id` 或 `selector` |
| `type` | 在元素中输入文本 | `tab_id`, `ref_id` 或 `selector`, `text` |
| `fill` | 替换元素值 | `tab_id`, `ref_id` 或 `selector`, `text` |
| `scroll` | 滚动页面/元素 | `tab_id`, `direction` |
| `hover` | 悬停在元素上 | `tab_id`, `ref_id` 或 `selector` |
| `screenshot` | 捕获屏幕截图 | `tab_id`, 可选 `full_page` |
| `snapshot` | ARIA 无障碍树 | `tab_id` |
| `evaluate` | 运行 JavaScript | `tab_id`, `js` |

详细用法请参阅 [浏览器自动化](/docs/zh/tools/browser)。

## 内存

### memory_search

使用混合检索和智能冗余消除搜索个人记忆。

| 参数 | 类型 | 必需 | 描述 |
|------|------|------|------|
| `query` | string | 是 | 自然语言搜索查询 |
| `max_results` | integer | 否 | 最大结果数（默认：10） |

该工具在压缩事实和原始对话记录之间执行混合检索，使用 `ContextComptroller` 对结果进行去重：

```
memory_search(query)
  -> FactRetrieval.retrieve(query)
    -> 混合搜索（事实 + 原始记忆后备）
  -> ContextComptroller.arbitrate(results, budget)
    -> 通过余弦相似度检测冗余（阈值：0.95）
    -> 当事实存在时删除冗余记录
    -> 按相似度分数排序（降序）
    -> 修剪以适应 token 预算
  -> 返回去重结果
```

```json
{ "query": "What are my coding preferences?", "max_results": 10 }
```

|------|------|------|------|

## 概览

Aleph 的工具系统是一个类型安全、可扩展的框架，赋予 AI Agent 对世界进行操作的能力——执行 shell 命令、读取文件、搜索网络、控制浏览器等。每个工具都定义为带有自动 JSON Schema 生成的 Rust trait 实现，确保编译时正确性和运行时灵活性。

工具系统围绕三个关键原则构建：

1. **类型安全优先** — 工具参数和输出是强类型的 Rust 结构体。通过 `schemars` 自动从这些类型生成发送给 LLM 的 JSON Schema，消除 schema 漂移。
2. **静态 + 动态分发** — 内置工具使用零成本静态分发；MCP 和插件工具使用动态分发。一个全覆盖实现无缝桥接两者。
3. **支持热重载** — `AlephToolServer` 支持在运行时添加、移除和替换工具，无需重启 Agent。

## 架构

工具系统组织为三层：

```
┌─────────────────────────────────────────────────────────────┐
│                    AlephTool (静态)                          │
│   编译时已知的工具，带有类型化的 Args/Output                   │
│   通过 schemars 自动生成 JSON Schema                          │
└─────────────────────────────────┬───────────────────────────┘
                                   │ 全覆盖实现
                                   ▼
┌─────────────────────────────────────────────────────────────┐
│                   AlephToolDyn (动态)                        │
│   运行时分发，JSON Value 参数                                 │
│   使用者：MCP 工具、插件工具、热重载工具                      │
└─────────────────────────────────┬───────────────────────────┘
                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────┐
│                   AlephToolServer                            │
│   支持热重载的工具注册表                                      │
│   线程安全的 add/remove/list/call 操作                       │
└─────────────────────────────────────────────────────────────┘
```

**源代码位置：**
- Traits：`src/tools/traits.rs`
- 服务器：`src/tools/server.rs`
- 内置工具：`src/builtin_tools/`
- MCP 客户端：`src/mcp/`
- 浏览器运行时：`src/browser/`

## AlephTool Trait

`AlephTool` trait 是定义工具的主要接口。它使用关联类型进行编译时类型安全，并自动从 `Args` 类型生成 JSON Schema：

```rust
#[async_trait]
pub trait AlephTool: Clone + Send + Sync + 'static {
    /// 函数调用中使用的工具名称（例如 "search"、"file_read"）
    const NAME: &'static str;

    /// 供 LLM 工具选择使用的人类可读描述
    const DESCRIPTION: &'static str;

    /// 输入参数类型（必须派生 JsonSchema 以进行自动 schema 生成）
    type Args: Serialize + DeserializeOwned + JsonSchema + Send;

    /// 输出类型（序列化为 JSON 供 LLM 使用）
    type Output: Serialize + Send;

    /// 获取工具类别（默认：Builtin）
    fn category(&self) -> ToolCategory { ToolCategory::Builtin }

    /// 此工具是否需要用户确认才能执行
    fn requires_confirmation(&self) -> bool { false }

    /// 用于少样本学习上下文的用法示例
    fn examples(&self) -> Option<Vec<String>> { None }

    /// 获取带有自动生成 JSON Schema 的工具定义
    fn definition(&self) -> ToolDefinition { /* 自动实现 */ }

    /// 使用类型化参数执行工具
    async fn call(&self, args: Self::Args) -> Result<Self::Output>;

    /// 使用 JSON 参数执行（自动实现）
    async fn call_json(&self, args: Value) -> Result<Value> { /* 自动实现 */ }
}
```

### 通过 schemars 生成 Schema

工具参数类型派生 `JsonSchema`，自动生成发送给 LLM 的 JSON Schema。字段级文档注释成为参数描述：

```rust
#[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]
pub struct SearchArgs {
    /// 搜索查询（自然语言）
    pub query: String,

    /// 最大返回结果数（默认：10）
    #[serde(default)]
    pub max_results: Option<u32>,
}
```

这会生成以下 schema，作为工具定义的一部分发送给 LLM：

```json
{
  "type": "object",
  "properties": {
    "query": {
      "type": "string",
      "description": "搜索查询（自然语言）"
    },
    "max_results": {
      "type": "integer",
      "description": "最大返回结果数（默认：10）"
    }
  },
  "required": ["query"]
}
```

## 动态分发

对于运行时加载的工具（MCP 服务器、插件），Aleph 提供 `AlephToolDyn` trait：

```rust
pub trait AlephToolDyn: Send + Sync {
    fn name(&self) -> &str;
    fn definition(&self) -> ToolDefinition;
    fn call(&self, args: Value) -> Pin<Box<dyn Future<Output = Result<Value>> + Send + '_>>;
}
```

全覆盖实现确保每个 `AlephTool` 自动也是 `AlephToolDyn`：

```rust
impl<T: AlephTool> AlephToolDyn for T {
    fn name(&self) -> &str { T::NAME }
    fn definition(&self) -> ToolDefinition { AlephTool::definition(self) }
    fn call(&self, args: Value) -> Pin<Box<dyn Future<Output = Result<Value>> + Send + '_>> {
        Box::pin(async move { self.call_json(args).await })
    }
}
```

这意味着你可以在同一个集合中混合使用静态和动态工具：

```rust
let tools: Vec<Box<dyn AlephToolDyn>> = vec![
    Box::new(SearchTool::new()),     // 静态 (AlephTool)
    Box::new(mcp_tool_wrapper),      // 动态 (仅 AlephToolDyn)
];
```

## 工具类别

Aleph 中的工具分为四类：

| 类别 | 来源 | 分发 | 示例 |
|----------|--------|----------|----------|
| **内置** | `src/builtin_tools/` | 静态 | `bash`、`file_ops`、`web_fetch`、`browser` |
| **MCP** | 外部进程 | 动态 | 文件系统服务器、GitHub 服务器 |
| **扩展** | WASM/Node.js 插件 | 动态 | 自定义插件 |
| **Markdown Skill** | `SKILL.md` 文件 | 动态 | 用户定义的提示工作流 |

## AlephToolServer

`AlephToolServer` 是一个线程安全、支持热重载的注册表，管理所有工具：

```rust
pub struct AlephToolServer {
    tools: Arc<RwLock<HashMap<String, Arc<dyn AlephToolDyn>>>>,
}
```

### 注册

可以使用构建器模式或在运行时注册工具：

```rust
// 构建期间使用构建器模式
let server = AlephToolServer::new()
    .with_bash()
    .with_file_ops()
    .with_search()
    .with_web_fetch()
    .with_browser();

// 运行时注册
server.add_tool(MyCustomTool::new()).await;

// 热重载并跟踪更新
let info = server.replace_tool(UpdatedTool::new()).await;
if info.was_replaced {
    println!("Updated: {}", info.tool_name);
}
```

### 带自动修复的执行

服务器包括自动工具名称修复，灵感来自 OpenCode 的 `experimental_repairToolCall` 模式。当 LLM 发送错误的工具名称时，服务器会尝试：

1. **精确匹配** — `search` 匹配 `search`
2. **不区分大小写** — `Search` 修复为 `search`
3. **Snake-case 转换** — `WebSearch` 修复为 `web_search`
4. **无效回退** — 路由到 `invalid` 工具处理程序并显示错误消息

```rust
let (result, repair_info) = server.call_with_repair("WebSearch", args).await;
if let Some(repair) = repair_info {
    tracing::info!("Repaired {} -> {}", repair.original_name, repair.repaired_name);
}
```

### 用于并发的句柄

服务器提供轻量级、可克隆的句柄，用于跨异步任务共享：

```rust
let handle = server.handle();
tokio::spawn(async move {
    handle.call("search", args).await
});
```

## 工具过滤

Aleph 支持对 Agent 可用的工具进行细粒度控制：

```rust
pub struct ToolFilter {
    pub allowed: Option<HashSet<String>>,      // 白名单
    pub blocked: HashSet<String>,              // 黑名单
    pub require_confirmation: HashSet<String>, // 需要用户批准
}
```

通过 JSON 配置：

```json
{
  "tools": {
    "allowed": ["file_read", "web_fetch", "memory_*"],
    "blocked": ["bash"],
    "requireConfirmation": ["file_write", "file_delete"]
  }
}
```

<Callout type="info">
白名单支持 `memory_*` 等通配符模式，匹配任何以该前缀开头的工具名称。
</Callout>

## 工具结果

每个工具执行返回标准化的 `ToolResult`：

pub struct ToolResult {

## 工具进度回调

Aleph 提供全局回调系统，用于监控工具执行，支持向 UI 流式传输进度更新：

```rust
pub trait ToolProgressCallback: Send + Sync {
    fn on_tool_start(&self, tool_name: &str, args_summary: &str);
    fn on_tool_result(&self, tool_name: &str, result_summary: &str, success: bool);
}

// 在 Agent 执行前设置处理程序
set_tool_progress_handler(Some(Arc::new(MyProgressHandler)));
```

## 与 Agent 循环集成

当 LLM 以 `tool_use` 块响应时，Agent 循环：

1. **解析** 工具名称通过 `AlephToolServer`（带自动修复）
2. **过滤** 调用通过 `ToolFilter`（白名单/黑名单检查）
3. **检查确认** 对于标记为需要用户批准的工具
4. **执行** 工具使用提供的 JSON 参数
5. **返回** `ToolResult` 给 LLM 作为 `tool_result` 消息

此循环重复直到 LLM 产生没有工具调用的最终文本响应。

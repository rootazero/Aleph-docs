## 概述

你可以通过创建自定义工具来扩展 Aleph 的功能。有三种方法，按集成深度排序：

| 方法 | 语言 | 注册 | 最适合 |
|------|------|------|--------|
| **Rust 内置** | Rust | 编译时 | 核心功能，最大性能 |
| **MCP 服务器** | 任意（Node.js, Python 等） | 运行时 | 外部集成，多语言团队 |
| **Markdown 技能** | Markdown + 提示 | 运行时 | 基于提示的工作流，无需代码 |

---

## 创建 Rust 内置工具

这是最集成的方法。你的工具被编译到 Aleph 的二进制文件中，并使用静态分发实现零成本抽象。

### 步骤 1：定义参数类型

为你的工具的输入参数创建一个结构体。派生 `JsonSchema` 以自动生成模式 —— 每个字段上的文档注释会成为 LLM 看到的参数描述：

```rust
// src/builtin_tools/weather.rs

use schemars::JsonSchema;
use serde::{Deserialize, Serialize};

/// 天气工具的参数
#[derive(Debug, Clone, Serialize, Deserialize, JsonSchema)]
pub struct WeatherArgs {
    /// 城市名称或坐标（例如 "Tokyo" 或 "35.6762,139.6503"）
    pub location: String,

    /// 温度单位："celsius" 或 "fahrenheit"（默认：celsius）
    #[serde(default = "default_unit")]
    pub unit: String,

    /// 预报天数（1-7，默认：1）
    #[serde(default = "default_days")]
    pub days: u8,
}

fn default_unit() -> String { "celsius".to_string() }
fn default_days() -> u8 { 1 }
```

### 步骤 2：定义输出类型

输出结构体被序列化为 JSON 并返回给 LLM：

```rust
/// 天气工具的输出
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct WeatherOutput {
    /// 城市名称
    pub city: String,
    /// 当前温度
    pub temperature: f64,
    /// 温度单位
    pub unit: String,
    /// 天气状况（例如 "sunny", "cloudy", "rain"）
    pub condition: String,
    /// 未来几天预报
    pub forecast: Vec<ForecastDay>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ForecastDay {
    pub date: String,
    pub high: f64,
    pub low: f64,
    pub condition: String,
}
```

### 步骤 3：实现 AlephTool Trait

```rust
use async_trait::async_trait;
use crate::error::Result;
use crate::tools::AlephTool;

/// 天气信息工具
#[derive(Clone)]
pub struct WeatherTool {
    api_key: String,
    client: reqwest::Client,
}

impl WeatherTool {
    pub fn new(api_key: String) -> Self {
        Self {
            api_key,
            client: reqwest::Client::new(),
        }
    }
}

#[async_trait]
impl AlephTool for WeatherTool {
    const NAME: &'static str = "weather";

    const DESCRIPTION: &'static str = "\
        Get current weather and forecast for a location. \
        Returns temperature, conditions, and multi-day forecast.";

    type Args = WeatherArgs;
    type Output = WeatherOutput;

    /// 可选：标记为需要确认以进行敏感操作
    fn requires_confirmation(&self) -> bool {
        false
    }

    /// 可选：提供使用示例以进行少样本学习
    fn examples(&self) -> Option<Vec<String>> {
        Some(vec![
            "weather(location='Tokyo', unit='celsius', days=3)".to_string(),
            "weather(location='New York')".to_string(),
            "weather(location='51.5074,-0.1278', unit='fahrenheit')".to_string(),
        ])
    }

    async fn call(&self, args: Self::Args) -> Result<Self::Output> {
        // 你的实现在这里
        let response = self.client
            .get("https://api.weather.example.com/v1/forecast")
            .query(&[
                ("location", &args.location),
                ("unit", &args.unit),
                ("days", &args.days.to_string()),
                ("key", &self.api_key),
            ])
            .send()
            .await?
            .json::<WeatherOutput>()
            .await?;

        Ok(response)
    }
}
```

### 步骤 4：注册工具

在 `src/tools/builtin.rs` 中为 `AlephToolServer` 添加一个 builder 方法：

```rust
impl AlephToolServer {
    /// 注册天气工具
    pub fn with_weather(self, api_key: String) -> Self {
        self.tool(WeatherTool::new(api_key))
    }
}
```

然后在服务器构建期间注册它：

```rust
let server = AlephToolServer::new()
    .with_bash()
    .with_file_ops()
    .with_weather(weather_api_key);
```

### 步骤 5：从模块导出

将你的工具添加到 `src/builtin_tools/mod.rs`：

```rust
pub mod weather;
pub use weather::{WeatherArgs, WeatherOutput, WeatherTool};
```

### 生成的 JSON Schema

当你的工具被注册时，Aleph 会自动生成此工具定义供 LLM 使用：

```json
{
  "type": "function",
  "function": {
    "name": "weather",
    "description": "Get current weather and forecast for a location. Returns temperature, conditions, and multi-day forecast.",
    "parameters": {
      "type": "object",
      "properties": {
        "location": {
          "type": "string",
          "description": "City name or coordinates (e.g., \"Tokyo\" or \"35.6762,139.6503\")"
        },
        "unit": {
          "type": "string",
          "description": "Temperature unit: \"celsius\" or \"fahrenheit\" (default: celsius)",
          "default": "celsius"
        },
        "days": {
          "type": "integer",
          "description": "Number of forecast days (1-7, default: 1)",
          "default": 1
        }
      },
      "required": ["location"]
    }
  }
}
```

`required` 数组是自动计算的 —— 只有没有 `#[serde(default)]` 的字段才是必需的。

## 创建 MCP 服务器工具

对于 Rust 以外的语言，或者当你想要在不重新编译 Aleph 的情况下进行迭代时，创建一个 MCP 服务器。服务器作为单独进程运行，并通过模型上下文协议与 Aleph 通信。

### Node.js 示例

使用官方的 `@modelcontextprotocol/sdk`：

```typescript
// weather-server/index.ts
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

const server = new Server(
  { name: "weather-server", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

// 定义工具
server.setRequestHandler("tools/list", async () => ({
  tools: [
    {
      name: "get_weather",
      description: "Get current weather for a location",
      inputSchema: {
        type: "object",
        properties: {
          location: {
            type: "string",
            description: "City name or coordinates",
          },
          unit: {
            type: "string",
            enum: ["celsius", "fahrenheit"],
            default: "celsius",
          },
        },
        required: ["location"],
      },
    },
  ],
}));

// 处理工具调用
server.setRequestHandler("tools/call", async (request) => {
  if (request.params.name === "get_weather") {
    const { location, unit } = request.params.arguments;

    const response = await fetch(
      `https://api.weather.example.com/v1?q=${location}&units=${unit}`
    );
    const data = await response.json();

    return {
      content: [
        {
          type: "text",
          text: JSON.stringify(data, null, 2),
        },
      ],
    };
  }

  throw new Error(`Unknown tool: ${request.params.name}`);
});

// 启动服务器
const transport = new StdioServerTransport();
await server.connect(transport);
```

### Python 示例

使用官方的 `mcp` 包：

```python
# weather_server.py
import json
import httpx
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent

app = Server("weather-server")

@app.list_tools()
async def list_tools():
    return [
        Tool(
            name="get_weather",
            description="Get current weather for a location",
            inputSchema={
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City name or coordinates",
                    },
                    "unit": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "default": "celsius",
                    },
                },
                "required": ["location"],
            },
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "get_weather":
        location = arguments["location"]
        unit = arguments.get("unit", "celsius")

        async with httpx.AsyncClient() as client:
            resp = await client.get(
                "https://api.weather.example.com/v1",
                params={"q": location, "units": unit},
            )
            data = resp.json()

        return [TextContent(type="text", text=json.dumps(data, indent=2))]

    raise ValueError(f"Unknown tool: {name}")

async def main():
    async with stdio_server() as (read, write):
        await app.run(read, write)

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

### 注册到 Aleph

将服务器添加到 `~/.aleph/mcp_config.json`：

```json
{
  "version": 1,
  "servers": {
    "weather": {
      "id": "weather",
      "name": "Weather Server",
      "transport": "stdio",
      "command": "npx",
      "args": ["tsx", "/path/to/weather-server/index.ts"],
      "requires_runtime": "node",
      "auto_start": true,
      "timeout_seconds": 30
    }
  }
}
```

对于 Python 示例：

```json
{
  "weather-py": {
    "id": "weather-py",
    "name": "Weather Server (Python)",
    "transport": "stdio",
    "command": "python",
    "args": ["/path/to/weather_server.py"],
    "requires_runtime": "python",
    "auto_start": true

handle.add_server(config).await?;
```

## 创建 Markdown 技能

Markdown 技能是扩展 Aleph 的最简单方式 —— 无需代码。它们在 `SKILL.md` 文件中定义基于提示的工作流。

### 技能文件结构

在技能目录之一中创建 `SKILL.md` 文件：

```markdown
---
name: summarize_article
description: Summarize a web article into key points
---

# Summarize Article

Given the URL: {{url}}

Please:
1. Fetch the content from the URL
2. Identify the main topic and key arguments
3. Produce a concise summary with bullet points
4. Note any important data or statistics mentioned

Output the summary in markdown format.
```

### 技能目录

Aleph 从配置的目录加载技能：

```rust
let server = AlephToolServer::new_with_skills(vec![
    PathBuf::from("skills"),
    PathBuf::from("~/.aleph/skills"),
]).await;
```

技能在启动时自动加载，并在 `AlephToolServer` 中注册为动态工具。

<Callout type="info">
Markdown 技能通过文件 watcher 支持热重载。对 `SKILL.md` 文件的更改会被检测到，并且技能会自动重新注册，无需重启 Aleph。
</Callout>

## 工具设计最佳实践

### 命名约定

- 对工具名称使用 `snake_case`：`web_fetch`, `file_read`
- 描述性但简洁：`memory_search` 而不是 `search_memory_database`
- 对相关工具使用通用前缀：`sessions_spawn`, `sessions_send`, `sessions_list`

### 描述质量

工具描述至关重要 —— 它是 LLM 用来决定是否调用你的工具的依据。编写清晰、具体的描述：

```rust
// 好：具体，解释它做什么和返回什么
const DESCRIPTION: &'static str = "\
    Get current weather and forecast for a location. \
    Returns temperature, conditions, and multi-day forecast.";

// 差：模糊，不解释输出
const DESCRIPTION: &'static str = "Weather tool";
```

### 少样本示例

提供示例可以显著提高 LLM 正确使用你的工具的能力：

```rust
fn examples(&self) -> Option<Vec<String>> {
    Some(vec![
        "weather(location='Tokyo', unit='celsius', days=3)".to_string(),
        "weather(location='New York')".to_string(),
    ])
}
```

示例被注入到工具定义的 `llm_context` 字段中，并与模式一起展示给 LLM。

### 错误处理

工具应返回 `Result<Self::Output>` 并使用 Aleph 的错误类型：

```rust
async fn call(&self, args: Self::Args) -> Result<Self::Output> {
    if args.days > 7 {
        return Err(AlephError::InvalidArgument(
            "days must be between 1 and 7".to_string()
        ));
    }

    let response = self.client.get(&url).send().await
        .map_err(|e| AlephError::ToolExecution(format!("HTTP request failed: {e}")))?;

    Ok(output)
}
```

### 破坏性操作的确认

标记执行破坏性或敏感操作的工具：

```rust
fn requires_confirmation(&self) -> bool {
    true  // 用户必须在执行前批准
}
```

这与审批策略系统集成 —— 当配置时，agent 循环会暂停并在执行工具前请求用户确认。

## 测试自定义工具

### 单元测试

直接测试 `AlephTool` 实现：

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_tool_definition() {
        let tool = WeatherTool::new("test_key".to_string());
        let def = AlephTool::definition(&tool);

        assert_eq!(def.name, "weather");
        assert!(def.description.contains("weather"));
        assert!(!def.requires_confirmation);
    }

    #[tokio::test]
    async fn test_tool_call() {
        let tool = WeatherTool::new("test_key".to_string());
        let result = AlephTool::call(&tool, WeatherArgs {
            location: "Tokyo".to_string(),
            unit: "celsius".to_string(),
            days: 1,
        }).await;

        assert!(result.is_ok());
    }

    #[tokio::test]
    async fn test_tool_call_json() {
        let tool = WeatherTool::new("test_key".to_string());
        let args = serde_json::json!({
            "location": "London",
            "unit": "celsius"
        });

        let result = AlephTool::call_json(&tool, args).await;
        assert!(result.is_ok());
    }
}
```

### 使用 AlephToolServer 进行集成测试

测试完整的工具服务器集成：

```rust
#[tokio::test]
async fn test_tool_server_integration() {
    let server = AlephToolServer::new()
        .with_weather("test_key".to_string());

    assert!(server.has_tool("weather").await);
    assert_eq!(server.len().await, 1);

    let result = server.call(
        "weather",
        serde_json::json!({ "location": "Paris" })
    ).await;

    assert!(result.is_ok());
}
```

### 测试动态分发

验证 blanket `AlephToolDyn` 实现：

```rust
#[tokio::test]
async fn test_dynamic_dispatch() {
    let tool: Box<dyn AlephToolDyn> = Box::new(WeatherTool::new("key".to_string()));

    assert_eq!(tool.name(), "weather");

    let args = serde_json::json!({ "location": "Berlin" });
    let result = tool.call(args).await;
    assert!(result.is_ok());
}
```

### 测试名称修复

验证自动修复系统是否正确处理你的工具名称：

```rust
#[tokio::test]
async fn test_name_repair() {
    let server = AlephToolServer::new()
        .with_weather("key".to_string());

    // Snake case 修复
    let repaired = server.try_repair_tool_name("Weather").await;
    assert_eq!(repaired, Some("weather".to_string()));
}
```

## 总结

| 要构建什么 | 使用此方法 |
|-----------|-----------|
| 高性能核心工具 | Rust `AlephTool` trait |
| 与外部 API 集成 | MCP 服务器（Node.js/Python） |
| 基于提示的工作流 | Markdown 技能 |
| 需要热重载的工具 | MCP 服务器或 Markdown 技能 |
| 需要类型安全的工具 | Rust `AlephTool` trait |

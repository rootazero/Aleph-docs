## 概览

Aleph 的沙箱系统在工具执行周围强制执行细粒度的安全边界。不仅仅依赖审批提示来防止损害，沙箱在操作系统级别限制*进程可以做什么*——即使命令本身已被批准。

沙箱模型是**基于能力**的：每个工具声明它需要的权限（文件系统路径、网络访问、内存限制），系统生成一个平台原生的沙箱配置文件，强制执行这些边界。声明 "对 `/tmp` 只读访问" 的工具物理上无法写入 `/tmp`，无论它运行什么命令。

**源代码位置：**
- 能力：`src/sandbox/capabilities.rs`
- 工作空间：`src/sandbox/workspace.rs`
- 驱动：`src/sandbox/driver.rs`
- 工厂：`src/sandbox/factory.rs`
- 配置：`src/sandbox/config.rs`
- macOS 适配器：`src/exec/sandbox/executor.rs`（旧路径）
- 审批门：`src/sandbox/exec_approval/gate.rs`
- 审计日志：`src/tools/middleware/audit.rs`

## 能力模型

`Capabilities` 结构体定义沙箱的四个维度：

```rust
pub struct Capabilities {
    pub filesystem: Vec<FileSystemCapability>,
    pub network: NetworkCapability,
    pub process: ProcessCapability,
    pub environment: EnvironmentCapability,
}
```

### 文件系统能力

每个文件系统能力授予对特定路径的特定模式的访问：

```rust
pub enum FileSystemCapability {
    ReadOnly { path: PathBuf },   // 对路径树的只读访问
    ReadWrite { path: PathBuf },  // 对路径树的完全读/写访问
    TempWorkspace,                // 自动创建的临时目录
}
```

**TempWorkspace** 在系统临时路径下创建唯一的临时目录（例如 `/tmp/aleph-sandbox-Xk9mRa/`）。执行完成后自动清理该目录。

### 网络能力

```rust
pub enum NetworkCapability {
    Deny,                        // 无网络访问
    AllowDomains(Vec<String>),   // 仅允许特定域
    AllowAll,                    // 无限制的网络访问
}
```

使用 `AllowDomains` 时，沙箱配置文件默认拒绝所有网络，并为每个域添加显式的允许规则：

```
(deny network*)
(allow network-outbound (remote tcp "api.github.com:*"))
(allow network-outbound (remote tcp "registry.npmjs.org:*"))
```

### 进程能力

```rust
pub struct ProcessCapability {
    pub no_fork: bool,            // 防止生成子进程
    pub max_execution_time: u64,  // 超时（秒）
    pub max_memory_mb: Option<u64>,  // 内存限制
}
```

### 环境能力

```rust
pub enum EnvironmentCapability {
    None,        // 无环境变量
    Restricted,  // 仅安全变量（PATH、HOME、USER）
    Full,        // 完整环境传递
}
```

### 默认值

默认能力是保守的：

```rust
impl Default for Capabilities {
    fn default() -> Self {
        Self {
            filesystem: vec![FileSystemCapability::TempWorkspace],
            network: NetworkCapability::Deny,
            process: ProcessCapability {
                no_fork: true,
                max_execution_time: 300,  // 5 分钟
                max_memory_mb: Some(512),
            },
            environment: EnvironmentCapability::Restricted,
        }
    }
}
```

<Callout type="warn">
默认能力仅授予对临时工作空间的文件系统访问，并拒绝所有网络访问。工具必须显式请求它们需要的任何访问权限。
</Callout>

## 能力预设

为简化工具声明，系统提供定义常见能力配置文件的预设模板：

### `file_processor`

用于处理本地文件而不需要网络访问的工具。

| 维度 | 值 |
|-----------|-------|
| 文件系统 | TempWorkspace |
| 网络 | Deny |
| 进程 | no_fork、300 秒超时、512MB 内存 |
| 环境 | Restricted |
| 不可变 | `network`（不能覆盖为允许） |

### `web_scraper`

用于从网络获取数据的工具。

| 维度 | 值 |
|-----------|-------|
| 文件系统 | TempWorkspace |
| 网络 | AllowAll |
| 进程 | no_fork、600 秒超时、1024MB 内存 |
| 环境 | Restricted |
| 不可变 | `filesystem`（不能添加任意路径） |

### `code_analyzer`

用于读取项目源代码的静态分析工具。

| 维度 | 值 |
|-----------|-------|
| 文件系统 | ReadOnly `${WORKSPACE}` |
| 网络 | Deny |
| 进程 | no_fork、900 秒超时、2048MB 内存 |
| 环境 | Restricted |
| 不可变 | `network` |

### `data_transformer`

用于需要读取数据访问权限和可写工作空间的数据处理工具。

| 维度 | 值 |
|-----------|-------|
| 文件系统 | TempWorkspace + ReadOnly `${PROJECT_ROOT}/data` |
| 网络 | Deny |
| 进程 | no_fork、1800 秒超时、4096MB 内存 |
| 环境 | Restricted |
| 不可变 | 无 |

### 不可变字段

每个预设可以声明不能被覆盖的字段。例如，`file_processor` 预设将 `network` 标记为不可变，意味着使用此预设的工具不能将其覆盖为 `AllowAll`。这防止了通过覆盖滥用进行权限升级。

## 参数绑定

工具可以基于其输入参数声明动态文件系统访问。例如，文件转换工具可能接收输入路径和输出路径——沙箱应该授予输入的读取权限和输出的写入权限：

```rust
pub struct ParameterBinding {
    pub capability: String,      // "filesystem.read_only" 或 "filesystem.read_write"
    pub validation: ValidationRule,  // is_file、is_directory、is_path
    pub mapping: MappingType,    // single 或 each_element（用于数组）
}
```

### 示例声明

```rust
RequiredCapabilities {
    base_preset: "file_processor",
    description: "转换图像格式",
    overrides: CapabilityOverrides::default(),
    parameter_bindings: {
        "input_path" => ParameterBinding {
            capability: "filesystem.read_only",
            validation: ValidationRule::IsFile,
            mapping: MappingType::Single,
        },
        "output_dir" => ParameterBinding {
            capability: "filesystem.read_write",
            validation: ValidationRule::IsDirectory,
            mapping: MappingType::Single,
        },
    },
}
```

在执行时，能力解析器将实际参数值替换到文件系统能力中，生成具有真实路径的具体 `Capabilities` 实例。

## 信任阶段

沙箱系统通过三个阶段实现渐进信任：

```rust
pub enum TrustStage {
    Draft,     // 新生成的工具，等待首次审批
    Trial,     // 已批准，等待首次执行确认
    Verified,  // 已执行多次，进入静默模式
}
```

### 阶段进展

```
Draft ──[所有者批准能力]──► Trial ──[N 次成功执行]──► Verified
```

- **Draft**：工具的能力声明显示给所有者进行审核。不允许执行。
- **Trial**：工具可以执行，但每次执行都会触发一个审批提示，显示工具将要做什么。
- **Verified**：工具在没有提示的情况下静默执行，使用先前批准的能力。

### 升级

如果已验证的工具尝试超过其声明的能力，系统会触发升级：

```rust
pub enum EscalationReason {
    PathOutOfScope,       // 参数路径超出批准范围
    SensitiveDirectory,   // 访问 ~/.ssh、~/.gnupg、~/.aws 等
    UndeclaredBinding,    // 使用绑定中未声明的参数
    FirstExecution,       // Trial 阶段工具的首次运行
}
```

升级将工具重置为 Draft 阶段，需要重新审批：

```rust
pub struct EscalationTrigger {
    pub reason: EscalationReason,
    pub requested_path: Option<PathBuf>,
    pub approved_paths: Vec<String>,
}
```

### 敏感目录检测

升级系统包含一个内置的敏感目录列表，始终触发升级：

| 目录 | 内容 |
|-----------|---------|
| `~/.ssh/` | SSH 密钥和配置 |
| `~/.gnupg/` | GPG 密钥 |
| `~/.aws/` | AWS 凭据 |
| `Keychain.app` | macOS 钥匙串 |
| `~/.config/gcloud` | Google Cloud 凭据 |

## 平台：macOS sandbox-exec

在 macOS 上，使用 Apple 的 `sandbox-exec` 工具强制执行沙箱，使用 Seatbelt 配置文件语言（.sb 文件）。

### 配置文件生成

macOS 适配器从 `Capabilities` 结构体生成 Seatbelt 配置文件：

```scheme
(version 1)
(allow default)

;; 文件系统限制
(deny file-write* (subpath "/Users/alice/project"))

;; 网络限制
(deny network*)
```

<Callout type="info">
配置文件以 `(allow default)` 开头并有选择地拒绝操作，而不是以 `(deny default)` 开头并有选择地允许。这是因为 macOS `sandbox-exec` 通过信号终止尝试不允许的操作的进程，这会破坏 `echo` 和 `sh` 等基本工具。
</Callout>

### 执行

命令用 `sandbox-exec` 包装：

```bash
sandbox-exec -f /tmp/aleph-profile-Xk9m.sb echo "hello"
```

适配器处理：
1. 将配置文件写入临时 `.sb` 文件
2. 通过 `sandbox-exec -f <profile>` 执行命令
3. 捕获 stdout、stderr 和带超时的退出代码
4. 清理配置文件和临时工作空间

### 清理

执行后（无论成功还是失败），适配器删除：
- 临时沙箱配置文件
- 临时工作空间目录（如果使用了 TempWorkspace）

```rust
fn cleanup(&self, profile: &SandboxProfile) -> Result<()> {
    if profile.path.exists() {
        std::fs::remove_file(&profile.path)?;
    }
    if let Some(ref workspace) = profile.temp_workspace {
        if workspace.exists() {
            std::fs::remove_dir_all(workspace)?;
        }
    }
    Ok(())
}
```

## 回退策略

当沙箱强制执行不可用时（例如在 Linux 上 `sandbox-exec` 不存在），`FallbackPolicy` 确定行为：

```rust
pub enum FallbackPolicy {
    Deny,              // 拒绝执行（默认）
    RequestApproval,   // 在无沙箱执行前询问用户
    WarnAndExecute,    // 记录警告并执行而不使用沙箱
}
```

<Callout type="warn">
默认回退策略是 `Deny`。将其更改为 `WarnAndExecute` 会在不支持平台的平台上完全移除沙箱保护。这应仅在受信任的开发环境中使用。
</Callout>

## Shell 命令沙箱

除了工具的基于能力的沙箱之外，Aleph 还通过 `CodeExecutor` 为直接 shell 命令执行提供单独的沙箱层：

### 命令拦截

基于正则的 `CommandChecker` 在执行前拦截已知的危险模式：

```rust
const DEFAULT_BLOCKED: &[&str] = &[
    r"rm\s+-rf\s+/\s*$",             // rm -rf /
    r"sudo\s+",                      // 任何 sudo 命令
    r"chmod\s+777\s+/",              // chmod 777 /
    r":\(\)\s*\{\s*:\|:\u0026\s*\}\s*;:", // fork bomb
    r">\s*/dev/sd[a-z]",             // 覆盖磁盘
    r"mkfs\.",                       // 格式化文件系统
    r"dd\s+if=.*of=/dev/",           // dd 到设备
];
```

### 路径权限检查

`PathPermissionChecker` 使用 glob 模式根据允许和拒绝列表验证文件路径：

```rust
pub struct PathPermissionChecker {
    allowed_patterns: Vec<Pattern>,   // 允许的路径 glob
    denied_patterns: Vec<Pattern>,    // 拒绝的路径 glob（优先检查）
    max_file_size: u64,              // 文件操作的大小限制
}
```

默认拒绝路径包括敏感凭据目录：

| 模式 | 受保护内容 |
|---------|-------------------|
| `~/.ssh/**` | SSH 密钥 |
| `~/.gnupg/**` | GPG 密钥 |
| `~/.aws/**` | AWS 凭据 |
| `~/.kube/**` | Kubernetes 配置 |
| `/etc/passwd` | 系统用户 |
| `/etc/shadow` | 密码哈希 |
| `/etc/sudoers` | Sudo 配置 |

<Callout type="info">
拒绝路径优先于允许路径。即使 `~/**` 在允许列表中，`~/.ssh/**` 仍会被默认拒绝规则拒绝。
</Callout>

### 路径遍历检测

系统检测并拒绝路径遍历尝试：

```rust
fn has_path_traversal(&self, path: &Path) -> bool {
    let path_str = path.to_string_lossy();
    path_str.contains("..") || path_str.contains("./")
}
```

对 `/tmp/../etc/passwd` 的请求在路径甚至未被检查允许/拒绝列表之前就被拒绝。

### 运行时限制

`CodeExecutor` 支持限制哪些语言运行时可用：

```rust
pub struct CodeExecutor {
    allowed_runtimes: Vec<String>,  // 空 = 允许所有
    timeout_seconds: u64,           // 执行超时
    // ...
}
```

如果 `allowed_runtimes` 设置为 `["bash", "python3"]`，尝试通过 `node` 执行 JavaScript 将被拒绝。

## 审计日志

每个沙箱执行都记录在结构化审计日志中：

```rust
pub struct SandboxAuditLog {
    pub skill_id: String,
    pub capabilities: Capabilities,
    pub status: ExecutionStatus,
    pub platform: String,
    pub timestamp: DateTime<Utc>,
}

pub enum ExecutionStatus {
    Success { exit_code: i32, duration_ms: u64 },
    Timeout { duration_ms: u64 },
    Error { error: String },
}
```

这提供了关于授予了什么能力、什么平台强制执行了它们以及执行是否成功的完整记录。

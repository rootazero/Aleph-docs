---
title: 五层涌现
description: L1-L5 模型描述了智能如何在 Aleph 中涌现 —— 从原始知识通过领域分类、原子技能、功能模块到多态代理。
---

Aleph 的智能不是一块单一的能力。它是一个从五个不同层涌现的特性，每个层都建立在前一个层之上。这个模型 —— 五层涌现 —— 既是理解 AI 智能的概念框架，也是 Aleph 构建方式的实际蓝图。

像 LEGO 积木从混乱中组装成创造物一样，Aleph 中的智能通过渐进结构涌现。每一层都将前一层的输出转化为质上不同的东西。

## 概览

```
┌──────────────────────────────────────────────────────────────┐
│  L5: 多态代理                                        ℵ₃      │
│  灵魂有了外壳 —— 自主、自适应、具身化                       │
├──────────────────────────────────────────────────────────────┤
│  L4: 功能模块                                        ℵ₂      │
│  可组合构建块 —— 即插即用的能力                             │
├──────────────────────────────────────────────────────────────┤
│  L3: 原子技能                                        ℵ₁      │
│  知变为行 —— 知识转化为行动                               │
├──────────────────────────────────────────────────────────────┤
│  L2: 领域分类                                        ℵ₀      │
│  结构涌现 —— 知识获得类别和上下文                           │
├──────────────────────────────────────────────────────────────┤
│  L1: 知识之海                                               │
│  原始海洋 —— 训练数据、文本、代码、历史、智慧               │
└──────────────────────────────────────────────────────────────┘
```

每一层对应超限数学中的一个阿列夫数，代表 progressively larger order of infinity。代理从基础向上攀登 —— 但所有层同时保持活动。高层不会取代低层；它们建立在低层之上。

## 阿列夫阶梯

层与阿列夫数之间的对应是故意的：

| 层 | 阿列夫数 | 含义 |
|-------|-------------|---------|
| L1 | -- | 原始、非结构化知识（前可数） |
| L2 | ℵ₀ | 可数、分类的知识 |
| L3 | ℵ₁ | 可操作的技能（更高基数的能力） |
| L4 | ℵ₂ | 可组合模块（可组合的能力） |
| L5 | ℵ₃ | 多态代理（自主、自适应实体） |

就像每个阿列夫数代表比前一个严格更大的无穷大一样，每一层代表一种不能还原到其下层的质上不同的能力。

## L1：知识之海

**人类经验的原始海洋。**

第 1 层是基础 —— 构成所有 AI 能力基础的庞大、非结构化知识语料库。这包括：

- **训练数据**：大型语言模型训练用的文本、代码、科学文献、历史记录和对话模式。
- **检索上下文**：通过 RAG（检索增强生成）和记忆查询在运行时获取的文档、文件和数据。
- **用户输入**：用户提供的原始消息、命令和上下文流。

在这一层，知识存在但没有结构。它是一个 token 的海洋 —— 总体强大，但没有更高层来组织它就缺乏方向。

### 在 Aleph 中

L1 表现为为思考器组件提供动力的 LLM 提供商（OpenAI、Anthropic、本地模型），结合记忆系统的原始事实存储。思考器可以通过其提供商连接访问这个知识之海，记忆系统存储和检索上下文事实。

```
知识之海 = LLM 训练数据 + 记忆事实 + 用户上下文
```

## L2：领域分类

**静态知识获得结构。**

第 2 层获取原始海洋并将其组织成领域。知识不再是平坦的广阔 —— 它有类别、边界和关系：

- **医学知识**：症状、诊断、治疗、药物相互作用
- **法律知识**：法规、判例、程序、司法管辖区
- **编程知识**：语言、框架、模式、最佳实践
- **科学知识**：物理、化学、生物、数学

领域分类允许 AI 知道编码上下文中关于"Python"的问题意味着编程语言，而非蛇。它为原始知识提供赋予意义的结构支柱。

### 在 Aleph 中

L2 通过意图检测系统和记忆系统的上下文锚定表现出来。当用户发送消息时，意图系统将其分类为领域并适当地路由它。记忆系统使用 `ContextAnchor` 值对象为事实标记领域上下文，实现相关检索。

领域分类还驱动代理选择适当工具的能力。编码问题路由到代码执行工具；调度问题路由到日历工具；研究问题路由到网络搜索工具。

## L3：原子技能

**知变为行。**

这是关键转换发生的地方：知识转化为能力。原子技能是系统可以可靠执行的离散、可重用能力：

- **总结文档** —— 不仅知道总结是什么，而且能够很好地完成它。
- **编写单元测试** —— 不仅知道测试模式，而且产生正确、可运行的测试。
- **解析日志文件** —— 不仅理解日志格式，而且从它们中提取结构化数据。
- **起草电子邮件** —— 不仅知道电子邮件约定，而且产生上下文适当的消息。

原子技能是智能行为的构建块。它们小到足以可靠，具体到足以可测试，通用到足以跨上下文可重用。

### 在 Aleph 中

L3 通过经验固化系统表现出来。当代理成功完成任务时，结果作为经验条目记录在向量数据库中。当类似任务重复出现时，检索和应用这些经验：

```
单次成功        -->  经验条目（存储在向量数据库中）
3+ 类似成功     -->  候选技能（检测到模式）
5+ 成功         -->  永久技能（提升为可重用）
```

技能提升管道将临时成功转化为可靠、可重用的能力。这就是 Aleph 真正学习的方式 —— 不仅通过积累聊天历史，而是通过将成功模式固化为可执行技能。

技能也通过扩展系统暴露，开发人员可以在其中注册自定义技能，绕过固化过程直接提供能力。

## L4：功能模块

**技能成为可组合构建块。**

第 4 层获取原子技能并将它们组合成更高级别的模块 —— 将多个技能组合起来以完成复杂任务的功能单元：

- **代码审查模块**：读取代码 + 分析模式 + 检查风格 + 生成反馈
- **研究模块**：搜索网络 + 总结来源 + 交叉引用 + 综合报告
- **DevOps 模块**：检查系统状态 + 分析日志 + 诊断问题 + 执行修复
- **通信模块**：理解上下文 + 起草消息 + 调整语气 + 为通道格式化

功能模块是"即插即用"层。它们封装了否则需要手动编排单个技能的工作流。

### 在 Aleph 中

L4 通过调度器的任务图系统表现出来。调度器将复杂请求分解为 DAG（有向无环图）子任务，其中每个节点都是原子技能或工具调用。`TaskGraph` 聚合根管理这些组合工作流的执行顺序、依赖项和状态。

```
用户请求："审查此 PR 并修复任何问题"
                    │
        ┌───────────┴───────────┐
        │     TaskGraph (L4)    │
        ├───────────────────────┤
        │  1. 读取 PR diff      │ ─── L3 技能
        │  2. 分析更改          │ ─── L3 技能
        │  3. 运行测试          │ ─── L3 技能
        │  4. 修复问题          │ ─── L3 技能
        │  5. 编写摘要          │ ─── L3 技能
        └───────────────────────┘
```

MCP（模型上下文协议）集成也在 L4 运行，允许外部工具和服务作为功能模块组合到 Aleph 的工作流中。

## L5：多态代理

**灵魂有了外壳。**

第 5 层是一切汇集的地方。多态代理是一个可以：

- **感知**其环境通过多个接口（CLI、桌面、消息、语音）
- **思考**使用双过程认知架构（系统 1 + 系统 2）
- **行动**通过组合功能模块来实现目标
- **学习**通过固化经验和进化其技能集
- **适应**其行为、个性和方法以适应不同的上下文和通道

"多态"这个词是关键：相同的底层智能根据接口和上下文以不同的方式表现出来。在 Telegram 上，它可能简洁且对话式。在 CLI 上，它可能精确且面向工具。在桌面上，它可能使用丰富的格式和可视化。智能是相同的 —— 表现形式会改变。

### 在 Aleph 中

L5 是协同工作的完整系统：Gateway 管理多通道连接，代理循环编排观察-思考-行动-反馈循环，记忆系统提供持久上下文，调度器将技能组合到工作流中，安全系统确保安全的自主操作。

```
L5 多态代理 = Gateway + 代理循环 + 记忆 + 调度器 + 安全
             (I/O)    (认知)    (状态)   (行动)    (控制)
```

这是 POE（原则-操作-评估）架构在其最高级别运行的地方。代理不仅响应提示 —— 它有目的地追求目标，评估自己的性能，并从结果中学习。

## 实践中的涌现

五层模型不仅仅是理论性的。它描述了 Aleph 代码库中的具体机制：

| 层 | Aleph 组件 | 关键职责 |
|-------|----------------|-------------------|
| L1 | 思考器 + 记忆存储 | LLM 访问 + 原始事实存储 |
| L2 | 意图系统 + 上下文锚点 | 领域分类 + 路由 |
| L3 | 经验固化器 + 技能 | 模式提取 + 技能提升 |
| L4 | 调度器 + TaskGraph + MCP | 任务组合 + 工作流编排 |
| L5 | 代理循环 + Gateway + POE | 自主行为 + 多通道存在 |

### 反馈循环

层不是单向栈。有一个连续的反馈循环，高层通知低层：

1. L5 代理行为生成新经验
2. L3 固化从那些经验中提取技能
3. L2 分类由 L3 中发现的模式细化
4. L1 记忆由成功操作中的新事实丰富
5. 丰富的 L1 反馈到 L2，再次开始循环

这个反馈循环是使 Aleph 真正自适应的原因。系统不仅执行 —— 它还改进。

## AGI 地平线

在 L5 之外，模型承认一个渐近极限：**ℵ_omega**，AGI 地平线。这代表一个 AI 系统在所有领域和上下文中实现完全通用智能 —— 无限适应的理论终点。

Aleph 不声称达到这个地平线。但通过构建涌现的基础设施 —— 固化管道、技能进化系统、多态架构 —— 它创建了一个基础，可以随着 AI 能力的发展攀登阿列夫数的阶梯。

```
ℵ₀  -->  原始知识 (L1-L2)
ℵ₁  -->  原子技能 (L3)
ℵ₂  -->  功能模块 (L4)
ℵ₃  -->  多态代理 (L5)
ℵ_omega  -->  AGI 地平线 (极限)
```

每一层超越前一层，但所有层都包含在同一个点 —— Aleph 中。

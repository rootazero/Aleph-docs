网关支持基于设备的 HMAC-SHA256 签名令牌认证。当启用 `require_auth` 时，每个客户端在发起 RPC 调用前必须先进行认证。

## 认证流程

```
客户端通过 WebSocket 连接
    |
    v
+-----------------------------+
| require_auth 是否启用？      |
|   是 -> 第一帧必须是           |
|          "connect"           |
|   否 -> 直接访问              |
+-----------------------------+
    |
    v（如果需要认证）
+-----------------------------+
| 验证凭据                     |
|   - Bearer token + HMAC 签名 |
|   - 设备指纹                 |
|   - 公钥验证                 |
+-----------------------------+
    |
    v
+-----------------------------+
| 颁发会话令牌                  |
| 分配客户端角色                |
|（operator 或 node）          |
+-----------------------------+
```

当不需要认证时（仅限本地部署），客户端连接后可立即开始发起 RPC 调用。启用认证时，第一个 WebSocket 帧必须是 `connect` 请求。

## Connect 请求

`connect` 方法用于协商协议版本、识别客户端并提交凭据。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "connect",
  "params": {
    "minProtocol": 1,
    "maxProtocol": 1,
    "client": {
      "id": "macos-app",
      "version": "1.0.0",
      "platform": "macos"
    },
    "role": "operator",
    "auth": {
      "token": "bearer_token",
      "signature": "hmac_sha256_signature"
    }
  }
}
```

**响应（成功）：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "session_token": "session-uuid",
    "role": "operator",
    "scopes": ["*"],
    "expires_at": 1706400000000
  }
}
```

**响应（失败）：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "error": {
    "code": -32001,
    "message": "Authentication failed: invalid token"
  }
}
```

### Connect 参数

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `minProtocol` | number | 是 | 客户端支持的最低协议版本 |
| `maxProtocol` | number | 是 | 客户端支持的最高协议版本 |
| `client.id` | string | 是 | 客户端标识符（如 `"cli"`、`"macos-app"`、`"web"`） |
| `client.version` | string | 是 | 客户端版本字符串 |
| `client.platform` | string | 是 | 平台（`"macos"`、`"ios"`、`"android"`、`"cli"`、`"web"`） |
| `role` | string | 是 | 请求的角色：`"operator"` 或 `"node"` |
| `auth.token` | string | 是 | 设备配对期间颁发的 Bearer 令牌 |
| `auth.signature` | string | 是 | 令牌的 HMAC-SHA256 签名 |

## 令牌类型

Aleph 使用两类令牌，分别用于认证生命周期的不同阶段。

### 设备令牌

设备配对期间颁发的长期令牌。安全存储在客户端设备上（macOS/iOS 使用钥匙串，其他平台使用加密存储）。用于通过 `connect` 方法获取会话令牌。

| 属性 | 值 |
|----------|-------|
| 有效期 | 直到被撤销 |
| 存储位置 | 客户端安全存储 |
| 范围 | 设备级身份 |
| 轮换 | 通过 `auth.rotate` 手动轮换 |

### 会话令牌

网关成功完成 `connect` 握手后颁发的短期令牌。在 WebSocket 会话期间自动包含在后续请求中。默认 24 小时后过期。

| 属性 | 值 |
|----------|-------|
| 有效期 | 24 小时（默认） |
| 存储位置 | 内存（网关） |
| 范围 | 每会话 |
| 轮换 | 重新连接时自动轮换 |

## HMAC-SHA256 签名

所有令牌均使用 HMAC-SHA256 签名。网关从不存储原始令牌值——仅在 SQLite 中持久化其 HMAC 哈希值。

```
令牌颁发：
  token        = UUID v4
  signature    = HMAC-SHA256(secret, token)
  stored_hash  = HMAC-SHA256(secret, token)

令牌验证：
  1. 验证 HMAC 签名与令牌匹配
  2. 计算 hash = HMAC-SHA256(secret, token)
  3. 在数据库中查找 hash
  4. 检查过期和撤销状态
  5. 更新 last_used_at 时间戳
```

HMAC 密钥在服务器启动时生成（32 字节加密安全随机数据）。可持久化以在重启后保持令牌连续性。

## 令牌验证结果

令牌通过验证后，网关返回内部 `TokenValidation`，包含会话的身份上下文。

```json
{
  "token_id": "tok-uuid",
  "device_id": "dev-uuid",
  "role": "operator",
  "scopes": ["*"],
  "remaining_ms": 86400000
}
```

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `token_id` | string | 唯一令牌标识符 |
| `device_id` | string | 拥有此令牌的设备 |
| `role` | string | `"operator"`（完全控制）或 `"node"`（受限） |
| `scopes` | string[] | 权限范围（`["*"]` = 全部） |
| `remaining_ms` | number | 令牌过期前的毫秒数 |

## 设备角色

| 角色 | 说明 | 权限 |
|------|-------------|-------------|
| `operator` | 完全控制（CLI、macOS App、Web UI） | 所有 RPC 方法、工具执行、配置更改 |
| `node` | 受限执行（iOS/Android 节点） | 代理交互、只读配置、无执行审批 |

## 设备配对

新设备在认证前必须先进行配对。配对流程使用质询-响应协议。

### 配对流程

```
新设备                          网关
    |                              |
    |--- auth.pairing.request ---->|
    |    { device_name, platform,  |
    |      public_key }            |
    |                              |
    |<-- pairing_id, challenge ----|
    |                              |
    |                  所有者通过 CLI/UI 审批
    |                              |
    |--- auth.pairing.complete --->|
    |    { pairing_id, signed_challenge}|
    |                              |
    |<-- device_token, signature --|
```

### 审批/拒绝配对

所有者通过 `auth.pairing.approve` 或 `auth.pairing.reject` 审批或拒绝待处理的配对请求。

**审批：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "auth.pairing.approve",
  "params": {
    "pairing_id": "pair-uuid",
    "role": "operator",
    "scopes": ["*"]
  }
}
```

**拒绝：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "auth.pairing.reject",
  "params": {
    "pairing_id": "pair-uuid",
    "reason": "Unknown device"
  }
}
```

### 设备列表

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "auth.devices.list"
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "devices": [
      {
        "device_id": "dev-uuid-1",
        "device_name": "MacBook Pro",
        "device_type": "macos",
        "role": "operator",
        "scopes": ["*"],
        "last_seen_at": 1706400000000,
        "created_at": 1706313600000
      }
    ]
  }
}
```

## 令牌轮换

令牌可在不重新配对的情况下进行轮换。旧令牌被撤销，同时颁发一个具有相同权限的新令牌。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "auth.rotate",
  "params": {
    "token": "old-token",
    "signature": "old-signature"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "token": "new-token",
    "signature": "new-signature",
    "token_id": "new-tok-uuid",
    "expires_at": 1706486400000
  }
}
```

## 令牌撤销

撤销特定令牌或设备的所有令牌。

**撤销单个令牌：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "auth.revoke",
  "params": {
    "token_id": "tok-uuid"
  }
}
```

**撤销设备的所有令牌：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "auth.revoke_device",
  "params": {
    "device_id": "dev-uuid"
  }
}
```

## 访客访问

Aleph 支持基于邀请的访客访问，具有限定权限。访客获得对特定工具的有限访问权限。

### 访客邀请流程

1. **所有者创建邀请**，附带限定权限和可选的过期时间
2. **访客使用一次性令牌激活邀请**
3. **网关创建身份上下文**，权限被冻结
4. **访客在范围内操作**——策略引擎强制执行边界

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "auth.invite",
  "params": {
    "guest_name": "Mom",
    "allowed_tools": ["translate"],
    "expires_in_seconds": 86400,
    "display_name": "Mom"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "invitation_token": "inv-token-uuid",
    "url": "https://aleph.local/join?t=inv-token-uuid",
    "expires_at": 1706486400000
  }
}
```

### 权限匹配

访客权限使用分层匹配系统：

| 模式 | 匹配项 |
|---------|---------|
| `"translate"` | 精确工具名 `translate` |
| `"shell"` | 类别匹配：`shell:exec`、`shell:read` 等 |
| `"*"` | 通配符：所有工具 |

## 安全请求头

使用 HTTP 回退端点时，在请求头中包含认证信息：

| 请求头 | 值 | 说明 |
|--------|-------|-------------|
| `Authorization` | `Bearer <token>` | 设备或会话令牌 |
| `X-Aleph-Signature` | `<hmac_signature>` | 令牌的 HMAC-SHA256 |
| `X-Aleph-Device-Id` | `<device_id>` | 设备标识符 |

## 错误代码

| 代码 | 常量 | 说明 |
|------|----------|-------------|
| -32000 | `AUTH_REQUIRED` | 未提供凭据 |
| -32001 | `AUTH_FAILED` | 令牌或签名无效 |
| -32002 | `PERMISSION_DENIED` | 认证有效但权限不足 |

## 配置

```json5
{
  "gateway": {
    "require_auth": true,
    "token_expiry_ms": 86400000,  // 24 小时
    "max_devices": 10,
    "auto_cleanup_expired": true
  }
}
```

事件方法管理 WebSocket 连接上的实时事件订阅。使用 glob 模式订阅主题，以接收有关代理活动、流式输出、配置更改等的通知。

## 事件传递

事件作为 WebSocket 连接上的 JSON-RPC 通知（无 `id` 字段）传递：

```json
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.chunk",
    "data": {
      "run_id": "run-uuid-123",
      "content": "Here is the response..."
    },
    "timestamp": 1706400000000
  }
}
```

事件是即发即弃的：服务器发送它们而不等待确认。如果客户端断开连接，断开连接期间的事件将丢失。

## 方法

### events.subscribe

使用 glob 模式订阅一个或多个事件主题。订阅是累加的——多次调用 subscribe 会添加新模式而不会删除现有模式。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "events.subscribe",
  "params": {
    "patterns": ["stream.*", "agent.*"]
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "subscribed": ["stream.*", "agent.*"],
    "active_patterns": ["stream.*", "agent.*"]
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `patterns` | string[] | 是 | 要订阅的 glob 模式数组 |

### events.unsubscribe

删除一个或多个事件订阅。模式必须与订阅的内容完全匹配。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "events.unsubscribe",
  "params": {
    "patterns": ["stream.*"]
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "unsubscribed": ["stream.*"],
    "active_patterns": ["agent.*"]
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `patterns` | string[] | 是 | 要取消订阅的模式数组 |

### events.list

列出当前 WebSocket 连接的所有活动事件订阅。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "events.list"
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "patterns": ["agent.*"],
    "total": 1
  }
}
```

此方法不接受参数。

## 事件主题

### 流式事件（`stream.*`）

在代理响应生成期间发出。

| 主题 | 说明 | 数据字段 |
|-------|-------------|-------------|
| `stream.start` | 响应流式传输开始 | `run_id` |
| `stream.chunk` | 代理的文本块 | `run_id`, `content` |
| `stream.tool_start` | 代理正在调用工具 | `run_id`, `tool_name`, `args` |
| `stream.tool_end` | 工具执行完成 | `run_id`, `tool_name`, `result`, `duration_ms` |
| `stream.thinking` | 推理步骤（CoT 透明性） | `run_id`, `step_type`, `content` |
| `stream.end` | 响应流式传输完成 | `run_id` |

**示例——stream.chunk：**
```json
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.chunk",
    "data": {
      "run_id": "run-uuid-123",
      "content": "Based on my analysis, "
    }
  }
}
```

**示例——stream.tool_start：**
```json
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.tool_start",
    "data": {
      "run_id": "run-uuid-123",
      "tool_name": "shell:exec",
      "args": { "command": "git log --oneline -5" }
    }
  }
}
```

### 代理事件（`agent.*`）

在代理生命周期更改时发出。

| 主题 | 说明 | 数据字段 |
|-------|-------------|-------------|
| `agent.started` | 代理运行已启动 | `run_id`, `session_key` |
| `agent.completed` | 代理运行完成 | `run_id`, `session_key`, `tokens_used` |
| `agent.error` | 代理运行遇到错误 | `run_id`, `error` |
| `agent.cancelled` | 代理运行被取消 | `run_id` |

**示例——agent.completed：**
```json
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "agent.completed",
    "data": {
      "run_id": "run-uuid-123",
      "session_key": "agent:main:main",
      "tokens_used": 1247,
      "duration_ms": 5000
    }
  }
}
```

### 会话事件（`session.*`）

在会话状态更改时发出。

| 主题 | 说明 | 数据字段 |
|-------|-------------|-------------|
| `session.created` | 新会话已创建 | `session_key` |
| `session.deleted` | 会话已删除 | `session_key` |
| `session.compacted` | 会话已压缩 | `session_key`, `tokens_before`, `tokens_after` |

### 配置事件（`config.*`）

在配置更改时发出。

| 主题 | 说明 | 数据字段 |
|-------|-------------|-------------|
| `config.changed` | 配置已修改 | `changed_keys`, `source` |
| `config.reloaded` | 配置已从文件重新加载 | `source` |

### Cron 事件（`cron.*`）

在计划任务执行期间发出。

| 主题 | 说明 | 数据字段 |
|-------|-------------|-------------|
| `cron.triggered` | 任务已触发 | `job_id`, `run_id`, `trigger_source` |
| `cron.completed` | 任务运行完成 | `job_id`, `run_id`, `duration_ms` |
| `cron.failed` | 任务运行失败 | `job_id`, `run_id`, `error` |

### 执行事件（`exec.*`）

在命令执行审批期间发出。

| 主题 | 说明 | 数据字段 |
|-------|-------------|-------------|
| `exec.approval_requested` | 命令需要审批 | `id`, `command`, `risk_level` |
| `exec.approval_resolved` | 审批决策已做出 | `id`, `decision` |

## Glob 模式匹配

事件订阅模式使用 glob 风格匹配：

| 模式 | 匹配项 |
|---------|---------|
| `stream.*` | 所有流式事件：`stream.chunk`、`stream.start` 等 |
| `agent.*` | 所有代理生命周期事件 |
| `*` | 所有事件 |
| `stream.chunk` | 仅精确匹配 |
| `*.completed` | 任何 `completed` 事件：`agent.completed`、`cron.completed` 等 |

模式与完整主题字符串匹配。`*` 通配符匹配任何非点字符序列。单独使用 `*` 可订阅所有内容。

## 典型客户端流程

典型客户端在启动代理交互之前订阅事件：

```json
// 1. 订阅相关事件
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "events.subscribe",
  "params": {
    "patterns": ["stream.*", "agent.*", "exec.*"]
  }
}

// 2. 启动代理运行
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "agent.run",
  "params": {
    "message": "Check disk usage and alert if over 80%"
  }
}

// 3. 接收流式事件
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "agent.started", "data": { "run_id": "run-123" } } }
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "stream.start", "data": { "run_id": "run-123" } } }
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "stream.tool_start", "data": { "run_id": "run-123", "tool_name": "shell:exec" } } }
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "stream.tool_end", "data": { "run_id": "run-123", "tool_name": "shell:exec" } } }
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "stream.chunk", "data": { "run_id": "run-123", "content": "Disk usage is at 45%..." } } }
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "stream.end", "data": { "run_id": "run-123" } } }
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "agent.completed", "data": { "run_id": "run-123" } } }
```

Cron 方法管理按计划执行代理提示的计划任务。任务持久化在 SQLite 中，支持标准 cron 表达式，可以暂停、恢复或手动触发。

## Cron 表达式格式

```
+-------------- 分钟 (0 - 59)
| +------------ 小时 (0 - 23)
| | +---------- 日期 (1 - 31)
| | | +-------- 月份 (1 - 12)
| | | | +------ 星期 (0 - 6, 周日 = 0)
| | | | |
* * * * *
```

**示例：**

| 表达式 | 计划 |
|------------|----------|
| `0 9 * * *` | 每天早上 9:00 |
| `*/15 * * * *` | 每 15 分钟 |
| `0 0 * * 0` | 每周日午夜 |
| `0 9 1 * *` | 每月第一天早上 9:00 |
| `30 18 * * 1-5` | 工作日下午 6:30 |

## 方法

### cron.create

创建新的计划任务。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "cron.create",
  "params": {
    "name": "Daily Summary",
    "schedule": "0 18 * * *",
    "agent_id": "main",
    "prompt": "Summarize today's activities and upcoming tasks",
    "enabled": true,
    "timezone": "America/New_York",
    "tags": ["productivity", "daily"]
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "job_id": "job-uuid-123",
    "name": "Daily Summary",
    "schedule": "0 18 * * *",
    "agent_id": "main",
    "enabled": true,
    "next_run_at": 1706468400000,
    "created_at": 1706400000000
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `name` | string | 是 | 人类可读的任务名称 |
| `schedule` | string | 是 | Cron 表达式（5 字段标准格式） |
| `agent_id` | string | 是 | 执行提示的代理 |
| `prompt` | string | 是 | 发送给代理的消息 |
| `enabled` | boolean | 否 | 任务是否处于活动状态。默认：`true` |
| `timezone` | string | 否 | 用于计划评估的 IANA 时区 |
| `tags` | string[] | 否 | 用于过滤和组织的标签 |
| `max_retries` | number | 否 | 失败时的最大重试次数。默认：3 |
| `priority` | number | 否 | 任务优先级 (1-10)。默认：5 |
| `prompt_template` | string | 否 | 带有 `{{variables}}` 的动态提示模板 |
| `context_vars` | object | 否 | 模板替换变量 |
| `delivery` | object | 否 | 交付目标配置（见下文） |

### cron.list

列出所有配置的 cron 任务。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "cron.list",
  "params": {
    "filter": {
      "enabled": true,
      "agent_id": "main"
    }
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "jobs": [
      {
        "job_id": "job-uuid-123",
        "name": "Daily Summary",
        "schedule": "0 18 * * *",
        "agent_id": "main",
        "enabled": true,
        "next_run_at": 1706468400000,
        "last_run_at": 1706382000000,
        "consecutive_failures": 0,
        "tags": ["productivity", "daily"]
      },
      {
        "job_id": "job-uuid-456",
        "name": "Weekly Backup Check",
        "schedule": "0 0 * * 0",
        "agent_id": "main",
        "enabled": true,
        "next_run_at": 1706572800000,
        "last_run_at": null,
        "consecutive_failures": 0,
        "tags": ["maintenance"]
      }
    ],
    "total": 2
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `filter` | object | 否 | 过滤条件 |
| `filter.enabled` | boolean | 否 | 按启用/禁用状态过滤 |
| `filter.agent_id` | string | 否 | 按代理过滤 |
| `filter.tags` | string[] | 否 | 按标签过滤（匹配任意） |

### cron.get

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "cron.get",
  "params": {
    "job_id": "job-uuid-123"
  }
}
```

**响应：**
```json

  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "job_id": "job-uuid-123",
    "name": "Daily Summary",
    "schedule": "0 18 * * *",
    "schedule_kind": "cron",
    "agent_id": "main",
    "prompt": "Summarize today's activities and upcoming tasks",
    "enabled": true,
    "timezone": "America/New_York",
    "tags": ["productivity", "daily"],
    "priority": 5,
    "max_retries": 3,
    "consecutive_failures": 0,
    "next_run_at": 1706468400000,
    "last_run_at": 1706382000000,
    "created_at": 1706313600000,
    "updated_at": 1706400000000,
    "recent_runs": [
      {
        "run_id": "run-uuid-001",
        "status": "completed",
        "started_at": 1706382000000,
        "ended_at": 1706382015000,
        "duration_ms": 15000,
        "trigger_source": "schedule"
      },
      {
        "run_id": "run-uuid-002",
        "status": "completed",
        "started_at": 1706295600000,
        "ended_at": 1706295612000,
        "duration_ms": 12000,
        "trigger_source": "schedule"
      }
    ]
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `job_id` | string | 是 | 要检索的任务 ID |

### cron.delete

删除计划任务。任务的运行历史也会被删除。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "cron.delete",
  "params": {
    "job_id": "job-uuid-123"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "result": {
    "deleted": true,
    "job_id": "job-uuid-123"
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `job_id` | string | 是 | 要删除的任务 |

### cron.status

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "cron.status",
  "params": {
    "job_id": "job-uuid-123"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "result": {
    "job_id": "job-uuid-123",
    "enabled": true,
    "last_run_at": 1706295612000,
    "next_run_at": 1706572800000,
    "consecutive_failures": 0,
    "status": "healthy"
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `job_id` | string | 是 | 要查询的任务 ID |

### cron.trigger

手动立即触发任务的执行，绕过计划。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "method": "cron.trigger",
  "params": {
    "job_id": "job-uuid-123"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "result": {
    "triggered": true,
    "job_id": "job-uuid-123",
    "run_id": "run-uuid-003",
    "status": "queued",
    "trigger_source": "manual"
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `job_id` | string | 是 | 要触发的任务 |

### cron.pause

暂停正在运行的任务。任务保持配置状态，但在下次计划时间不会执行。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 6,
  "method": "cron.pause",
  "params": {
    "job_id": "job-uuid-123"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 6,
  "result": {
    "paused": true,
    "job_id": "job-uuid-123",
    "enabled": false
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `job_id` | string | 是 | 要暂停的任务 |

### cron.resume

恢复暂停的任务。下次运行将从当前时间计算。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 7,
  "method": "cron.resume",
  "params": {
    "job_id": "job-uuid-123"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 7,
  "result": {
    "resumed": true,
    "job_id": "job-uuid-123",
    "enabled": true,
    "next_run_at": 1706468400000
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `job_id` | string | 是 | 要恢复的任务 |

## 计划类型

除了标准 cron 表达式外，Aleph 还支持其他计划种类：

| 种类 | 说明 | 示例 |
|------|-------------|---------|
| `cron` | 标准 cron 表达式 | `"0 9 * * *"` |
| `every` | 固定间隔（毫秒） | `900000` (15 分钟) |
| `once` | 在特定时间戳一次性执行 | `1706468400000` |

对于 `every` 和 `once` 计划，使用 `schedule_kind` 参数：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "cron.create",
  "params": {
    "name": "Heartbeat Check",
    "schedule_kind": "every",
    "every_ms": 900000,
    "agent_id": "main",
    "prompt": "Check system health"
  }
}
```

## 任务链

任务可以被链接，以便一个任务完成后触发另一个任务：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "cron.create",
  "params": {
    "name": "Backup Database",
    "schedule": "0 2 * * *",
    "agent_id": "main",
    "prompt": "Run database backup",
    "next_job_id_on_success": "job-uuid-verify",
    "next_job_id_on_failure": "job-uuid-alert"
  }
}
```

## 运行状态值

| 状态 | 说明 |
|--------|-------------|
| `queued` | 任务已排队等待执行 |
| `running` | 正在执行 |
| `completed` | 成功完成 |
| `failed` | 执行失败 |
| `retrying` | 失败，正在尝试重试 |

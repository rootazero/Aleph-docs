---

## memory_search

搜索知识库以获取相关事实。

```
<builtin_tool>memory_search</builtin_tool>
<parameter name="query">What database did we decide on?</parameter>
<parameter name="limit">5</parameter>
```

**参数：**
- `query`（必填）— 搜索查询
- `limit`（可选）— 最大结果数，默认 5
- `agent_id`（可选）— 目标代理命名空间
- `mode`（可选）— `context` | `tools` | `hybrid`，默认 `hybrid`

**返回：** 带分数的 `MemoryFact` 排名列表。

---

## memory_reflect

**规范 2。** 从记忆中综合出连贯的答案，而不是返回原始事实。

```
<builtin_tool>memory_reflect</builtin_tool>
<parameter name="query">Summarise what we know about the user's project requirements</parameter>
<parameter name="depth">comprehensive</parameter>
```

**参数：**
- `query`（必填）— 要回答的问题
- `depth`（可选）— `brief` | `standard` | `comprehensive`，默认 `standard`
- `agent_id`（可选）— 目标代理命名空间

**返回：** 带引用来源的综合答案。

---

## session_search

**规范 B。** 跨会话记录搜索。

```
<builtin_tool>session_search</builtin_tool>
<parameter name="query">When did we discuss authentication?</parameter>
<parameter name="limit">5</parameter>
```

**参数：**
- `query`（必填）— 搜索查询
- `limit`（可选）— 返回的最大会话数，默认 5
- `summarise`（可选）— `true` | `false`，默认 `true`

**返回：** 匹配的会话 + AI 生成的摘要（如果 `summarise=true`）。

---

## remember

**规范 A。** 将事实固定到精选热记忆中。

```
<builtin_tool>remember</builtin_tool>
<parameter name="content">User prefers dark mode in all applications</parameter>
<parameter name="priority">8</parameter>
```

**参数：**
- `content`（必填）— 要记住的事实
- `priority`（可选）— 1-10，越高 = 越重要，默认 5
- `category`（可选）— 如 `preference`、`fact`、`warning`

**返回：** 带事实 ID 的确认。

**注意：** 精选记忆是一个**冻结快照**。更改需要显式重新策划。

---

## forget

从记忆中删除事实。

```
<builtin_tool>forget</builtin_tool>
<parameter name="query">The user's old email address</parameter>
```

**参数：**
- `query`（必填）— 要删除的事实的描述
- `confirm`（可选）— `true` | `false`，默认 `false`

**返回：** 候选事实列表 + 确认提示（如果 `confirm=false`）。

---

## memory_status

检查记忆系统健康和统计信息。

```
<builtin_tool>memory_status</builtin_tool>
```

**返回：**
- 总笔记数
- 原始记忆数（未处理）
- 索引健康（FTS + 向量）
- 上次 dream 运行时间戳
- 精选记忆数

## 可观测性与治理（`memory.*` / `dreaming.*`）

下列方法是 **RPC 方法**（而非内置聊天工具）。它们对记忆流水线暴露只读的
可观测性，并提供梦境守护进程控制。写入仍由 LLM/工具驱动——这些方法
呈现状态，不会变更事实。

### memory.retrieve_with_trace

运行真实的 `NoteFactRetrieval` 评分流水线，返回结果**以及**每个阶段的
遥测数据，以便调试某条事实为何浮现或未浮现。

**参数：**
- `query`（必填）— 搜索查询
- `limit`（可选）— 最大结果数
- `agent_id`（可选）— 目标代理命名空间

**返回：** 排名结果加上一个 `trace` 阶段数组，每项为
`{ stage, count, elapsed_ms, scores }`。各阶段按流水线顺序排列：
query → candidate pool → BM25 + vector scores → RRF fusion → cutoff → ranked。

### memory.list_corrections

列出纠正记录 → 蒸馏的生命周期（只读）。呈现用户纠正如何流入蒸馏记忆。

**参数：**
- `agent_id`（可选）— 目标代理命名空间
- `limit`（可选）— 最大条目数

**返回：** 纠正记录列表及其蒸馏状态。

### dreaming.list_insights

列出梦境守护进程产生的近期每日综合洞察，以及 dream 运行历史。

**参数：**
- `limit`（可选）— 返回的最大洞察/运行数

**返回：** 近期综合笔记 + dream 运行历史条目。

### dreaming.run_now

立即触发一次梦境周期，而无需等待计划运行。

**返回：** 带所触发 dream 运行标识符的确认。

> 这四个方法位于 `memory.*` 和 `dreaming.*` RPC 命名空间中，与现有的
> `memory.search` / `store` / `delete` / `clear` / `stats` / `compress`
> 方法并列——区别于上文记录的内置聊天工具。

## 方法比较

| 方法 | 使用时机 | 返回 |
|---|---|---|
| `memory_search` | 需要特定事实 | 排名事实 |
| `memory_reflect` | 需要综合/答案 | 连贯答案 + 来源 |
| `session_search` | 寻找过去的对话 | 会话 + 摘要 |
| `remember` | 关键事实不能被遗漏 | 确认 |
| `forget` | 删除过时信息 | 确认 |
| `memory_status` | 调试记忆问题 | 统计 |

---

## 错误处理

所有记忆方法都返回结构化错误：

| 错误 | 含义 |
|---|---|
| `MemoryNotFound` | 没有匹配的事实/会话 |
| `IndexCorrupted` | SQLite 索引需要重建 |
| `QuotaExceeded` | 令牌预算已耗尽 |
| `LockTimeout` | 跨进程争用（规范 C） |

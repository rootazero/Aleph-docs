运行方法管理活动代理执行的生命周期，并通过向运行中的代理排队消息来支持人工参与循环场景。

## 方法

### run.wait

等待运行完成、失败或被取消。阻塞直到运行结束或超时到期。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "run.wait",
  "params": {
    "run_id": "run-uuid-123",
    "timeout_ms": 30000
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "status": "completed",
    "output": "Task completed successfully",
    "input_tokens": 0,
    "output_tokens": 150,
    "duration_ms": 500
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `run_id` | string | 是 | 要等待的运行 ID |
| `timeout_ms` | number | 否 | 最大等待时间（毫秒）。默认：30000，最大：300000 |

**状态值：**

| 状态 | 说明 |
|--------|-------------|
| `completed` | 运行成功完成 |
| `failed` | 运行失败并出现错误 |
| `cancelled` | 运行被用户取消 |
| `timeout` | 等待超时，运行未完成 |
| `not_found` | 未找到运行 ID |

### run.queue_message

向活动运行排队消息，用于人工参与循环场景。当代理下次请求输入时，消息会被传递给它。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "run.queue_message",
  "params": {
    "run_id": "run-uuid-123",
    "message": "Yes, proceed with the file deletion"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "success": true
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `run_id` | string | 是 | 要发送消息的运行 ID |
| `message` | string | 是 | 要排队的消息内容 |

**错误情况：**

| 错误 | 说明 |
|-------|-------------|
| `Run not found` | 指定的 run_id 不存在 |
| `Input queue is full` | 运行的输入队列已达到容量上限 |
| `Run has ended` | 运行已完成或已被取消 |

## 用例

### 同步执行

使用 `run.wait` 阻塞直到代理运行完成，适用于脚本编写或需要在继续之前获取最终结果的情况：

```json
// 1. 启动运行
{ "jsonrpc": "2.0", "id": 1, "method": "agent.run", "params": { "message": "Analyze this data" } }

// 2. 等待完成（最多阻塞 30 秒）
{ "jsonrpc": "2.0", "id": 2, "method": "run.wait", "params": { "run_id": "run-uuid-123" } }
```

### 人工参与循环

使用 `run.queue_message` 响应代理审批请求：

```json
// 代理通过事件发送审批请求
{ "topic": "approval.request", "data": { "run_id": "run-uuid-123", "message": "Approve rm -rf /tmp?" } }

// 用户通过 queue_message 响应
{ "jsonrpc": "2.0", "id": 3, "method": "run.queue_message", "params": { "run_id": "run-uuid-123", "message": "yes" } }
```

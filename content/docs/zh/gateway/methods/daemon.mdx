守护进程方法控制 Aleph 后台守护进程——启动、停止、重启和查询托管网关并管理所有子系统的守护进程。

## 方法

### daemon.status

获取当前守护进程状态。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "daemon.status",
  "params": {}
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "status": "running",
    "pid": 12345,
    "version": "0.1.0",
    "started_at": "2024-01-15T10:00:00Z",
    "uptime_seconds": 1800,
    "components": {
      "gateway": "healthy",
      "memory": "healthy",
      "mcp": "healthy"
    }
  }
}
```

### daemon.start

如果守护进程未运行，则启动它。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "daemon.start",
  "params": {}
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "started": true,
    "pid": 12346
  }
}
```

### daemon.stop

优雅地停止守护进程。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "daemon.stop",
  "params": {
    "force": false
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "stopped": true
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `force` | boolean | 否 | 强制停止而不优雅关闭 |

### daemon.restart

重启守护进程。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "daemon.restart",
  "params": {
    "force": false
  }
}
```

### daemon.reload

不重启重新加载守护进程配置。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "method": "daemon.reload",
  "params": {}
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "result": {
    "reloaded": true,
    "config_valid": true
  }
}
```

## CLI 等效命令

这些 RPC 方法镜像 CLI 命令：

```bash
aleph-server start    # 启动守护进程
aleph-server stop     # 停止守护进程
aleph-server restart  # 重启守护进程
aleph-server status   # 检查状态
```

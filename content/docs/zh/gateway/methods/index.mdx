Aleph 网关通过命名空间的 JSON-RPC 2.0 方法公开其功能。每个命名空间对相关操作进行分组，每个方法都遵循相同的请求/响应约定。

## 请求格式

所有 RPC 请求都使用 JSON-RPC 2.0 信封：

```json
{
  "jsonrpc": "2.0",
  "id": "unique-id",
  "method": "namespace.method",
  "params": { ... }
}
```

- `id` 可以是字符串、数字或 null（null 用于通知）。
- `method` 遵循 `namespace.action` 命名约定。
- `params` 是每个方法的可选 JSON 对象。

## 响应格式

**成功：**
```json
{
  "jsonrpc": "2.0",
  "id": "unique-id",
  "result": { ... }
}
```

**错误：**
```json
{
  "jsonrpc": "2.0",
  "id": "unique-id",
  "error": {
    "code": -32601,
    "message": "Method not found",
    "data": null
  }
}
```

## 错误代码

| 代码 | 名称 | 说明 |
|------|------|-------------|
| -32700 | Parse error | 格式错误的 JSON |
| -32600 | Invalid Request | 缺少必填字段 |
| -32601 | Method not found | 未知方法名 |
| -32602 | Invalid params | 错误或缺少参数 |
| -32603 | Internal error | 服务器端故障 |
| -32000 | Auth required | 未提供凭据 |
| -32001 | Auth failed | 令牌或签名无效 |
| -32002 | Permission denied | 权限不足 |
| -32003 | Rate limited | 请求过多 |
| -32004 | Resource not found | 请求的实体不存在 |
| -32005 | Timeout | 操作超时 |
| -32006 | Tool error | 工具执行失败 |

## 常见模式

### 分页

返回列表的方法接受可选的 `limit` 和 `offset` 参数：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "session.list",
  "params": {
    "limit": 20,
    "offset": 0
  }
}
```

### 过滤

列表方法支持可选的 `filter` 参数用于缩小结果范围：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "cron.list",
  "params": {
    "filter": {
      "enabled": true,
      "agent_id": "main"
    }
  }
}
```

### 流式方法

像 `agent.run` 这样的方法触发流式事件。订阅 `stream.*` 主题以接收实时数据：

```json
// 1. 订阅流式事件
{ "jsonrpc": "2.0", "id": 1, "method": "events.subscribe", "params": { "patterns": ["stream.*"] } }

// 2. 启动代理运行
{ "jsonrpc": "2.0", "id": 2, "method": "agent.run", "params": { "message": "Hello" } }

// 3. 接收流式事件（无 id = 通知）
{ "jsonrpc": "2.0", "method": "event", "params": { "topic": "stream.chunk", "data": { "text": "Hi!" } } }
```

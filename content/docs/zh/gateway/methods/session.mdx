会话方法管理对话会话——存储消息历史、令牌计数和身份元数据的持久化上下文。会话由结构化会话键标识，并由 SQLite 提供支持。

:::warning
本页记录的是预期 API。实际运行时注册的方法有所不同：

**实际方法（复数 `sessions.*`）：**
`sessions.list`、`sessions.history`、`sessions.reset`、`sessions.delete`、`sessions.new`、`sessions.set_topic`、`sessions.patch`、`sessions.preview`、`sessions.compaction.list`、`sessions.compaction.restore`、`sessions.compaction.branch`

**实际方法（单数 `session.*`）：**
`session.create`、`session.usage`、`session.compact`

以下记录的方法（`session.get`、`session.list`、`session.history`、`session.compact`、`session.delete`、`session.export`）反映的是设计目标，但可能并非都在当前的运行时中接线。请参阅[方法参考](/docs/zh/gateway/methods)获取准确的方法列表。
:::

## 方法

### session.get

获取特定会话的详细信息。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "session.get",
  "params": {
    "session_key": "agent:main:main"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "session_key": "agent:main:main",
    "agent_id": "main",
    "channel": "gateway",
    "message_count": 42,
    "token_count": 12840,
    "created_at": 1706313600000,
    "updated_at": 1706400000000,
    "last_compaction": 1706356800000,
    "identity": {
      "role": "owner",
      "identity_id": "owner",
      "source_channel": "gateway"
    }
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `session_key` | string | 是 | 要查找的会话键 |

### session.list

列出所有会话，可选按代理、频道或状态过滤。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "session.list",
  "params": {
    "filter": {
      "agent_id": "main",
      "channel": "telegram"
    },
    "limit": 20,
    "offset": 0
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "sessions": [
      {
        "session_key": "agent:main:telegram:dm:user123",
        "agent_id": "main",
        "channel": "telegram",
        "message_count": 15,
        "token_count": 4200,
        "created_at": 1706313600000,
        "updated_at": 1706399000000
      },
      {
        "session_key": "agent:main:telegram:dm:user456",
        "agent_id": "main",
        "channel": "telegram",
        "message_count": 8,
        "token_count": 2100,
        "created_at": 1706350000000,
        "updated_at": 1706398000000
      }
    ],
    "total": 2
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `filter` | object | 否 | 过滤条件 |
| `filter.agent_id` | string | 否 | 按代理 ID 过滤 |
| `filter.channel` | string | 否 | 按频道过滤（如 `"telegram"`、`"discord"`、`"gateway"`） |
| `limit` | number | 否 | 最大结果数。默认：50 |
| `offset` | number | 否 | 跳过这么多结果。默认：0 |

### session.history

获取会话的完整消息历史。这与 `agent.history` 类似，但返回包含元数据的原始会话数据。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "session.history",
  "params": {
    "session_key": "agent:main:main",
    "limit": 100
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "session_key": "agent:main:main",
    "messages": [
      {
        "role": "system",
        "content": "[Compacted summary from 2024-01-27]...",
        "timestamp": 1706356800000
      },
      {
        "role": "user",
        "content": "What did we discuss yesterday?",
        "timestamp": 1706400000000
      },
      {
        "role": "assistant",
        "content": "Based on our conversation history...",
        "timestamp": 1706400003000,
        "tool_calls": []
      }
    ],
    "total": 3,
    "token_count": 890
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `session_key` | string | 是 | 要检索的会话 |
| `limit` | number | 否 | 最大消息数。默认：100 |

### session.compact

触发会话的手动压缩。压缩从旧消息中提取关键事实，将它们存储在记忆系统中，并将原始消息替换为摘要。这减少了令牌计数，同时保留了重要上下文。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "session.compact",
  "params": {
    "session_key": "agent:main:main"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "result": {
    "session_key": "agent:main:main",
    "compacted": true,
    "messages_before": 120,
    "messages_after": 15,
    "tokens_before": 48000,
    "tokens_after": 6200,
    "facts_extracted": 8
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `session_key` | string | 是 | 要压缩的会话 |

### session.delete

永久删除会话及其所有消息。此操作无法撤销。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "method": "session.delete",
  "params": {
    "session_key": "agent:main:telegram:dm:user123"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "result": {
    "deleted": true,
    "session_key": "agent:main:telegram:dm:user123",
    "messages_removed": 15
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `session_key` | string | 是 | 要删除的会话 |

### session.export

将会话导出为可移植的 JSON 对象。可用于备份、迁移或共享对话日志。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 6,
  "method": "session.export",
  "params": {
    "session_key": "agent:main:main",
    "format": "json"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 6,
  "result": {
    "session_key": "agent:main:main",
    "format": "json",
    "data": {
      "session_key": "agent:main:main",
      "agent_id": "main",
      "messages": [ ... ],
      "metadata": {
        "created_at": 1706313600000,
        "message_count": 42,
        "token_count": 12840
      },
      "exported_at": 1706400000000
    }
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `session_key` | string | 是 | 要导出的会话 |
| `format` | string | 否 | 导出格式：`"json"`（默认）或 `"markdown"` |

## 会话键结构

会话由编码路由信息的结构化键标识。

| 变体 | 格式 | 说明 |
|---------|--------|-------------|
| Main | `agent:{agent_id}:main` | 共享跨频道会话 |
| DirectMessage | `agent:{agent_id}:{channel}:dm:{user_id}` | 每个用户隔离的会话 |
| Group | `agent:{agent_id}:{channel}:group:{group_id}` | 群组/频道聊天 |
| Task | `agent:{agent_id}:cron:{job_name}` | 自动化任务会话 |
| Subagent | `subagent:agent:{agent_id}:{subagent_id}` | 子代理委托 |
| Ephemeral | `agent:{agent_id}:ephemeral:{uuid}` | 临时的，不持久化 |

## DM 范围策略

`dmScope` 配置控制私信如何路由到会话：

| 策略 | 行为 |
|----------|----------|
| `main` | 所有私信共享主会话 |
| `per_peer` | 每个用户获得一个隔离的会话（默认） |
| `per_channel_peer` | 按频道 + 用户组合隔离 |

## 存储模式

会话以以下结构存储在 SQLite 中：

```sql
CREATE TABLE sessions (
    session_key  TEXT PRIMARY KEY,
    messages     TEXT,          -- JSON 数组
    created_at   INTEGER,
    updated_at   INTEGER,
    message_count INTEGER,
    token_count  INTEGER
);

CREATE TABLE session_metadata (
    session_key  TEXT PRIMARY KEY,
    agent_id     TEXT,
    channel      TEXT,
    last_compaction INTEGER
);
```

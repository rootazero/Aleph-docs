浏览器方法通过 Chrome DevTools 协议 (CDP) 提供浏览器自动化。导航网页、与元素交互、捕获截图、运行 JavaScript 和提取无障碍快照——全部通过 RPC 控制。

:::warning
本页记录的是预期 API。下面显示的 `browser.*` 方法在源处理程序（`src/gateway/handlers/browser_config.rs`）中定义，但在运行时中**当前未作为独立的 JSON-RPC 方法公开**。浏览器自动化在代理执行中作为内置工具（`browser_navigate`、`browser_click` 等）可用。`browser_config.*` 命名空间（`browser_config.get`、`browser_config.update`）已注册用于配置。请参阅[方法参考](/docs/zh/gateway/methods)获取准确的方法列表。
:::

## 先决条件

浏览器自动化需要基于 Chromium 的浏览器。Aleph 可以自动检测已安装的浏览器，或通过 WebSocket 连接到正在运行的实例。

```toml
# ~/.aleph/config.toml
[browser]
mode = "auto"        # "auto", "connect", or "binary"
headless = false
cdp_port = 9222
```

## 方法

### browser.navigate

导航到当前标签页中的 URL。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "browser.navigate",
  "params": {
    "url": "https://example.com"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "url": "https://example.com",
    "title": "Example Domain",
    "status": 200,
    "loaded": true
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `url` | string | 是 | 要导航到的 URL |
| `wait_until` | string | 否 | 等待条件：`"load"`（默认）、`"domcontentloaded"`、`"networkidle"` |
| `timeout_ms` | number | 否 | 导航超时（毫秒）。默认：30000 |

### browser.click

点击页面上的元素。可以通过 ARIA 快照引用 ID、CSS 选择器或视口坐标来定位元素。

**请求（通过 CSS 选择器）：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "browser.click",
  "params": {
    "target": {
      "type": "selector",
      "css": "#login-button"
    }
  }
}
```

**请求（通过 ARIA 引用）：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "browser.click",
  "params": {
    "target": {
      "type": "ref",
      "ref_id": "button[0]"
    }
  }
}
```

**请求（通过坐标）：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "browser.click",
  "params": {
    "target": {
      "type": "coordinates",
      "x": 150.0,
      "y": 300.0
    }
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "clicked": true
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `target` | ActionTarget | 是 | 元素定位（见下方的操作目标） |

### browser.type

在输入元素中键入文本。首先点击元素以确保焦点，然后将文本追加到当前值。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "browser.type",
  "params": {
    "target": {
      "type": "selector",
      "css": "input[name='email']"
    },
    "text": "user@example.com"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "typed": true
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `target` | ActionTarget | 是 | 要键入的元素 |
| `text` | string | 是 | 要键入的文本 |
| `clear` | boolean | 否 | 键入前清除现有值。默认：`false` |

### browser.screenshot

捕获当前页面或特定元素的截图。

**请求（完整页面）：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "browser.screenshot",
  "params": {
    "full_page": true,
    "format": "png"
  }
}
```

**请求（元素）：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "browser.screenshot",
  "params": {
    "target": {
      "type": "selector",
      "css": "#main-content"
    }
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "result": {
    "data_base64": "iVBORw0KGgoAAAANSUhEUg...",
    "width": 1920,
    "height": 1080,
    "format": "png"
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `target` | ActionTarget | 否 | 要捕获的元素。如果省略，则捕获视口。 |
| `full_page` | boolean | 否 | 捕获整个可滚动页面。默认：`false` |
| `format` | string | 否 | 图片格式：`"png"`（默认）或 `"jpeg"` |
| `quality` | number | 否 | JPEG 质量 (1-100)。默认：80。PNG 忽略此项。 |

### browser.evaluate

在页面上下文中执行任意 JavaScript 并返回结果。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "method": "browser.evaluate",
  "params": {
    "script": "document.querySelectorAll('a').length"
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "result": {
    "value": 42,
    "type": "number"
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `script` | string | 是 | 要评估的 JavaScript 表达式或 IIFE |
| `timeout_ms` | number | 否 | 评估超时。默认：5000 |

脚本作为表达式进行评估。对于复杂逻辑，请将其包装在 IIFE 中：

```json
{
  "jsonrpc": "2.0",
  "id": 6,
  "method": "browser.evaluate",
  "params": {
    "script": "(() => { const links = [...document.querySelectorAll('a')]; return links.map(a => ({ href: a.href, text: a.textContent.trim() })); })()"
  }
}
```

### browser.snapshot

获取页面的 ARIA 无障碍快照。返回带有角色、名称、值和边界矩形的可访问元素的结构化树。这是代理用来"查看"页面内容的主要方法。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 7,
  "method": "browser.snapshot"
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 7,
  "result": {
    "page_title": "Example Domain",
    "page_url": "https://example.com",
    "focused_ref": null,
    "elements": [
      {
        "ref_id": "heading[0]",
        "role": "heading",
        "name": "Example Domain",
        "value": null,
        "state": [],
        "bounds": { "x": 100, "y": 50, "width": 300, "height": 40 }
      },
      {
        "ref_id": "link[0]",
        "role": "link",
        "name": "More information...",
        "value": null,
        "state": [],
        "bounds": { "x": 100, "y": 200, "width": 200, "height": 20 }
      }
    ]
  }
}
```

此方法不接受参数。快照提供 `ref_id` 值，可通过 `ref` 操作目标类型与其他浏览器方法一起使用。

## 操作目标

浏览器操作方法（点击、键入等）接受 `ActionTarget` 来标识要与之交互的元素。有三种定位策略可用：

| 类型 | 说明 | 示例 |
|------|-------------|---------|
| `ref` | 通过 ARIA 快照引用 ID 定位 | `{ "type": "ref", "ref_id": "button[0]" }` |
| `selector` | 通过 CSS 选择器定位 | `{ "type": "selector", "css": "#submit-btn" }` |
| `coordinates` | 通过视口坐标定位 | `{ "type": "coordinates", "x": 150, "y": 300 }` |

### 解析顺序

所有目标在分派操作之前都会解析为视口 `(x, y)` 坐标：

1. **坐标**——直接使用
2. **引用**——获取 ARIA 快照，找到元素，返回其中心点
3. **选择器**——评估 `querySelector` + `getBoundingClientRect`，返回中心点

### 滚动方向

`browser.scroll` 操作（由代理内部使用）支持定向滚动：

| 方向 | 增量 |
|-----------|-------|
| `up` | 向上 300px |
| `down` | 向下 300px |
| `left` | 向左 300px |
| `right` | 向右 300px |

## 浏览器启动模式

| 模式 | 说明 |
|------|-------------|
| `auto` | 自动检测并启动 Chromium 浏览器 |
| `connect` | 通过 WebSocket 端点连接到现有浏览器 |
| `binary` | 通过路径启动特定的浏览器二进制文件 |

**连接模式示例：**
```json
{
  "browser": {
    "mode": {
      "type": "connect",
      "endpoint": "ws://127.0.0.1:9222/devtools/browser/abc-123"
    }
  }
}
```

## 标签页管理

浏览器方法在活动标签页上操作。对于多标签页场景，所有方法都提供 `tab_id` 参数：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "browser.navigate",
  "params": {
    "url": "https://example.com",
    "tab_id": "tab-uuid"
  }
}
```

标签页信息包括：

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `id` | string | 唯一标签页标识符 |
| `url` | string | 当前页面 URL |
| `title` | string | 当前页面标题 |

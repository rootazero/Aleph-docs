团队方法用于管理多代理团队——一个**队长**加若干**成员**，他们通过一条持久的群聊线程和一块共享任务板进行协作。在团队的第一条消息时，队长会先行规划（"团队战略协调"，Team Strata）；该规划被焊入每位成员的提示词，成员在分配的协调任务下工作（`Pending -> InProgress -> WaitingReview -> Completed`）。

> 相关列表端点：`agents.teams` 返回的团队列表会为每个团队附带一个精简的 `members_preview` 和 `last_message`（用于渲染侧边栏的"群聊"历史区）。关于实时的群聊会话原语，参见 [`group_chat.*`](/docs/zh/gateway/methods/group_chat)。

## CRUD

### teams.create

从一个队长、一组成员以及一个可选的自动命名标志创建团队。当设置了 `auto_name` 时，空名称团队会在其第一条消息时获得 LLM 自动命名。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "teams.create",
  "params": {
    "leader": "main",
    "members": ["reviewer", "tester", "documenter"],
    "auto_name": true
  }
}
```

**响应：**
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "team": {
      "id": "team-1",
      "name": "",
      "leader": "main",
      "members": ["reviewer", "tester", "documenter"]
    }
  }
}
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|-----------|------|----------|-------------|
| `leader` | string | 是 | 负责规划与审查成员工作的代理 id |
| `members` | string[] | 是 | 在分配的协调任务下工作的代理 id |
| `auto_name` | boolean | 否 | 让 LLM 在第一条消息时为空名称团队命名 |

### teams.get

获取单个团队的详情（队长、成员、名称）。

### teams.list

列出所有配置的团队。

### teams.rename

重命名团队。

### teams.disband

解散团队——拆除其工作状态，同时保留历史记录。

### teams.delete

删除团队配置。

## 群聊（Chat）

团队工作线程是一条持久的、每团队一份的群聊线程。其历史记录驱动群聊窗口中的归属气泡回放。

### teams.chat.send

向团队的群聊线程发送一条消息。第一条消息会被路由给队长进行规划，之后按成员各自的协调任务路由给成员。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "teams.chat.send",
  "params": {
    "team_id": "team-1",
    "text": "Review this pull request for security issues",
    "mention": "@all"
  }
}
```

### teams.chat.thread

获取团队持久群聊线程的元数据。

### teams.chat.history

回放某团队线程的消息历史（驱动归属气泡）。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "teams.chat.history",
  "params": {
    "team_id": "team-1"
  }
}
```

## 任务 / 看板（Tasks / Kanban）

成员在协调任务下工作（`Pending -> InProgress -> WaitingReview -> Completed`）。队长接受或拒绝交付物；当成员提交任务时，一次合成的重新派发会将该任务路由回队长进行审查。

### teams.create_task

为成员创建一个协调任务。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "teams.create_task",
  "params": {
    "team_id": "team-1",
    "assignee": "reviewer",
    "title": "Audit auth module for injection",
    "description": "Check all query construction paths."
  }
}
```

### teams.update_task

更新任务的状态、负责人或字段。

**请求：**
```json
{
  "jsonrpc": "2.0",
  "id": 5,
  "method": "teams.update_task",
  "params": {
    "team_id": "team-1",
    "task_id": "task-7",
    "status": "InProgress"
  }
}
```

| 方法 | 说明 |
|--------|-------------|
| `teams.create_task` | 为成员创建一个协调任务 |
| `teams.update_task` | 更新任务的状态/负责人/字段 |
| `teams.list_tasks` | 列出团队任务板上的任务 |
| `teams.add_task_comment` | 为任务添加评论 |
| `teams.list_task_comments` | 列出任务的评论 |
| `teams.list_task_events` | 列出任务的生命周期事件 |
| `teams.list_task_runs` | 列出任务的执行运行记录 |
| `teams.list_templates` | 列出可用的任务模板 |
| `teams.task.pause` | 暂停正在运行的任务 |
| `teams.task.resume` | 恢复已暂停的任务 |
| `teams.task.retry` | 重试失败的任务 |
| `teams.task.skip` | 跳过任务 |
| `teams.task.trace` | 获取任务的执行追踪 |
| `teams.task.journal.get` | 获取任务的日志条目 |
| `teams.task.journal.list` | 列出任务的日志条目 |

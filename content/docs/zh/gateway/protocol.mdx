title: "协议"
description: "WebSocket JSON-RPC 2.0 协议"
---

Aleph 网关使用基于 WebSocket 的 JSON-RPC 2.0 进行所有通信。

## 连接

```
ws://127.0.0.1:18790
```

当 `require_auth: true` 时，第一条消息必须是 `connect` 请求：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "connect",
  "params": {
    "minProtocol": 1,
    "maxProtocol": 1,
    "client": {
      "id": "cli",
      "version": "0.1.0",
      "platform": "macos"
    },
    "role": "operator",
    "auth": {
      "token": "bearer_token"
    }
  }
}
```

## 消息类型

### 请求（客户端 → 网关）

```json
{
  "jsonrpc": "2.0",
  "id": "unique-id",
  "method": "method.name",
  "params": { ... }
}
```

### 响应（网关 → 客户端）

成功：
```json
{
  "jsonrpc": "2.0",
  "id": "unique-id",
  "result": { ... }
}
```

错误：
```json
{
  "jsonrpc": "2.0",
  "id": "unique-id",
  "error": {
    "code": -32600,
    "message": "Invalid Request"
  }
}
```

### 事件（网关 → 客户端）

```json
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.token",
    "data": { ... }
  }
}
```

## 方法类别

| 前缀 | 说明 |
|--------|-------------|
| `agent.*` | 代理执行和控制 |
| `session.*` | 会话管理 |
| `config.*` | 配置操作 |
| `exec.*` | 执行审批 |
| `wizard.*` | 配置向导 |
| `cron.*` | 定时任务 |
| `events.*` | 事件订阅 |
| `browser.*` | CDP 浏览器控制 |

## 流式传输

对于支持流式传输的方法（如 `agent.run`），事件以单独消息发送：

```json
// 流开始
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.start",
    "data": { "run_id": "..." }
  }
}

// 令牌流
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.token",
    "data": { "text": "Hello" }
  }
}

// 流结束
{
  "jsonrpc": "2.0",
  "method": "event",
  "params": {
    "topic": "stream.end",
    "data": { "run_id": "..." }
  }
}
```

## 错误代码

| 代码 | 含义 |
|------|---------|
| -32700 | 解析错误 |
| -32600 | 无效请求 |
| -32601 | 方法未找到 |
| -32602 | 无效参数 |
| -32603 | 内部错误 |
| -32000 | 需要认证 |
| -32001 | 认证失败 |
| -32002 | 权限不足 |

## 事件订阅

订阅事件：

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "events.subscribe",
  "params": {
    "patterns": ["stream.*", "agent.*"]
  }
}
```

取消订阅：

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "events.unsubscribe",
  "params": {
    "patterns": ["stream.*"]
  }
}
```

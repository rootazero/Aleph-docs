本指南涵盖从源码编译 Aleph 所需的一切，从安装 Rust 工具链到生成优化的发布二进制文件。

## 前置条件

### Rust 工具链

Aleph 需要 **Rust 1.92 或更高版本**（edition 2021）。通过 [rustup](https://rustup.rs/) 安装：

```bash
# 安装 Rust（如果尚未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 确保 stable 工具链处于活动状态
rustup default stable

# 验证版本
rustc --version   # 必须 >= 1.92
cargo --version
```

如果你计划构建 **Control Plane UI**（嵌入式 Web 仪表板），还需要 WASM 目标：

```bash
rustup target add wasm32-unknown-unknown
cargo install wasm-bindgen-cli
```

### 系统依赖

Aleph 通过 Cargo 捆绑了大部分依赖，但需要少量系统级库：

| 依赖 | 用途 | 说明 |
|------|------|------|
| **SQLite** | 会话存储、内存数据库 | 通过 `rusqlite` 的 `bundled` 特性捆绑 —— 无需系统安装 |
| **OpenSSL** | HTTP 客户端的 TLS | macOS：由系统提供。Linux：`libssl-dev`（Debian）或 `openssl-devel`（Fedora） |
| **pkg-config** | 定位系统库 | 仅 Linux：`apt install pkg-config` 或 `dnf install pkgconf` |
| **C 编译器** | 原生依赖编译 | Xcode Command Line Tools（macOS）或 `build-essential`（Linux） |
| **Node.js + npm** | Control Plane CSS 构建（可选） | 仅在启用 `control-plane` 特性时需要 |

在 **macOS** 上，Xcode Command Line Tools 提供了一切：

```bash
xcode-select --install
```

在 **Debian/Ubuntu** 上：

```bash
sudo apt update
sudo apt install build-essential pkg-config libssl-dev
```

### 环境变量

服务器在运行时（而非构建时）需要 API 密钥：

```bash
# 在你的 shell 配置文件或 ~/.aleph/config.toml 中设置
export ANTHROPIC_API_KEY="your-api-key"
```

## 工作区结构

Aleph 使用 **Cargo 工作区**，成员按角色组织：

```
Aleph/
├── Cargo.toml                  # 工作区根 + alephcore 包
├── src/                        # alephcore —— 主库 + 服务器二进制文件
│   ├── lib.rs                  # 库 crate 入口
│   └── bin/aleph-server/       # 服务器二进制文件
├── desktop/
│   ├── shared/                 # DesktopCapability trait + IPC 协议
│   ├── macos/                  # macOS 原生实现
│   ├── linux/                  # Linux 原生实现
│   └── windows/                # Windows 原生实现
├── shared/
│   ├── protocol/               # aleph-protocol —— 共享类型
│   ├── logging/                # 日志基础设施
│   ├── ui_logic/               # 共享 UI 逻辑
│   └── client/                 # 共享客户端工具
└── interfaces/
    ├── cli/                    # 终端客户端
    ├── tui/                    # TUI 客户端
    └── webchat/                # 基于 Web 的聊天界面（WASM）
```

工作区根目录的 `Cargo.toml` 定义了共享依赖版本，使所有成员保持同步：

```toml
[workspace]
resolver = "2"
members = [
    "desktop/shared",
    "desktop/macos",
    "desktop/linux",
    "desktop/windows",
    "shared/logging",
    "shared/protocol",
    "shared/ui_logic",
    "shared/client",
    "interfaces/cli",
    "interfaces/tui",
    "interfaces/webchat",
]

[workspace.package]
version = "0.1.0"
edition = "2021"
license = "MIT"
rust-version = "1.92"
```

## 特性标志

Aleph 默认编译**所有生产特性**。只有测试和实验特性会被门控：

```toml
[features]
default = []
loom = ["dep:loom"]               # 使用 Loom 进行并发测试
test-helpers = []                  # 集成测试工具
control-plane = []                 # 实验性：Control Plane UI
disabled-tests = []                # 禁用的测试模块（等待重写）
telegram-draft-api = []            # 实验性：Telegram Draft API
```

**生产构建不需要任何特性标志。** 所有通道、工具和集成都在编译时包含，并通过 `aleph.toml` 配置在运行时启用/禁用。

### 构建

```bash
# 默认构建（所有生产特性）
cargo run -p alephcore --bin aleph-server

# 发布构建
cargo build -p alephcore --bin aleph-server --release

# 使用 Loom 并发测试
cargo test -p alephcore --features loom --lib loom
```

## 构建配置

### 调试构建（开发）

```bash
# 快速编译，较慢的运行时，完整的调试符号
cargo build -p alephcore --bin aleph-server

# 直接运行（构建 + 执行）
cargo run -p alephcore --bin aleph-server
```

调试构建编译速度快，但生成的大型未优化二进制文件。将其用于日常开发。

### 发布构建（生产）

```bash
# 优化的二进制文件
cargo build -p alephcore --bin aleph-server --release

# 二进制文件位置
ls -lh target/release/aleph-server
```

工作区的发布配置会去除调试符号：

```toml
[profile.release]
strip = true
```

### 验证构建

```bash
# 检查版本
./target/release/aleph-server --version

# 检查帮助
./target/release/aleph-server --help

# 验证嵌入式 UI 资源（如果构建了 control-plane）
strings target/release/aleph-server | grep "index.html"
```

## 构建 WebChat Panel UI

WebChat Panel 是一个基于 Web 的聊天界面，编译为 WASM 并由网关提供服务。`just` 构建管道通过 `wasm` 配方自动处理此过程。

### 手动 WASM 构建

如果你需要单独构建 WASM 面板：

```bash
# justfile 处理完整的 WASM 构建管道：
just wasm
```

这将：
1. 编译 Tailwind CSS（`interfaces/webchat`）
2. 将 Rust 代码构建为 WASM（`cargo build -p aleph-panel --target wasm32-unknown-unknown`）
3. 使用 `wasm-bindgen` 生成 JavaScript 绑定
4. 创建运行时 `index.html`

网关直接从 `interfaces/webchat/dist/` 提供 WASM 资源。

## 使用 Just（推荐）

项目包含一个 `justfile`，其中有常见的构建任务。首先安装 `just`：

```bash
cargo install just
```

可用的配方：

```bash
# 显示所有可用配方
just --list

# 开发：构建 WASM 面板 + 以调试模式运行服务器
just dev

# 完整发布构建（WASM + Swift 桥接 + 服务器）
just build

# 仅构建服务器（调试）
just build-debug

# 仅构建 WASM Panel UI
just wasm

# 快速编译检查
cargo check -p alephcore

# 运行核心测试
just test

# 运行所有测试（核心 + 桌面 + proptest）
just test-all

# 使用 clippy 进行代码检查
just clippy

# 清理构建产物
just clean
```

## 交叉编译说明

Aleph 主要在 **macOS (ARM64)** 上开发和测试。需要注意的交叉编译目标：

| 目标 | 状态 | 说明 |
|------|------|------|
| `aarch64-apple-darwin` | 主要 | 默认开发目标 |
| `x86_64-apple-darwin` | 支持 | Intel Mac 构建 |
| `x86_64-unknown-linux-gnu` | 支持 | Linux 服务器部署 |
| `aarch64-unknown-linux-gnu` | 支持 | Linux ARM（树莓派等） |
| `wasm32-unknown-unknown` | 必需 | 仅 Control Plane UI |

对于从 macOS 到 Linux 的交叉编译，可以考虑使用 [cross](https://github.com/cross-rs/cross)：

```bash
cargo install cross
cross build --bin aleph-server --target x86_64-unknown-linux-gnu --release
```

## 构建故障排除

### `rusqlite` 编译失败

`bundled` 特性标志从源码编译 SQLite，需要 C 编译器。在 Linux 上：

```bash
sudo apt install build-essential
```

### OpenSSL 链接错误

如果 `reqwest` 在 Linux 上链接 OpenSSL 失败：

```bash
sudo apt install libssl-dev pkg-config
```

在 macOS 上，这不应该发生，因为 `native-tls` 使用系统 Security 框架。

### WASM 构建失败

如果 Control Plane WASM 构建失败，请确保已安装目标：

```bash
rustup target add wasm32-unknown-unknown
cargo install wasm-bindgen-cli
```

注意：服务器可以在没有 UI 的情况下成功构建。如果 `dist/` 目录存在，`build.rs` 会跳过 UI 构建步骤。`control-plane` 特性是可选的。

### 端口已被占用

如果服务器因端口 18790 被占用而无法启动：

```bash
# 查找使用该端口的进程
lsof -i :18790

# 终止它
kill -9 <PID>

# 或在不同端口上启动
cargo run --bin aleph-server -- --port 8080
```

### 构建缓慢

为了在开发期间加快增量构建速度：

```bash
# 使用 mold 链接器（Linux）
cargo install mold
RUSTFLAGS="-C link-arg=-fuse-ld=mold" cargo build

# 或在 dev 配置中减少 codegen 单元（已经是默认）
# [profile.dev]
# codegen-units = 256  # 更多单元 = 更快编译，更慢运行时
```

当你只需要验证编译而不生成二进制文件时，考虑使用 `cargo check` 代替 `cargo build`。

> *"这是人类历史上第一次，机器的灵魂被赋予了身体。"*
> — 《攻壳机动队》

**Aleph** 是一个用 Rust 构建的强大个人 AI 助手，旨在让 AI 具备与世界交互的能力。它结合了高性能的 Gateway 控制平面和多通道消息支持，使 AI Agent 能够跨各种平台无缝工作。

> **一核多端，壳核分离；网关互通，权限控制。**
>
> 一个 `aleph-server` 核心驱动所有界面——桌面应用、CLI、TUI、Telegram、Discord 等。原生桌面壳是零业务逻辑的薄容器；所有推理、工具和状态都在核心中。每个终端通过同一个经过认证的 WebSocket 网关连接，无论是本地还是远程。

## 主要特性

- **多提供商 AI** - Claude、GPT、Gemini、Ollama，支持自动故障转移
- **多通道** - Telegram、Discord、iMessage、Slack、WebChat、桌面应用
- **原生性能** - 100% Rust 核心，无解释型语言
- **壳核分离** - 桌面壳纯 OS 集成；核心可独立运行
- **远程核心** - 桌面壳可通过 Tailnet/VPN 连接远程 `aleph-server`
- **集群联邦** - 一个中心核编排多台机器（节点）作为执行臂
- **安全执行** - 三级安全模型用于命令执行
- **可扩展** - MCP 集成、插件系统、自定义工具
- **自托管** - 在你自己的硬件上运行，完全控制

## 架构

```
                               终端
    ┌─────────────────────────────────────────────────────────────┐
    │ 桌面应用  │  Telegram  │  Discord  │  iMessage  │  CLI   │
    │ (Tauri 壳)│            │           │            │        │
    └───────┬───────┴────────────┴───────────┴────────────┴───┬───┘
            │                                                │
            │  WebSocket (JSON-RPC 2.0)                      │
            │  ws://127.0.0.1:18790/ws                       │
            │                                                │
    ┌───────▼────────────────────────────────────────────────▼───┐
    │                    Aleph 核心 (aleph-server)                │
    │  ┌─────────┐  ┌───────────┐  ┌─────────┐  ┌─────────────┐ │
    │  │  Agent  │  │  Session  │  │  Tools  │  │   Gateway   │ │
    │  │  Loop   │  │  Manager  │  │ Registry│  │  (Auth)     │ │
    │  └─────────┘  └───────────┘  └─────────┘  └─────────────┘ │
    └────────────────────────────────────────────────────────────┘
```

**壳核分离。** 桌面应用是一个薄 Tauri v2 壳——窗口、托盘、通知、自动更新、全局热键。它包含**零业务逻辑和零业务 UI**。所有产品 UI 在由核心提供的 Leptos Panel 中（`http://127.0.0.1:18790`）。所有推理、工具和状态在 `aleph-server` 中。壳可以连接本地核心或远程核心。

## 快速开始

```bash
# 通过 cargo 安装
cargo install aleph-cli

# 运行设置向导
aleph wizard

# 开始聊天
aleph chat "你好, Aleph!"
```

## 设计理念

Aleph 构建于五层涌现架构之上：

1. **知识之海** - AI 的预训练基础
2. **领域分类** - 组织化的专业知识
3. **原子技能** - 从知-什么到知-如何
4. **功能模块** - 可组合的能力
5. **多态 Agent** - 灵魂获得身体

目标不仅仅是构建一个工具，而是探索通往 AGI 的路径——当智能获得行动能力时。

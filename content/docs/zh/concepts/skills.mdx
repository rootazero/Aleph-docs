技能是由基于 markdown 的 `SKILL.md` 文件定义的可重用能力。与静态工具定义不同，技能提供自然语言指令，指导 LLM 在特定任务中的行为。

> 技能是三种**扩展**类型（`Skill`、`Plugin`、`Mcp`）之一。在日常使用中，可在**扩展商店**（[/docs/zh/concepts/extensions-store](/docs/zh/concepts/extensions-store)）中按功能类别浏览并安装技能，与插件和 MCP 服务器并列。本页聚焦于技能定义与激活。

## 什么是技能？

Aleph 中的技能包括：

- **SKILL.md** — 一个 markdown 文件，描述技能应该在何时以及如何使用，用自然语言编写供 LLM 解释
- **Manifest** — 从 SKILL.md frontmatter 解析的结构化元数据（名称、描述、触发器）
- **可选工具** — 技能可以调用内置工具、WASM 插件或 Node.js 扩展

技能从技能目录中发现并在运行时加载：

```
~/.aleph/skills/
├── code-review/
│   └── SKILL.md
├── git-commit/
│   └── SKILL.md
└── translate/
    └── SKILL.md
```

## 技能系统架构

```
SkillSystem (门面)
  ├── SkillRegistry — 存储加载的技能清单
  ├── SkillSnapshot — 用于提示构建的不可变视图
  ├── EligibilityService — 确定哪些技能适用
  └── Events — 广播技能系统更改
```

## 核心组件

### SkillSystem

主入口点，通过 `Arc` 廉价克隆：

```rust
// src/skill/mod.rs
pub struct SkillSystem {
    inner: Arc<Inner>,
}

struct Inner {
    registry: RwLock<SkillRegistry>,
    snapshot: RwLock<SkillSnapshot>,
    eligibility: EligibilityService,
    config: RwLock<SkillsConfig>,
    // ...
}

impl SkillSystem {
    pub async fn init(&self, 
        dirs: Vec<PathBuf>
    ) -> Result<(), SkillSystemError>;
    
    pub async fn current_snapshot(&self
    ) -> SkillSnapshot;
}
```

### 技能注册表

维护加载的技能集合：

```rust
// src/skill/registry.rs
pub struct SkillRegistry {
    skills: HashMap<SkillId, SkillManifest>,
}

impl SkillRegistry {
    pub fn register(&mut self, 
        manifest: SkillManifest
    );
    
    pub fn get(&self, 
        id: &SkillId
    ) -> Option<&SkillManifest>;
}
```

### 技能清单

从 SKILL.md frontmatter 解析：

```rust
// src/domain/skill.rs
pub struct SkillManifest {
    pub id: SkillId,
    pub name: String,
    pub description: String,
    pub triggers: Vec<String>,
    pub tools: Vec<String>,
    pub prompt: String,
}
```

### 资格服务

确定哪些技能与给定上下文相关：

```rust
// src/skill/eligibility.rs
pub struct EligibilityService;

impl EligibilityService {
    pub fn evaluate(
        &self,
        skills: &[SkillManifest],
        context: &Context,
    ) -> Vec<EligibilityResult>;
}
```

### 提示构建器

将符合条件的技能注入系统提示词：

```rust
// src/skill/prompt.rs
pub fn build_skills_prompt_xml(
    snapshot: &SkillSnapshot,
    eligible: &[SkillId],
) -> String;
```

技能在提示词中格式化为 XML：

```xml
<skills>
  <skill id="code-review">
    <description>Review code for bugs and style issues</description>
    <instructions>
      When the user shares code, analyze it for:
      - Logic errors
      - Security vulnerabilities
      - Performance issues
    </instructions>
  </skill>
</skills>
```

## 技能加载流程

1. **扫描** — 在配置的目录中发现 SKILL.md 文件
2. **解析** — 从每个文件中提取 frontmatter 和正文
3. **注册** — 使用唯一 ID 添加到 SkillRegistry
4. **快照** — 构建用于查询的不可变 SkillSnapshot
5. **注入** — 符合条件的技能注入 LLM 提示词

## SKILL.md 格式

```markdown
---
id: code-review
name: Code Review
description: Review code for bugs and improvements
triggers:
  - code review
  - review this
tools:
  - file_read
---

When the user shares code, analyze it systematically:

1. Check for logic errors and edge cases
2. Identify security vulnerabilities
3. Suggest performance improvements
4. Note style inconsistencies

Always provide specific line references when possible.
```

## 配置

技能在 `~/.aleph/data/skills.toml` 中配置：

```toml
[skills]
enabled = true
auto_match = false  # 需要显式激活

[[skills.directories]]
path = "~/.aleph/skills"
priority = 1
```

## 斜杠命令

技能可以定义用于直接调用的斜杠命令：

```rust
// src/skill/commands.rs
pub fn resolve_command(
    input: &str,
    skills: &SkillSnapshot,
) -> Option<SkillCommandSpec>;
```

示例: `/code-review` 为当前会话激活 code-review 技能。

## 模块结构

| 模块 | 目的 |
|--------|---------|
| `commands` | 斜杠命令解析 |
| `compat` | 向后兼容性 |
| `config` | 技能配置 |
| `eligibility` | 基于上下文的技能过滤 |
| `events` | 技能系统事件 |
| `installer` | 技能安装 |
| `manifest` | SKILL.md 解析 |
| `prompt` | 提示 XML 生成 |
| `recaller` | 技能召回跟踪 |
| `registry` | 技能注册 |
| `snapshot` | 不可变技能视图 |
| `status` | 技能状态跟踪 |
| `tools` | 技能工具集成 |

`tools` 模块为 Aleph 的工具系统提供基础设施。它处理工具注册、模式生成、执行、修复和包装工具调用的中间件管道。

## 设计理念

1. **模式驱动** — 所有工具通过 JSON 模式声明其输入模式
2. **修复友好** — 失败的工具调用可以自动修复并重试
3. **中间件管道** — 权限检查、日志记录和指标发生在分层中间件中
4. **类型安全执行** — 带有动态调度支持的强类型工具 trait

## 工具 Traits

```rust
/// 核心工具 trait
pub trait AlephTool: Send + Sync {
    fn name(&self) -> &str;
    fn description(&self) -> &str;
    fn schema(&self) -> Value; // JSON 模式
    async fn execute(&self, input: Value, ctx: &ExecutionContext) -> ToolResult;
}
```

## 模式生成

工具模式使用反射从 Rust 类型生成：

```rust
// 派生宏生成 JSON 模式
#[derive(ToolSchema)]
pub struct SearchInput {
    pub query: String,
    pub limit: Option<u32>,
}
```

`schema_strictify` 模块移除 LLM 不友好的字段（如复杂类型上的 `format`、`pattern`）以减少 token 使用。

## 修复系统

当工具调用失败时，修复系统尝试常见修复：

1. **大小写纠正** — `getMP3File` → `get_mp3_file`
2. **Unicode 处理** — 非 ASCII 的适当 `to_lowercase()`
3. **模式验证** — 执行前根据模式检查输入

工具调用流经中间件层：

```
用户请求
    │
    ▼

    │
    ▼

    │
    ▼

```

## Docker 沙箱

对于不受信任的工具，Docker 提供容器化执行：

- **允许列表** — 仅允许显式允许的 Docker 标志
- **卷挂载** — 工作区目录以读写方式挂载
- **网络** — 可配置的网络访问（启用/禁用）

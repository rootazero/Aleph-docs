`secrets` 模块为敏感凭据（API 密钥、令牌、密码）提供加密存储。它使用 AES-256-GCM 和每条目 HKDF-SHA256 密钥派生，支持多个秘密提供商（1Password、环境变量），并检测工具输出中的意外秘密泄漏。

## 设计理念

1. **每条目加密** — 每个秘密都有自己的派生密钥和随机 nonce/盐
2. **丢弃时清零** — 敏感数据在不再需要时从内存中清除
3. **泄漏检测** — 扫描工具输出以发现意外的秘密暴露

## 核心组件

### SecretVault

基于文件的加密存储：

```rust
pub struct SecretVault {
    data: VaultData,
    path: PathBuf,
}

impl SecretVault {
    pub fn open(path: impl Into<PathBuf>) -> Result<Self> { /* ... */ }

    pub fn get(&self, name: &str) -> Option<&EncryptedEntry> { /* ... */ }

    pub fn store(
        &mut self,
        name: &str,
        entry: EncryptedEntry,
    ) -> Result<()> { /* ... */ }
}
```

**位置：** `~/.aleph/secrets.vault`

**原子写入：** Vault 文件写入临时位置并原子重命名以防止损坏。

### 加密

```rust
// 通过 HKDF-SHA256 的每条目密钥派生
let key = hkdf_derive(master_key, salt, info);

// 使用随机 nonce 的 AES-256-GCM 加密
let ciphertext = aes_256_gcm_encrypt(plaintext, key, nonce);
```

**主密钥：** 受 `SecretString` 保护（丢弃时清零）。

### DecryptedSecret

```rust
pub struct DecryptedSecret {
    pub value: SecretString,
    pub metadata: EntryMetadata,
}
```

`SecretString` 实现：
- `Zeroize` — 丢弃时清除内存
- 修订的 `Debug`/`Display` — 打印 `[REDACTED]` 而非值

## 秘密提供商

秘密可以从多个提供商获取：

### 1Password 集成

```rust
pub struct OnePasswordProvider {
    vault: String,
    item: String,
}
```

使用 1Password CLI (`op`) 检索秘密。健康检查验证认证而不在错误消息中暴露凭据。

### 环境变量

```rust
pub struct EnvProvider {
    prefix: String,
}
```

从具有可配置前缀的环境变量中读取秘密。

### Web3 签名者

```rust
pub struct EvmSigner {
    resolver: Box<dyn SecretResolver>,
}
```

使用安全存储的私钥签署以太坊交易。

## 秘密注入

模板可以通过占位符语法引用秘密：

```rust
pub fn render_with_secrets(
    template: &str,
    resolver: &dyn AsyncSecretResolver,
) -> Result<String>
```

**占位符语法：** `{{ secret:name }}`

解析器从库中获取命名秘密并将其替换到模板中。

## 泄漏检测

扫描工具输出以发现意外的秘密暴露：

```rust
pub struct LeakDetector {
    patterns: Vec<Regex>,
}

impl LeakDetector {
    pub fn check(&self,
        output: &str,
    ) -> LeakDecision { /* ... */ }
}
```

**决策：**
- `LeakDecision::Clean` — 未检测到秘密
- `LeakDecision::Suspicious` — 可能的泄漏，记录警告
- `LeakDecision::Leaked` — 确认的泄漏，阻止输出

**集成：** `RuntimeSecurityGuard` 在发送到 LLM 之前对所有工具输出运行泄漏检测。

---

## 安全属性

- **锁恢复** — `unwrap_or_else(|e| e.into_inner())` 用于所有 Mutex 锁
- **无 stderr 泄漏** — 错误消息使用固定安全文本，原始 stderr 通过 `tracing::debug` 记录
- **JSON 解析失败记录** — `serde_json` 错误被调试记录，而非静默吞咽
- **清零** — `SecretString` 丢弃时清除内存

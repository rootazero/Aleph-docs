---

`daemon` 模块将 Aleph 管理为跨平台的持久系统服务。它处理服务安装、生命周期管理、资源监控、进程间通信和事件驱动的主动行动。

## 设计理念

守护进程遵循三个原则：

1. **跨平台抽象** —— 平台特定的服务 API 隐藏在 `ServiceManager` trait 后面
2. **资源感知** —— 守护进程监控系统负载并在资源稀缺时限制操作
3. **事件驱动主动性** —— OS 级事件触发代理行动，无需用户干预

## 核心组件

### ServiceManager

系统服务管理的跨平台 trait：

```rust
pub trait ServiceManager: Send + Sync {
    async fn install(&self, config: &DaemonConfig) -> Result<()>;
    async fn start(&self) -> Result<()>;
    async fn stop(&self) -> Result<()>;
    async fn status(&self) -> ServiceStatus;
}
```

**实现：**
- `LaunchdService` (macOS) —— 管理 launchd plist 和服务生命周期
- `SystemdService` (Linux) —— 计划中
- `WindowsService` (Windows) —— 计划中

### ResourceGovernor

监控系统资源并做出执行决策：

```rust
pub struct ResourceGovernor {
    limits: ResourceLimits,
}

impl ResourceGovernor {
    pub async fn is_safe_to_run(&self,
    ) -> bool { /* ... */ }
    
    pub fn current_pressure(&self) -> PressureLevel { /* ... */ }
}
```

**监控：**
- CPU 使用率
- 内存消耗
- 电池电量
- 系统平均负载

### IPC 服务器

通过 Unix 域套接字的 JSON-RPC 2.0 服务器：

```rust
pub struct IpcServer {
    socket_path: PathBuf,
}
```

**套接字路径：** `~/.aleph/daemon.sock`

**方法：**
- `daemon.status` —— 获取守护进程状态
- `daemon.ping` —— 健康检查
- `daemon.shutdown` —— 优雅关闭

### 事件总线

为感知层发布 OS 级事件：

```rust
pub struct DaemonEventBus {
    sender: broadcast::Sender<DaemonEvent>,
}
```

**事件类型：**
- `ProcessEvent` —— 应用程序启动/退出
- `FsEvent` —— 文件创建/修改/删除
- `TimeTrigger` —— 基于时间的计划事件
- `SystemEvent` —— 电池、网络、显示状态变化

## CLI 命令

```bash
# 将守护进程安装为系统服务
aleph daemon install

# 启动守护进程
aleph daemon start

# 检查状态
aleph daemon status

# 停止守护进程
aleph daemon stop

# 卸载守护进程
aleph daemon uninstall

# 在前台运行（开发）
aleph daemon run
```

## 分派器

`Dispatcher` 将事件路由到适当的行动：

```rust
pub struct Dispatcher {
    policy_engine: PolicyEngine,
    action_executor: ActionExecutor,
}
```

**流程：**
1. 原始 OS 事件进入事件总线
2. `PolicyEngine` 评估风险并决定行动
3. `ActionExecutor` 执行行动（如果资源受限则排队）

## 安全属性

- **最小 unsafe** —— 只有 `libc::kill` 用于 SIGTERM 信号
- **UTF-8 安全** —— 没有字节级字符串切片
- **锁恢复** —— 使用 `unwrap_or_else(|e| e.into_inner())` 模式
- **无 SQL 注入** —— 守护进程不使用数据库查询

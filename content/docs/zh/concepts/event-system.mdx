`event` 模块实现一个事件驱动架构，解耦 Aleph 的子系统。模块之间不是直接方法调用，而是组件将事件发布到中央总线并订阅它们关心的事件。

## 设计理念

事件驱动架构在 Aleph 中提供三个好处：

1. **解耦** — 内存系统不需要了解网关；两者都只使用事件
2. **可观察性** — 每个重要动作都会产生一个事件，可以被记录、审计或流式传输到客户端
3. **可扩展性** — 新功能可以订阅现有事件而无需修改发布者

EventBus 是一个单例（通过 `Lazy` 初始化），在进程的生命周期内存在。所有事件处理都是异步且非阻塞的。

## 核心组件

### EventBus

中央事件分发系统。

```rust
// 单例访问
let bus = event::bus();
bus.publish(SessionCreated { session_id: id }).await?;
```

EventBus：
- 维护每个事件类型的类型化事件队列
- 异步地将事件分发给所有已注册的处理程序
- 永远不会阻塞发布者 —— 事件被排队并并发处理

### EventHandler Trait

```rust
#[async_trait]
pub trait EventHandler<E: Event>: Send + Sync {
    async fn handle(&self, event: &E) -> Result<(), EventError>;
}
```

实现者接收特定类型的事件并执行副作用：
- 日志处理程序写入磁盘
- WebSocket 处理程序转发到连接的客户端
- 内存处理程序更新知识图谱

### EventFilter

选择性订阅 —— 处理程序仅接收与其过滤器匹配的事件。

```rust
pub enum EventFilter {
    All,                    // 接收所有事件
    Type(EventType),       // 仅特定事件类型
    Source(String),        // 仅来自特定源模块
    Custom(Box<dyn Fn(&Event) -> bool>),
}
```

**示例：** Gateway 的 WebSocket 处理程序过滤 `TraceEvent` 类型，以将代理执行跟踪流式传输到 Panel UI。

## 事件类型

### 核心事件

| 事件 | 说明 | 发布者 |
|---|---|---|
| `SessionCreated` | 新会话已初始化 | SessionService |
| `MessageReceived` | 用户消息已到达 | Gateway |
| `ToolExecuted` | 工具已完成执行 | Engine |
| `MemoryUpdated` | 事实已添加或失效 | Memory |
| `ConfigReloaded` | 配置已热重载 | Config watcher |

### 事件排序

每个事件携带单调序列号：

```rust
pub struct EventMetadata {
    pub sequence: u64,      // 全局排序
    pub timestamp: i64,     // Unix 毫秒
    pub source: String,     // 模块名称
}
```

序列号启用：
- 事件重放以进行调试
- 确定性状态重建
- 跨分布式处理程序的排序保证

## 权限和安全

### EventPermission

并非所有处理程序都能看到所有事件。权限层检查处理程序授权：

```rust
pub enum EventPermission {
    Owner,      // 完全访问
    Guest,      // 仅他们会话的事件
    System,     // 仅内部事件
}
```

**示例：** 访客的 WebSocket 连接仅接收他们自己会话的事件，防止跨会话信息泄漏。

### EventQuestion

对于敏感操作，EventBus 可以暂停并请求确认：

```rust
pub enum EventQuestion {
    Allow,      // 继续事件
    Deny,       // 丢弃事件
    Modify(Event), // 用修改后的事件替换
}
```

**示例：** 在执行 `bash_exec` 工具之前，系统发布 `ToolExecutionRequested` 事件。如果审批模块拒绝它，执行会在任何损害发生之前被取消。

## 使用示例

```rust
use aleph::event::{EventBus, EventHandler, Event};

// 定义一个事件
#[derive(Clone, Debug)]
pub struct MemoryCreated {
    pub fact_id: String,
    pub content: String,
}

impl Event for MemoryCreated {}

// 实现一个处理程序
pub struct MemoryLogger;

#[async_trait]
impl EventHandler<MemoryCreated> for MemoryLogger {
    async fn handle(&self, event: &MemoryCreated) -> Result<(), EventError> {
        tracing::info!("Memory created: {}", event.fact_id);
        Ok(())
    }
}

// 注册并发布
let bus = event::bus();
bus.register(MemoryLogger).await?;
bus.publish(MemoryCreated {
    fact_id: "fact-123".into(),
    content: "User prefers dark mode".into(),
}).await?;
```

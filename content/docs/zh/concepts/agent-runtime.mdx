`agents` 模块实现 Aleph 的代理运行时系统。它管理单个代理的生命周期，支持为并行任务生成子代理，并协调多代理团队。

## 设计理念

代理运行时遵循三个原则：

1. **状态机生命周期** —— 每个代理通过定义良好的状态转换（Pending → Running → Completed/Failed）
2. **确定性执行** —— 给定相同的输入和状态，代理行为是可重现的
3. **优雅降级** —— 失败的子代理不会使父代理崩溃；错误被收集和报告

## 核心组件

### 代理状态机

代理遵循具有每次转换验证的严格状态机：

```
┌─────────┐    start()    ┌─────────┐   complete()  ┌──────────┐
│ Pending │ ─────────────→│ Running │ ────────────→│ Completed│
└─────────┘               └────┬────┘              └──────────┘
                               │
                               │ fail()
                               ▼
                          ┌─────────┐
                          │ Failed  │
                          └─────────┘
```

每次转换都通过 `can_transition_to()` 验证：

```rust
impl Agent {
    pub fn can_transition_to(&self, target: AgentState) -> bool {
        match (self.state, target) {
            (Pending, Running) => true,
            (Running, Completed) => true,
            (Running, Failed) => true,
            _ => false,
        }
    }
}
```

这防止了从 Failed 返回到 Running 等非法转换。

### 子代理生成

代理可以为并行任务执行生成子代理：

```rust
pub struct Agent {
    id: AgentId,
    parent: Option<AgentId>,
    children: Vec<AgentId>,
    state: AgentState,
}

impl Agent {
    pub async fn spawn_subagent(
        &mut self,
        task: Task,
    ) -> Result<AgentId, AgentError> {
        let child = Agent::new(task);
        let id = child.id.clone();
        self.children.push(id.clone());
        registry.register(child).await?;
        Ok(id)
    }
}
```

**关键行为：**
- 父代理等待所有子代理完成后才能完成自身
- 子代理失败通过 `ResultCollector` 冒泡到父代理
- 子代理继承父代理的 `IdentityContext`，但可以有受限的范围

### 注册表

`AgentRegistry` 维护所有活动代理并提供按 ID 查找：

```rust
pub struct AgentRegistry {
    agents: RwLock<HashMap<AgentId, Agent>>,
}
```

**特性：**
- 确定性行为的排序迭代（`registry.iter().sorted_by_key(|(id, _)| id)`）
- 通过 `unwrap_or_else(|e| e.into_inner())` 的锁中毒安全
- TTL 到期后清理完成的代理

### 结果收集器

聚合来自子代理的结果：

```rust
pub struct ResultCollector {
    results: Vec<AgentResult>,
    max_preview_len: usize,
}

impl ResultCollector {
    pub fn add_result(&mut self,
        result: AgentResult,
    ) {
        // 使用 Unicode 安全截断截断预览
        let preview = safe_truncate(&result.output, self.max_preview_len);
        self.results.push(AgentResult {
            output: preview,
            ..result
        });
    }
}
```

## 团队协调

多个代理可以组成团队进行协作任务：

```rust
pub struct AgentTeam {
    leader: AgentId,
    members: Vec<AgentId>,
    shared_context: SharedContext,
}

impl AgentTeam {
    pub async fn broadcast(
        &self,
        message: TeamMessage,
    ) -> Result<(), TeamError> {
        for member in &self.members {
            send_message(member, message.clone()).await?;
        }
        Ok(())
    }
}
```

**模式：**
- **领导者选举** —— 一个代理充当协调器
- **共享上下文** —— 所有团队成员访问共享知识库
- **广播/多播** —— 消息可以发送给所有成员或特定子集

## 上下文提供程序

代理从多个来源接收上下文：

```rust
pub trait ContextProvider: Send + Sync {
    fn get_context(
        &self,
        agent_id: &AgentId,
    ) -> Context;
}
```

**提供程序：**
- `MemoryContextProvider` —— 来自知识库的相关记忆
- `SessionContextProvider` —— 当前会话历史
- `ToolContextProvider` —— 可用工具及其模式

**注意：** `get_context` 方法是同步的（返回 `Context`，不是 `Future<Context>`）。对于异步上下文源，使用 `block_in_place` —— 但这需要多线程 Tokio 运行时。

## 代理规则

行为规则约束代理可以做什么：

```rust
pub struct AgentRules {
    max_iterations: u32,
    allowed_tools: Vec<String>,
    forbidden_patterns: Vec<Regex>,
}
```

规则在 Harness 层强制执行，可以：
- **继承**自父代理
- **覆盖**每个子代理
- **在运行时**通过网关 API 更新

`a2a` 模块实现 A2A（代理到代理）协议适配器。A2A 使 Aleph 能够与其他 AI 代理通信、将任务委托给外部代理系统，并以标准化格式接收结果。

## 设计理念

A2A 集成遵循两个原则：

1. **协议抽象** —— Aleph 原生讲 A2A；适配器处理与内部类型的翻译
2. **信任边界** —— 外部代理使用与内部工具相同的权限模型对待

---

## 协议概览

A2A 是一种代理到代理通信协议，定义了：
- **任务委托** —— 一个代理如何请求另一个代理执行工作
- **状态更新** —— 进度如何报告回来
- **结果交付** —— 完成的工作如何返回
- **能力发现** —— 代理如何宣传它们能做什么

```
┌─────────────┐         A2A 协议          ┌─────────────┐
│   Aleph     │  ───────────────────────→ │ 外部        │
│  代理       │  任务请求 + 上下文         │  代理       │
│             │  ←─────────────────────── │             │
│             │  状态更新 + 结果           │             │
└─────────────┘                           └─────────────┘
```

## 核心类型

### A2ATask

表示委托给外部代理的任务：

```rust
pub struct A2ATask {
    pub task_id: String,
    pub description: String,
    pub input: serde_json::Value,
    pub deadline: Option<i64>,
}
```

### A2AResult

A2A 任务的结果：

```rust
pub struct A2AResult {
    pub task_id: String,
    pub status: A2AStatus,
    pub output: Option<serde_json::Value>,
    pub error: Option<String>,
}

pub enum A2AStatus {
    Pending,
    InProgress,
    Completed,
    Failed,
    Cancelled,
}
```

### FileContent

根据 A2A 规范不变量验证文件附件：

```rust
pub struct FileContent {
    pub mime_type: String,
    pub data: Vec<u8>,
    pub name: String,
}

impl FileContent {
    pub fn validate(&self) -> Result<(), ValidationError> {
        // 强制执行：非空 MIME 类型、非空数据、
        // 有效文件名（无路径遍历、无空字节）
    }
}
```

## 信任等级

A2A 连接使用与内部组件相同的 `TrustLevel` 系统：

| 等级 | 行为 |
|---|---|
| `Local` | 完全访问，无需审批 |
| `Private` | 标准审批流程 |
| `Public` | 最严格的审批，所有行动都经过审计 |
| `Untrusted` | 默认阻止 |

信任等级从连接来源（IP 地址、主机名或 mTLS 证书）推断。

## 能力发现

在委托任务之前，Aleph 查询外部代理的能力：

```rust
pub async fn discover_capabilities(
    &self,
    endpoint: &str,
) -> Result<Vec<Capability>, A2AError> {
    let response = self.client
        .get(format!("{}/capabilities", endpoint))
        .send().await?;
    
    response.json().await
}
```

能力在本地缓存以避免重复发现调用。

## 集成点

A2A 适配器与以下组件集成：
- **网关** —— 接收来自外部代理的 A2A 请求
- **编排器** —— 决定在本地处理还是通过 A2A 委托
- **审批** —— 将敏感 A2A 操作路由通过审批工作流
- **会话** —— 在重新启动时持久化 A2A 任务状态

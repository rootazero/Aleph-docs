`process_supervisor` 和 `verification` 模块处理外部进程控制和代理停止时间验证。监管器在伪终端中生成 CLI 工具并监控其输出，而验证提供在代理循环停止之前运行的钩子。

## 设计理念

1. **基于 PTY 的控制** — 外部工具在伪终端中运行以实现真实的 I/O
2. **事件检测** — 输出被解析以获取语义事件（审批请求、错误）
3. **提示级验证** — 代理自检存在于提示词中，而非 Rust 代码（R8, R10）

## 进程监管器

### PtySupervisor

生成和控制外部 CLI 工具：

```rust
pub struct PtySupervisor {
    master: Box<dyn MasterPty>,
    reader: mpsc::Receiver<String>,
    running: AtomicBool,
}

impl PtySupervisor {
    pub fn spawn(command: &str) -> Result<Self> { /* ... */ }

    pub fn send_input(&self,
        input: &str,
    ) -> Result<()> { /* ... */ }

    pub fn read_output(&self) -> Option<String> { /* ... */ }
}
```

**能力：**
- 在伪终端中生成进程
- 通过后台读取器线程实时读取输出
- 注入输入（命令、审批）
- 检测输出中的语义事件

### 事件检测

扫描输出以查找指示以下内容的模式：
- 审批请求（例如，"Allow this action?"）
- 错误条件
- 完成信号

**线程安全：** 读取器线程通过 `mpsc` + `AtomicBool` 与主线程通信，具有正确的 `Acquire/Release` 排序。

## 停止钩子

停止钩子在代理循环终止之前运行，启用外部验证：

```rust
#[async_trait]
pub trait StopHookHandler: Send + Sync {
    fn name(&self) -> &str;

    async fn evaluate(
        &self,
        ctx: &StopHookContext,
        cancel: &CancellationToken,
    ) -> StopHookVerdict;
}
```

### ShellStopHook

运行外部 shell 命令：
- **退出 0:** 允许停止
- **退出 2:** 阻止停止（stdout = 原因）
- **其他:** 钩子错误（记录，不阻止）

```rust
pub struct ShellStopHook {
    hook_name: String,
    command: String,
    timeout: Duration,
}
```

### StopHookContext

通过 stdin 作为 JSON 传递给钩子：

```rust
pub struct StopHookContext {
    pub final_text: Option<String>,
    pub iterations: usize,
    pub tool_calls_made: usize,
    pub stop_reason: String,
}
```

### 裁决聚合

```rust
pub enum StopHookVerdict {
    Allow,
    Block { reason: String },
    Error { hook_name: String, message: String },
}
```

钩子通过 `join_all` 并行运行。第一个 `Block` 裁决停止代理；`Error` 裁决被记录但不阻止。

### 配置

```toml
[[stop_hooks]]
name = "lint-check"
command = "cargo clippy -- -D warnings"
timeout_secs = 60
```

## 验证模型

Aleph 的验证逻辑存在于提示词中（参见 `src/thinker/layers/agent_role.rs`）：

代理被指示在停止之前发出 `VERDICT: PASS|FAIL|PARTIAL` 块来总结其自检。根据红线 R8（LLM 主权）和 R10（智能存在于提示词中），不引入 Rust 级验证器、评判者或批评者。

**注意：** `VerifyStopHook` Rust 结构体从 2026 年 4 月到 P0 存在，但从未接入生产环境。它在 P4 中被删除，因为提示级机制完全覆盖了该用例。

## 安全属性

- **生产中无 unwrap** — 所有错误路径使用 `map_err` + `?`
- **String::from_utf8_lossy** — 安全处理可能无效的 UTF-8 输出
- **原子排序** — `Acquire/Release` 用于读取器线程同步
- **超时** — Shell 钩子具有可配置的超时（默认 30s）
- **取消** — 钩子尊重 `CancellationToken` 以实现优雅关闭

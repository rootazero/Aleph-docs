能力和权限模块控制插件和代理被允许做什么。能力系统声明插件贡献什么（工具、频道、技能），而权限系统在运行时强制执行访问控制。

## 设计理念

1. **声明式能力** —— 插件通过 `CapabilityDeclaration` 声明它们提供什么
2. **分层权限** —— 全局默认值 + 每个代理覆盖，最严格者获胜
3. **中间件执行** —— 权限检查在执行前发生在工具中间件层

---

## 能力声明

插件通过统一枚举声明其贡献：

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum CapabilityDeclaration {
    Tool(ToolDeclaration),
    Hook(HookDeclaration),
    Channel(ChannelDeclaration),
    Provider(ProviderDeclaration),
    GatewayMethod(GatewayMethodDeclaration),
    HttpRoute(HttpRouteDeclaration),
    Cli(CliDeclaration),
    Service(ServiceDeclaration),
    Command(CommandDeclaration),
    Skill(SkillDeclaration),
    Agent(AgentDeclaration),
    McpServer(McpServerDeclaration),
}
```

每个变体与 PluginRegistry 中的注册类型 1:1 映射。`#[serde(tag = "type")]` 属性序列化为 `{ "type": "tool", "name": "my_tool", ... }`。

## 权限系统

### 分类

工具调用被分类为三种结果：

```rust
pub enum Classification {
    Allow,                       // 无需提示执行
    Confirm { reason: String },  // 请求用户批准
    Deny { reason: String },     // 无条件拒绝
}
```

### SmartFilter

```rust
pub trait SmartFilter: Send + Sync {
    fn classify(&self,
        name: &str,
        input: &Value,
    ) -> Classification;
}
```

### LayeredPermissionResolver

使用最严格者获胜语义合并全局 + 每个代理权限：

```rust
pub struct LayeredPermissionResolver {
    merged: Arc<ArcSwap<ToolPermissionsConfig>>,
}
```

**合并规则：** `Deny > Ask > Allow`

**Exec 类工具**（`code_exec`、`bash`）有特殊处理 —— 它们的审批由 `WorkspaceSandbox` 拥有，不是一般权限层。

**实时重载：** 使用 `ArcSwap`，因此来自配置 RPC 处理程序的权限更新无需重新启动即可生效。

### ApprovalGate

对于 `Confirm` 分类，`ApprovalGate` 提示用户：

```rust
#[async_trait]
pub trait Approver: Send + Sync {
    async fn ask(
        &self,
        tool_name: &str,
        reason: &str,
    ) -> ApprovalOutcome;
}
```

真实的 `ApprovalGate` 实现这一点；模拟实现支持无需完整审批 UI 管道的单元测试。

## 权限配置

权限在 `providers.toml` 中或通过配置 RPC 配置：

```toml
[permissions.tools]
code_exec = "ask"
bash = "deny"
file_ops = "allow"
```

**操作：**
- `allow` —— 无需确认执行
- `ask` —— 执行前提示用户
- `deny` —— 阻止执行

---

## 安全属性

- **拒绝优先评估** —— 在任何 `Ask` 提示之前检查所有模式的 `Deny`
- **确定性排序** —— 配置规则按键排序以实现可重现的评估
- **无递归溢出** —— 模式匹配使用有界递归（模式来自配置，不是用户输入）

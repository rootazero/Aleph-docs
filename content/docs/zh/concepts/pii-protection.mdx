title: "PII 保护"
description: "在到达 LLM API 提供商之前检测和编辑个人可识别信息的网关级 PII 过滤引擎。"
---

`pii` 模块提供网关级隐私保护，在到达 LLM API 提供商之前过滤出站消息。与日志清理工具（接受误报）不同，此引擎针对精度进行了调优 — 误报会降低 LLM 理解能力。

## 设计理念

1. **精度优于召回** — 误报会损害代理性能；引擎偏向准确性
2. **基于规则的检测** — 已知 PII 类型的正则模式，而非启发式 ML
3. **允许列表豁免** — 已知安全值可以绕过过滤

## 核心组件

### PiiEngine

主过滤引擎：

```rust
pub struct PiiEngine {
    rules: Vec<Box<dyn PiiRule>>,
    allowlist: PiiAllowlist,
    action: PiiAction,
}

impl PiiEngine {
    pub fn filter(&self,
        text: &str,
    ) -> FilterResult { /* ... */ }
}
```

### PiiSeverity

```rust
pub enum PiiSeverity {
    Low,
    Medium,
    High,
    Critical,
}
```

### FilterResult

```rust
pub struct FilterResult {
    pub text: String,           // 过滤后的文本（带替换）
    pub blocked_count: usize,   // 替换的匹配数
    pub warned_count: usize,    // 记录但未替换
    pub matches: Vec<PiiMatch>,
}
```

## 检测规则

内置规则检测：

| 类型 | 示例 | 严重性 |
|---|---|---|
| **电子邮件地址** | `user@example.com` | Medium |
| **电话号码** | `+1-555-123-4567` | High |
| **信用卡** | `4111-1111-1111-1111` | Critical |
| **SSN / 身份证** | 中国身份证: `110101199001011234` | Critical |
| **API 密钥** | `sk-abc123...` | Critical |
| **SSH 密钥** | `ssh-rsa AAAA...` | High |
| **IP 地址** | `192.168.1.1` | Low |

### 身份证验证

中国身份证号码通过以下方式验证：
- 长度检查 (18 位数字)
- 省份代码验证 (前 2 位)
- 日期验证 (第 7-14 位)
- 校验和验证 (最后一位通过加权和)

**安全：** 使用 `.get(..n)` 而非字节切片以确保 UTF-8 安全。

## 允许列表

已知安全值可以免于过滤：

```rust
pub struct PiiAllowlist {
    entries: Vec<Regex>,
}
```

允许列表条目是在替换之前针对检测到的值匹配的正则模式。

---

## 操作

```rust
pub enum PiiAction {
    Block,   // 用占位符替换
    Warn,    // 记录但不替换
    Allow,   // 忽略
}
```

**平台策略：** 不同的提供商可能有不同的 PII 要求。引擎遵循配置中的 `PlatformPiiPolicy`。

---

## 安全属性

- **UTF-8 安全** — `.get(..n)` 用于所有字符串切片，`char_indices()` 用于边界检查
- **锁恢复** — `unwrap_or_else(|e| e.into_inner())` 用于 RwLock
- **无 `static mut`** — 使用 `OnceLock` 用于静态正则
- **正则安全** — 所有正则字面量使用 `.expect("valid regex literal")`

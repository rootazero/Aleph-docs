群聊是一个面板界面，[团队](/docs/zh/concepts/teams)中的多个代理在一个共享、持久的线程中对话。它把团队从"能运行"提升为"好用、易回访"：发言者在视觉上彼此区分，历史可从侧边栏恢复，`@` 提及是可被发现的，而不必手敲。

## 设计理念

1. **身份优先** — 每条消息都通过 emoji 头像 + 显示名归属到某个发言者，而不是埋在元数据里。
2. **可恢复** — 团队聊天像单代理聊天一样进入侧边栏历史；你随时可以回到它。
3. **可发现的提及** — 输入 `@` 会打开一个花名册面板，因此你绝不会因为拼错 `@<id>` 而悄悄丢失它。

## 三面板窗口

打开一个团队即进入群聊窗口，它分为三个面板布局：

```
┌──────────────┬─────────────────────────────────┬──────────────────┐
│   Roster     │        Message Flow             │   Workspace      │
│  (left)      │        (middle)                 │   (right)        │
├──────────────┼─────────────────────────────────┼──────────────────┤
│ 👑 leader    │  🦊 Fox                          │  [Artifacts]     │
│  ● online    │   ┌──────────────────────────┐  │  [Tasks/Kanban]  │
│ 🦉 Owl       │   │ message text             │  │                  │
│  ● working   │   └──────────────────────────┘  │  ┌────────────┐  │
│ 🐢 Turtle    │   │ merged follow-up message │  │  │ artifact   │  │
│  ○ idle      │   └──────────────────────────┘  │  │ task card  │  │
│              │  🦉 Owl                          │  │ ...        │  │
│              │   ┌──────────────────────────┐  │  └────────────┘  │
│              │   │ message text             │  │   (collapsible)  │
│              │   └──────────────────────────┘  │                  │
└──────────────┴─────────────────────────────────┴──────────────────┘
```

### 左侧 — 花名册（Roster）

团队花名册，在团队队长上带有 **队长徽章**，每个成员带有 **状态点**（online / working / idle）。这是本线程中团队的实时成员构成。

### 中间 — 消息流（归属气泡）

对话以经典即时通讯（IM）布局渲染为 **归属气泡**：

- 每条收到的消息在气泡外显示一个 **头像**（代理的 `emoji`，回退为名字首字母字母组合），气泡上方显示 **显示名**，气泡内是纯文本。
- 头像的背景是一个 **由 `hash(agent_id)` 派生的稳定颜色** — 因此无论花名册顺序或成员变化如何，成员都保持同一种颜色。
- **连续的同一代理消息会合并**：当某条消息与其上方消息的 `agent_id` 相同时，头像和名字会被隐藏（气泡缩进于第一条之下），因此一连串回复会读起来像一个整块。
- 你自己的消息保持右对齐，没有头像或名字。

> 头像颜色以 `agent_id` 为键，而非花名册索引 — 添加或移除某个成员绝不会让其他任何人重新着色。

### 右侧 — 工作区（Workspace）

一个可折叠的工作区，含两个标签页：

- **Artifacts（产物）** — 对话过程中产出的交付物。
- **Tasks / Kanban（任务 / 看板）** — 团队的协调任务以看板形式呈现。

## 侧边栏 — 群聊历史

聊天侧边栏在单代理聊天之上新增一个 **Group Chats（群聊）** 区段，共享同一个滚动列表：

- 每一行显示一个 **成员预览头像簇**（团队成员，通过 `hash(agent_id)` 着色），并以 **最后一条消息片段** 作为副标题。
- 该区段 **可折叠**；如果所选代理不在任何团队中，它会完全隐藏。
- 点击某一行会重新进入该团队：加载花名册、将聊天历史重放到消息流中，并订阅实时团队事件。侧边栏行是持久的入口点 — 你通过点击列表来回到对话。

---

## @ 提及

在编排框中（处于某个团队时）输入 `@` 会打开一个 **花名册自动补全面板**：

- 每一项渲染成员的头像 + 显示名 + 一个灰色 `id`，外加一个 **`@all`** 特殊项。
- 过滤会针对 `@` 之后输入的文本，不区分大小写地匹配 **名字或 id**。
- 选中一项会插入规范的 `@<id>`（或 `@all`）。气泡内部会保留字面的 `@<id>` 令牌并轻微高亮 — 它不会被改写成名字，因此同名代理之间不存在歧义。

提及会针对花名册的 `agent_id` 进行解析；不匹配任何成员的令牌将不会被投递。

---

## 自动命名

一个创建时未命名（空名团队）的团队会 **在其第一条消息时由 LLM 自动命名**，因此它在群聊历史中显示一个有意义的标题，而不是占位符。

## RPC 层

群聊建立在两个不同的 RPC 层之上 — 务必将它们分开：

### 团队工作线程

持久的、按团队划分的群聊线程。当你从侧边栏重新进入某个团队时，它的历史会驱动归属气泡的重放。

| 方法 | 用途 |
|--------|---------|
| `teams.chat.send` | 向团队的群聊线程发送一条消息（扇出到各成员）。 |
| `teams.chat.thread` | 团队的工作线程（任务 / 交付物视图）。 |
| `teams.chat.history` | 按从旧到新的顺序重放某个团队的持久聊天气泡。 |

`agents.teams` 返回每个团队的摘要，并增补了一个小的 `members_preview`（用于头像簇）和 `last_message`（用于行副标题），因此侧边栏避免了按团队逐一查询的 N+1 问题。

### 群聊会话原语

驱动一场实时多代理对话的更底层会话原语：

| 方法 | 用途 |
|--------|---------|
| `group_chat.start` | 开始一个群聊会话。 |
| `group_chat.continue` | 以下一回合推进会话。 |
| `group_chat.end` | 结束会话。 |
| `group_chat.history` | 读取会话的消息历史。 |
| `group_chat.list` | 列出活跃 / 已知的会话。 |
| `group_chat.mention` | 在会话内解析 / 路由某个 `@` 提及。 |

`config` 模块管理 Aleph 的配置系统。它支持通过 TOML 进行基于文件的配置、运行时编程更新以及无需重启服务器的热重载。

## 设计理念

Aleph 中的配置遵循三个原则：

1. **原子性** — 配置更改永远不会是部分的；要么整个更改应用，要么都不应用
2. **安全性** — 运行时更新受冲突和意外擦除保护
3. **透明性** — 更改被记录、可审计且可逆

配置系统建立在标准 Rust 文件操作之上，具有防御性模式以防止数据丢失。

## 核心组件

### ConfigManager

中央配置协调器。

```rust
pub struct ConfigManager {
    store: FileStore,
    patcher: ConfigPatcher,
    defaults: ConfigDefaults,
}
```

**职责：**
- 启动时从 `~/.aleph/aleph.toml` 加载配置
- 监视文件更改（通过 `notify` crate）
- 应用来自 Gateway 管理 API 的运行时补丁
- 将更改原子性地持久化回磁盘

### FileStore

处理具有原子写入语义的所有文件 I/O。

```rust
impl FileStore {
    pub async fn save_to_file(&self,
        path: &Path,
        config: &Config,
    ) -> Result<(), ConfigError> {
        // 1. 写入临时文件
        // 2. fsync 以确保数据写入磁盘
        // 3. 原子重命名（替换旧文件）
        // 4. 成功时清理临时文件
    }
}
```

**原子写入模式：**
1. 将新内容写入同一目录中的临时文件
2. 调用 `fsync()` 将 OS 缓冲区刷新到磁盘
3. 将临时文件重命名为目标文件名（在 POSIX 上是原子的）
4. 如果重命名成功，删除临时文件

如果任何步骤失败，原始配置文件保持不变。

### ConfigPatcher

应用具有冲突检测的增量更新。

```rust
pub struct ConfigPatcher {
    last_modified: SystemTime,
}

impl ConfigPatcher {
    pub async fn apply(
        &mut self,
        config: &mut Config,
        patch: ConfigPatch,
    ) -> Result<(), PatchError> {
        // 1. 检查 mtime — 自上次读取以来文件是否已更改？
        // 2. 如果冲突，重新读取文件并重新应用补丁
        // 3. 验证补丁不变量
        // 4. 在写锁下应用
    }
}
```

**冲突检测 (TOCTOU 缓解)：**
- 读取之前记录文件的修改时间
- 写入之前，检查 mtime 是否已更改
- 如果已更改：重新读取文件，将补丁与当前状态合并，然后写入
- 这防止了当文件监视器和 API 同时尝试修改配置时的丢失更新

## 配置结构

```toml
# ~/.aleph/aleph.toml

[server]
host = "127.0.0.1"
port = 18790

[channels.telegram]
enabled = true
token = "${TELEGRAM_BOT_TOKEN}"

[channels.discord]
enabled = true
token = "${DISCORD_BOT_TOKEN}"

[memory]
embedding_model = "text-embedding-3-small"
max_facts = 10000

[tools]
allowed = ["search", "bash_exec", "file_read"]

[[presets]]
name = "fast"
provider = "openai"
model = "gpt-4o-mini"
temperature = 0.7
```

### 环境变量替换

以 `${}` 为前缀的值在加载时从环境变量解析：

```toml
token = "${TELEGRAM_BOT_TOKEN}"
```

这将秘密保留在配置文件之外，同时允许对非敏感设置进行版本控制的默认值。

### 预设覆盖

命名预设可以覆盖默认设置：

```rust
// 对此请求使用 "fast" 预设
let config = Config::load()
    .with_preset("fast")
    .build();
```

预设对于以下情况很有用：
- 在快速/廉价和慢速/准确模型之间切换
- A/B 测试配置
- 访客特定设置

## 防御性设计

### 嵌入提供商保护

防止嵌入提供商配置的意外擦除：

```rust
if patch.removes("memory.embedding_model") {
    // 捕获回溯以进行调试
    tracing::warn!("Attempted to remove embedding provider");
    return Err(PatchError::InvalidEmbeddingChange);
}
```

### 验证

所有补丁在应用之前都经过验证：
- 端口号必须在有效范围内 (1-65535)
- 文件路径不得包含路径遍历 (`../`)
- 数组长度必须在边界内
- 枚举值必须是有效的变体

## 热重载

通过 `notify-debouncer-full` 在 600ms 内检测到配置更改：

```rust
// 文件监视器设置
let mut watcher = notify::recommended_watcher(move |res| {
    match res {
        Ok(event) if event.kind.is_modify() => {
            // 重新加载配置
            config_manager.reload().await?;
        }
        _ => {}
    }
})?;

watcher.watch(config_path, RecursiveMode::NonRecursive)?;
```

**限制：**
- 嵌入模型更改需要服务器重启（模型权重在启动时加载）
- 端口更改需要重启（套接字绑定在启动时发生）
- 通道令牌更改立即生效

`acp` 模块实现 ACP（代理控制协议），一个用于控制和监控代理行为的标准化接口。ACP 为外部系统与 Aleph 的代理运行时交互提供通用语言。

## 设计理念

ACP 充当 Aleph 代理的"遥控器"：

1. **标准化命令** —— 启动、停止、暂停、恢复、检查 —— 全部通过统一协议
2. **可观察状态** —— 实时查看代理状态、进度和资源使用
3. **默认安全** —— 所有 ACP 命令都需要认证并尊重权限边界

## 协议命令

### 生命周期命令

| 命令 | 说明 | 所需权限 |
|---|---|---|
| `start` | 使用给定配置启动新代理 | `agent:write` |
| `stop` | 优雅终止运行中的代理 | `agent:write` |
| `pause` | 暂停代理执行（保留状态） | `agent:write` |
| `resume` | 恢复暂停的代理 | `agent:write` |
| `inspect` | 获取详细的代理状态和指标 | `agent:read` |
| `logs` | 流式传输代理日志输出 | `agent:read` |

### 状态检查

## ACP 服务器

ACP 服务器通过 WebSocket 暴露 JSON-RPC 接口：

```rust
pub struct AcpServer {
    handlers: HashMap<String, Box<dyn AcpHandler>>,
}

impl AcpServer {
    pub async fn handle_command(
        &self,
        cmd: AcpCommand,
        auth: &IdentityContext,
    ) -> Result<AcpResponse, AcpError> {
        // 验证权限
        if !auth.can(&cmd.required_permission()) {
            return Err(AcpError::Forbidden);
        }
        
        // 分派到处理程序
        let handler = self.handlers.get(&cmd.name)
            .ok_or(AcpError::UnknownCommand)?;
        handler.execute(cmd.params).await
    }
}
```

**认证：**
- ACP 命令使用与网关请求相同的 `IdentityContext` 流程
- 所有者会话拥有完整的 ACP 访问权限
- 访客会话只能检查自己的代理

## 事件流

ACP 支持实时监控的事件流：

```rust
pub enum AcpEvent {
    StateChange {
        agent_id: AgentId,
        old_state: AgentState,
        new_state: AgentState,
    },
    ProgressUpdate {
        agent_id: AgentId,
        progress: f32,
    },
    LogLine {
        agent_id: AgentId,
        level: LogLevel,
        message: String,
    },
}
```

客户端通过网关的 WebSocket 事件总线订阅事件。

## 集成点

ACP 与以下组件集成：
- **网关** —— 通过 JSON-RPC 暴露 ACP 端点
- **代理运行时** —— 控制代理生命周期
- **会话服务** —— 持久化代理状态
- **指标** —— 收集和报告代理指标

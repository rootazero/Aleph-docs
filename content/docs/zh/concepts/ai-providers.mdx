`providers` 模块为 LLM 后端提供统一接口。它支持多个提供商（OpenAI、Anthropic、Ollama 等），具有重试逻辑、增量流式传输、认证配置文件管理和自动故障转移。

## 设计理念

1. **提供商抽象** —— 所有 LLM 实现相同的 `LlmProvider` trait
2. **弹性连接** —— 具有可配置策略的指数退避重试
3. **流式优先** —— Token 级流式传输是默认的，不是事后考虑

## 核心组件

### LlmProvider Trait

```rust
#[async_trait]
pub trait LlmProvider: Send + Sync {
    async fn complete(
        &self,
        request: CompletionRequest,
    ) -> Result<CompletionResponse>;

    async fn complete_stream(
        &self,
        request: CompletionRequest,
    ) -> Result<Box<dyn Stream<Item = ProviderDelta>>>;

    fn name(&self) -> &str;
    fn capabilities(&self) -> ProviderCapabilities;
}
```

**能力：**
- `streaming` —— 支持 token 级流式传输
- `function_calling` —— 支持工具/函数调用
- `vision` —— 支持图像输入
- `json_mode` —— 支持结构化 JSON 输出

### ProviderRegistry

管理具有故障转移的多个提供商：

```rust
pub struct ProviderRegistry {
    providers: HashMap<String, Arc<dyn LlmProvider>>,
    default: String,
}

impl ProviderRegistry {
    pub fn get(
        &self,
        name: &str,
    ) -> Option<Arc<dyn LlmProvider>> { /* ... */ }

    pub fn list(&self,
    ) -> Vec<String> { /* ... */ }
}
```

**确定性：** 故障转移选择使用 `keys().min()` 进行字典序第一个提供商。

### 重试逻辑

```rust
pub async fn retry_with_policy<F, Fut, T>(
    operation: F,
    policy: RetryPolicy,
) -> Result<T>
where
    F: Fn() -> Fut,
    Fut: Future<Output = Result<T>>,
{
    let mut attempt = 0;
    loop {
        match operation().await {
            Ok(result) => return Ok(result),
            Err(e) if attempt < policy.max_attempts => {
                let delay = calculate_delay(attempt, &policy);
                sleep(delay).await;
                attempt += 1;
            }
            Err(e) => return Err(e),
        }
    }
}
```

**退避：** 带上限的指数退避（基于策略的最大 300 秒，简单退避的最大 30 秒）。

**安全：**
- 尝试计算的 `saturating_sub(1)`（防止下溢）
- 退避上限的 `.min(300.0)`（防止无穷大/NaN）

### 增量流式传输

将提供商特定的流式格式转换为统一的增量：

```rust
pub struct ProviderDelta {
    pub content: Option<String>,
    pub tool_call: Option<ToolCallDelta>,
    pub finish_reason: Option<String>,
}
```

**支持的格式：**
- OpenAI SSE (`data: {...}`)
- Anthropic SSE (`event: content_block_delta`)
- Ollama JSON 流式传输

### 认证配置文件

管理 API 凭证：

```rust
pub struct AuthProfile {
    pub provider: String,
    pub api_key: SecretString,
    pub base_url: Option<String>,
}

pub struct AuthProfileRegistry {
    profiles: HashMap<String, AuthProfile>,
}
```

**安全：** API 密钥存储为 `SecretString`（在 drop 时清零，在 Debug 中编辑）。

## 安全属性

- **无锁问题** —— 使用带有恢复模式的 `tokio::sync`
- **生产中无 unwrap** —— 所有重试返回 `Result`
- **确定性故障转移** —— 排序的提供商选择
- **有界退避** —— 防止无限延迟增长

`skill` 模块实现技能系统 v2，一个用于管理技能的领域驱动运行时。它处理技能注册、资格评估、`SKILL.md` 解析、提示注入和生命周期管理。

> 对于日常用户，技能通过统一的[扩展商店](/docs/zh/concepts/extensions-store)进行发现和安装，其中技能是三种扩展类型（`Skill`、`Plugin`、`Mcp`）之一，按功能类别浏览。本页描述商店所路由到的底层技能子系统。

## 设计理念

1. **领域驱动** — 技能是具有清单、资格规则和生命周期状态的一等领域对象
2. **声明式** — 技能在 `SKILL.md` 文件中声明其能力；系统评估何时使用它们
3. **可热重载** — 技能可以在不重启的情况下添加、更新或移除

## 核心组件

### SkillSystem

应用程序其余部分的统一门面：

```rust
pub struct SkillSystem {
    registry: SkillRegistry,
    eligibility: EligibilityService,
    installer: InstallExecutor,
}

impl SkillSystem {
    pub async fn init(&self,
    ) -> Result<()> { /* ... */ }

    pub async fn rebuild(&self,
    ) -> Result<()> { /* ... */ }

    pub fn skill_status(&self,
    ) -> Vec<SkillStatusEntry> { /* ... */ }
}
```

**确定性：** `skill_status()` 和 `full_status()` 按技能 ID 返回排序结果。

### SkillRegistry

维护已知技能集：

```rust
pub struct SkillRegistry {
    skills: HashMap<SkillId, SkillSnapshot>,
}
```

### Skill Manifest

从 `SKILL.md` frontmatter 解析：

```rust
pub struct SkillManifest {
    pub id: SkillId,
    pub name: String,
    pub description: String,
    pub triggers: Vec<String>,
    pub eligibility: EligibilityRules,
}
```

### EligibilityService

确定技能是否可以在当前上下文中使用：

```rust
pub struct EligibilityService;

impl EligibilityService {
    pub fn check(
        &self,
        skill: &SkillManifest,
        context: &ExecutionContext,
    ) -> EligibilityResult { /* ... */ }
}
```

**不合格原因：**
- 缺少依赖项
- 不支持的平台
- 缺少环境变量
- 冲突的技能

### Skill Commands

技能可以暴露 CLI 风格的命令：

```rust
pub struct SkillCommandSpec {
    pub name: String,
    pub description: String,
    pub args: Vec<CommandArg>,
}
```

**解析：** 当多个技能定义相同的命令名称时，工作区技能覆盖捆绑技能（按优先级排序，然后按 ID 决胜）。

### InstallExecutor

从外部源安装技能：

```rust
pub struct InstallExecutor;

impl InstallExecutor {
    pub async fn install(
        &self,
        spec: &InstallSpec,
    ) -> Result<InstallResult> { /* ... */ }
}
```

**安全：** 对包名称使用严格的允许列表 — 仅允许 `[a-zA-Z0-9\-_./:@+=~]`。阻止空格、引号、重定向、反引号、`$()` 和所有 shell 元字符。

## 提示注入

技能作为 XML 注入系统提示词：

```rust
pub fn build_skills_prompt_xml(
    skills: &[SkillSnapshot],
) -> String
```

```xml
<skills>
  <skill id="code-review" name="Code Review">
    <triggers>
      <trigger>review code</trigger>
      <trigger>check this</trigger>
    </triggers>
    <instructions>
      When reviewing code, check for: ...
    </instructions>
  </skill>
</skills>
```

## 生命周期

```
已发现 → 已评估 → 符合条件 → 已加载 → 激活
                ↓
            不符合条件
```

**状态：**
- `Discovered` — 在文件系统中发现
- `Evaluated` — 资格已检查
- `Eligible` — 可以在当前上下文中使用
- `Loaded` — 已解析并准备就绪
- `Active` — 当前对代理可用

---

## 安全属性

- **无命令注入** — 包名称的严格允许列表
- **确定性排序** — 按 ID 排序技能以产生可重现的输出
- **生产中无 unwrap** — 所有错误路径返回 `Result`
- **锁安全** — 使用 `tokio::sync::RwLock`（无中毒）

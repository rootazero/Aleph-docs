---
title: "捆绑内容"
description: "编译时嵌入的官方技能和插件，在启动时提取到磁盘，具有基于版本的更新检测。"
---

`bundled` 模块管理在编译时嵌入 Aleph 二进制文件的官方技能和插件。启动时，如果捆绑版本比已安装的版本更新，则将这些内容提取到 `~/.aleph/`。

## 设计理念

1. **自包含分发** —— 官方内容与二进制文件一起发布，无需外部下载
2. **版本感知更新** —— 仅当二进制文件的捆绑版本更新时才发生提取
3. **用户内容保护** —— 捆绑提取跳过会覆盖用户自定义的文件
4. **幂等提取** —— 多次运行安全；部分失败不会更新版本

## 工作原理

```
编译时                          启动时
───────────                     ───────
skills/ ──► include_dir! ──► binary ──► extract_bundled_content()
plugins/                                │
                                        ▼
                                  ~/.aleph/skills/
                                  ~/.aleph/plugins/
```

## 嵌入内容

```rust
/// 官方技能目录树，在编译时嵌入
pub static BUNDLED_SKILLS: Dir = include_dir!("$CARGO_MANIFEST_DIR/skills");

/// 官方插件（市场），在编译时嵌入
pub static BUNDLED_PLUGINS: Dir = include_dir!("$CARGO_MANIFEST_DIR/plugins");

/// 与服务器发布绑定的版本
pub const BUNDLED_VERSION: &str = env!("ALEPH_VERSION");
```

## 提取逻辑

1. **检查清单** —— 从 `~/.aleph/skills/` 读取 `manifest.json`
2. **比较版本** —— 如果 `manifest.bundled_version == BUNDLED_VERSION` 则跳过提取
3. **提取技能** —— 将嵌入的技能复制到 `~/.aleph/skills/`
4. **提取插件** —— 将嵌入的插件复制到 `~/.aleph/plugins/cache/aleph-official/`
5. **更新清单** —— 仅在成功提取后写入新版本

### 用户内容保护

提取技能时，系统检查 `SkillOrigin`：

- **Official** —— 可以通过捆绑提取更新
- **Custom/User** —— 跳过以防止覆盖用户修改

## 清单格式

```json
{
  "bundled_version": "0.9.1",
  "skills": [
    {
      "name": "git-commit",
      "version": "1.0.0",
      "origin": "official"
    }
  ]
}
```

## 安全

- **零 unwrap** —— 所有错误通过 `if let Err` / `match` / `?` 处理
- **目录创建** —— `create_dir_all` 带错误日志记录
- **BTreeMap** —— 确定性 JSON 输出（无 HashMap 排序问题）
- **无锁** —— 纯文件系统操作，无并发问题

## 关键源文件

- `src/bundled/mod.rs` —— 嵌入内容声明
- `src/bundled/extractor.rs` —— 提取逻辑
- `src/bundled/manifest.rs` —— 清单解析和序列化

title: "审批"
description: "用于代理发起的行动（包括浏览器导航、桌面点击和 shell 命令）的策略驱动审批系统。"
---

`approval` 模块提供策略驱动的审批系统，决定代理发起的行动应该被允许、拒绝还是升级为用户确认。这适用于浏览器导航、桌面 UI 交互、shell 命令和其他潜在敏感操作。

## 设计理念

审批系统遵循三个原则：

1. **默认拒绝** —— 除非策略明确允许，否则行动被阻止
2. **分层决策** —— 阻止列表 → 允许列表 → 默认值 → 询问，按优先级排序
3. **配置驱动** —— 策略在配置文件中定义，不是硬编码的

---

## 架构

```
ActionRequest ──▶ ApprovalPolicy::check() ──▶ ApprovalDecision
                        │
                   ┌────┴────┐
                   │ Config  │  (阻止列表 → 允许列表 → 默认值 → 询问)
                   └─────────┘
```

## 核心类型

### ActionRequest

表示代理想要执行的行动：

```rust
pub struct ActionRequest {
    pub action_type: ActionType,
    pub target: String,
    pub agent_id: String,
    pub context: String,
    pub timestamp: DateTime<Utc>,
}
```

### ActionType

可以请求的行动类别：

```rust
pub enum ActionType {
    BrowserNavigate,
    DesktopClick,
    ShellCommand,
    FileOperation,
    // ... 等等
}
```

### ApprovalDecision

策略检查的结果：

```rust
pub enum ApprovalDecision {
    Allow,
    Deny { reason: String },
    Ask { prompt: String },
}
```

## 策略类型

### ConfigApprovalPolicy

从配置加载规则的主要策略实现：

```rust
pub struct ConfigApprovalPolicy {
    config: PolicyConfig,
}

impl ConfigApprovalPolicy {
    pub fn load() -> Self { /* ... */ }
    
    pub async fn check(
        &self,
        request: &ActionRequest,
    ) -> ApprovalDecision { /* ... */ }
}
```

**决策流程：**
1. 检查阻止列表 —— 如果目标匹配，`Deny`
2. 检查允许列表 —— 如果目标匹配，`Allow`
3. 按行动类型检查默认规则
4. 如果没有规则匹配，`Ask` 用户

### PolicyRule

单个规则定义：

```rust
pub struct PolicyRule {
    pub pattern: String,        // Glob 模式
    pub decision: DefaultDecision,
    pub description: String,
}
```

## Glob 匹配

模式使用具有路径安全语义的 glob 语法：

```rust
pub fn matches_glob(pattern: &str, target: &str) -> bool
```

- `*` 匹配除 `/` 之外的任何字符
- `?` 匹配除 `/` 之外的任何单个字符
- `**` 匹配任何路径段

`?` 模式在内部使用 `[^/]` 正则表达式以实现一致的路径安全匹配。

## 配置格式

策略在主配置文件中定义：

```toml
[approval]

[[approval.blocklist]]
pattern = "**/secrets/*"
decision = "deny"
description = "永远不要访问秘密文件"

[[approval.allowlist]]
pattern = "https://github.com/**"
decision = "allow"
description = "允许 GitHub 导航"

[approval.defaults]
browser_navigate = "ask"
shell_command = "deny"
```

## 安全属性

- **无 SQL 注入** —— 所有查询使用参数化绑定
- **无锁中毒** —— 使用 `tokio::sync` 锁（无中毒）
- **安全算术** —— 风险评分使用 `saturating_add`/`saturating_mul`
- **无 UTF-8 切片** —— 没有 `&s[..n]` 模式

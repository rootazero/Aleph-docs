`thinker` 模块是 Aleph 的提示构建引擎。它构建系统提示词并管理发送到 LLM 的对话上下文。它使用基于层的架构，其中每层为最终提示词贡献一个部分。

## 设计理念

1. **基于层的组合** — 每个关注点（身份、技能、内存、工具）是一个单独的层
2. **提供商特定优化** — 不同的 LLM 提供商接收不同格式的提示词
3. **流式状态机** — 处理来自流式响应的部分 token

---

## 架构

```
┌─────────────────────────────────────────────┐
│                  Thinker                     │
├─────────────────────────────────────────────┤
│                                             │
│  第 1 层: 身份 (代理是谁)                   │
│  第 2 层: 技能 (可用能力)                   │
│  第 3 层: 内存 (相关上下文)                 │
│  第 4 层: 工具 (可用工具)                   │
│  第 5 层: 规则 (行为指南)                   │
│  第 6 层: 对话 (消息历史)                   │
│                                             │
└─────────────────────────────────────────────┘
```

## 核心组件

### Thinker

主提示词构建器：

```rust
pub struct Thinker {
    layers: Vec<Box<dyn PromptLayer>>,
    provider: ProviderType,
}

impl Thinker {
    pub async fn build_prompt(
        &self,
        context: &ThinkerContext,
    ) -> Result<String> {
        let mut prompt = String::new();
        for layer in &self.layers {
            let section = layer.render(context).await?;
            prompt.push_str(&section);
        }
        Ok(prompt)
    }
}
```

### PromptLayer Trait

```rust
#[async_trait]
pub trait PromptLayer: Send + Sync {
    async fn render(
        &self,
        context: &ThinkerContext,
    ) -> Result<String>;

    fn priority(&self) -> i32;
}
```

**内置层：**
- `IdentityLayer` — 代理名称、个性、通信风格
- `SkillsLayer` — 可用技能和触发器
- `MemoryLayer` — 来自知识库的相关记忆
- `ToolsLayer` — 工具描述和模式
- `RulesLayer` — 行为规则和红線
- `ConversationLayer` — 最近消息历史

### 流式状态机

处理来自 LLM 流式的部分 token：

```rust
pub struct BlockState {
    buffer: String,
    current_block: Option<BlockType>,
}

impl BlockState {
    pub fn feed(&mut self,
        token: &str,
    ) -> Vec<BlockEvent> { /* ... */ }
}
```

**块类型：**
- `Text` — 纯文本输出
- `ToolCall` — 工具调用块
- `Code` — 代码围栏块
- `Thinking` — 思维链块

### 提示词清理器

在发送到 LLM 之前清理提示词：

```rust
pub fn sanitize(
    prompt: &str,
) -> String
```

**移除：**
- 重复的系统提示词
- 过多的空白
- 截断的工具定义

## 提供商特定格式化

不同的提供商接收不同的提示词格式：

**OpenAI:**
- 系统消息 + 用户/助手交替
- `tools` 数组中的工具定义
- 函数调用格式

**Anthropic:**
- 用于结构的 XML 风格标签
- `\n\nHuman:` / `\n\nAssistant:` 分隔符
- `<function_calls>` 块中的工具使用

**Ollama / 本地:**
- 简单文本提示词
- 最小格式化开销

## Soul

`soul.rs` 模块管理代理的持久身份：

```rust
pub struct Soul {
    name: String,
    personality: String,
    values: Vec<String>,
}
```

soul 从 `~/.aleph/soul.md` 加载，并通过 IdentityLayer 注入到每个提示词中。

---

## 用户配置文件

每用户自定义：

```rust
pub struct UserProfile {
    preferences: HashMap<String, String>,
    custom_instructions: String,
}
```

用户配置文件覆盖默认行为而不修改 soul。

## 安全属性

- **UTF-8 安全** — 所有字符串切片使用 `.get(..n).unwrap_or_default()`
- **确定性排序** — 提供商列表按 ID 排序
- **无锁问题** — 使用 `tokio::sync` 锁并带有恢复
- **无 `static mut`** — 使用 `OnceLock` 和 `LazyLock`

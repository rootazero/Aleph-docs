`browser` 模块为 Aleph 提供浏览器自动化，支持两个后端：Playwright CLI 驱动程序和通过 MCP 的 Chrome DevTools Protocol (CDP)。它使代理能够导航网站、捕获截图、提取内容和与网页交互。

## 设计理念

1. **后端抽象** —— 所有浏览器操作都通过 `BrowserBackend` trait；代理不关心使用哪个驱动程序
2. **文本优先提取** —— 页面内容提取为干净文本，不是原始 HTML
3. **基于快照的交互** —— 代理接收简化的页面快照（链接、按钮、输入），而不是完整的 DOM

## 核心组件

### BrowserBackend Trait

```rust
pub trait BrowserBackend: Send + Sync {
    async fn navigate(&self, url: &str) -> Result<()>;
    async fn click(&self, target: &ActionTarget) -> Result<()>;
    async fn type_text(&self, selector: &str, text: &str) -> Result<()>;
    async fn screenshot(&self, opts: &ScreenshotOpts) -> Result<ScreenshotOutput>;
    async fn snapshot(&self) -> Result<SnapshotOutput>;
    async fn scroll(&self, direction: ScrollDirection) -> Result<()>;
}
```

### PlaywrightCliBackend

使用 Playwright 的 CLI 控制 headless 浏览器：

```rust
pub struct PlaywrightCliBackend {
    driver: PlaywrightCliDriver,
    profile: BrowserProfile,
}
```

**特性：**
- 启动 Chromium/Chrome/Firefox/WebKit
- 持久化配置文件（cookies、localStorage）
- 截图捕获
- 页面导航和交互
- 内容提取

### ChromeMcpBackend

通过 MCP（模型上下文协议）使用 Chrome DevTools Protocol：

```rust
pub struct ChromeMcpBackend {
    driver: ChromeMcpDriver,
}
```

**特性：**
- 连接到现有 Chrome 实例
- 标签页管理
- 网络拦截
- 控制台日志捕获
- 基于快照的交互

### BrowserManager

管理浏览器实例和配置文件：

```rust
pub struct BrowserManager {
    backends: HashMap<String, Box<dyn BrowserBackend>>,
    profiles: HashMap<String, BrowserProfile>,
}
```

## 页面快照

快照为代理提供页面的简化视图：

```rust
pub struct SnapshotOutput {
    pub url: String,
    pub title: String,
    pub text: String,
    pub links: Vec<Link>,
    pub buttons: Vec<Button>,
    pub inputs: Vec<Input>,
}
```

代理接收这种结构化数据而不是原始 HTML，使其更容易推理页面结构。

## 网络策略

控制浏览器可以访问哪些 URL：

```rust
pub struct BrowserNetworkPolicy {
    allowed_hosts: Vec<String>,
    blocked_hosts: Vec<String>,
}
```

默认阻止私有网络、localhost 和其他敏感 URL。

---

## 配置文件管理

浏览器配置文件持久化 cookies、localStorage 和会话数据：

```rust
pub struct BrowserProfile {
    name: String,
    data_dir: PathBuf,
}
```

配置文件是隔离的 —— 每个配置文件都有自己的 cookie 罐和存储。

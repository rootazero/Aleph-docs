`components` 模块提供 Aleph 代理循环的 6 个核心组件。每个组件都是一个独立的事件处理程序，订阅特定事件类型并将结果发布到事件总线。

## 设计理念

1. **事件驱动组合** — 每个组件都是一个具有类型化订阅的 `EventHandler`
2. **干净分离** — 组件仅通过事件通信，无直接耦合
3. **可恢复状态** — 会话记录将状态持久化到 SQLite 以进行崩溃恢复
4. **Token 感知** — 会话压缩管理上下文窗口限制

## IntentAnalyzer

检测输入复杂度以进行路由决策。使用基于规则的关键词匹配（中文和英文多步关键词）来识别请求是否需要多步规划。

**订阅：** `InputReceived`
**发布：** `PlanRequested`（多步）或 `ToolCallRequested`（简单）

## TaskPlanner

将多步请求分解为可执行的任务。识别并行组和依赖链以获得最佳执行顺序。

**关键特性：** 传递性深度计算以进行正确的并行组识别。顺序链 A→B→C 产生深度 0, 1, 2（不是 1, 1, 1）。

## ToolExecutor

使用重试逻辑和超时处理执行工具。包装工具调用并带有错误恢复和结果格式化。

**重试策略：** 可配置的最大尝试次数，带有指数退避。

## LoopController

使用保护机制控制代理循环：

- **最大迭代次数** — 防止无限循环
- **超时** — 每次迭代的时间限制
- **错误预算** — N 次连续失败后停止
- **让步点** — 允许在步骤之间取消

## SessionRecorder

将会话状态持久化到 SQLite：

- 带有时间戳的代理事件
- 任务执行记录
- 工具调用历史
- Token 使用统计

**错误处理：** 数据库错误通过 `RecorderError` 传播，而非静默吞咽。

## SessionCompactor

通过压缩管理上下文窗口限制：

- **Token 跟踪** — 监控累积的 token 使用
- **模型限制** — 每个模型的上下文窗口限制，带有最长前缀匹配
- **剪枝** — 接近限制时移除最旧的轮次
- **摘要** — 用紧凑的摘要替换剪枝的内容

## 关键源文件

- `src/components/mod.rs` — 组件重新导出
- `src/components/intent_analyzer.rs` — 复杂度检测
- `src/components/task_planner.rs` — 任务分解
- `src/components/tool_executor.rs` — 带有重试的工具执行
- `src/components/loop_controller.rs` — 循环保护
- `src/components/session_recorder/` — SQLite 持久化
- `src/components/session_compactor/` — Token 管理

`extension` 模块为 Aleph 提供统一的扩展系统。它支持插件（WASM 和 Node.js 运行时）和技能，具有清单驱动注册、统一注册表和基于钩子的集成。

> **辨析。** 在代码库中，`extension`（本页所述的 `src/extension/` 模块）是**插件宿主**——负责加载、注册并运行 WASM/Node.js 插件的后端。它不同于面向用户的伞形概念“Extension（扩展）”，后者把技能、插件和 MCP 服务器统一为同一个理念，并通过**扩展商店**来浏览。该伞形概念及其商店位于独立的 `src/store/` 模块；参见[扩展商店](/docs/zh/concepts/extensions-store)。

## 设计理念

1. **统一注册表** —— 插件可以提供的所有内容（工具、频道、HTTP 路由）都注册在单个 `PluginRegistry` 中
2. **能力声明** —— 插件通过 `CapabilityDeclaration` 声明它们提供什么（参见[能力系统](./capability-system)）
3. **沙箱运行时** —— WASM 插件在沙箱中运行；Node.js 插件在单独进程中运行
4. **热重载** —— 扩展可以在不重新启动的情况下添加、更新或移除

## 核心组件

### ExtensionManager

所有扩展的中央协调器：

```rust
pub struct ExtensionManager {
    registry: PluginRegistry,
    loader: PluginLoader,
    skill_system: SkillSystem,
}

impl ExtensionManager {
    pub async fn discover(&self,
    ) -> Result<Vec<PluginManifest>> { /* ... */ }

    pub async fn load(
        &self,
        manifest: &PluginManifest,
    ) -> Result<LoadedPlugin> { /* ... */ }

    pub async fn register(
        &self,
        plugin: LoadedPlugin,
    ) -> Result<()> { /* ... */ }
}
```

### PluginRegistry

所有插件提供能力的统一注册表：

```rust
pub struct PluginRegistry {
    tools: HashMap<String, ToolRegistration>,
    channels: HashMap<String, ChannelRegistration>,
    hooks: HashMap<String, HookRegistration>,
    http_routes: Vec<HttpRouteRegistration>,
    providers: HashMap<String, ProviderRegistration>,
}
```

**注册类型：**
- `Tool` —— 代理的可调用函数
- `Hook` —— 事件拦截器
- `Channel` —— 消息接口集成
- `Provider` —— AI 模型后端
- `HttpRoute` —— REST API 端点
- `Cli` —— CLI 命令
- `Service` —— 后台服务
- `Command` —— 聊天中命令
- `Skill` —— 基于提示的技能
- `Agent` —— 代理定义
- `McpServer` —— MCP 服务器配置

### PluginLoader

从不同来源加载插件：

```rust
pub struct PluginLoader;

impl PluginLoader {
    pub async fn load_wasm(
        &self,
        path: &Path,
    ) -> Result<WasmPlugin> { /* ... */ }

    pub async fn load_nodejs(
        &self,
        path: &Path,
    ) -> Result<NodeJsPlugin> { /* ... */ }
}
```

**WASM 运行时：** 使用 Extism 进行沙箱执行。有限的 I/O，快速启动。

**Node.js 运行时：** 生成具有 IPC 通信的 Node.js 进程。完全访问 Node.js API。

### Manifest

插件在清单文件中声明其能力：

```toml
[plugin]
id = "my-plugin"
name = "My Plugin"
version = "1.0.0"

[[capabilities]]
type = "tool"
name = "my_tool"
description = "Does something useful"

[[capabilities]]
type = "hook"
event = "message_received"
```

**清单格式：**
- `plugin.toml` —— TOML 格式
- `plugin.json` —— JSON 格式
- `cc-plugin.json` —— Claude Code 兼容
- `.mcp.json` —— MCP 服务器配置

### Scope

插件在限制其访问范围的范围内运行：

```rust
pub struct PluginScope {
    allowed_apis: Vec<String>,
    file_access: FileAccessPolicy,
    network_access: NetworkAccessPolicy,
}
```

## 钩子系统

钩子允许插件拦截和修改系统事件：

```rust
pub enum HookEvent {
    MessageReceived { message: Message },
    ToolInvoked { name: String, args: Value },
    ResponseGenerated { text: String },
}

pub trait HookHandler: Send + Sync {
    async fn handle(
        &self,
        event: &HookEvent,
    ) -> HookResult;
}
```

**HookResult：**
- `Continue` —— 继续事件
- `Intercept { modified }` —— 用修改后的版本替换事件
- `Block { reason }` —— 取消事件

## 发现

扩展从多个位置发现：

1. `~/.aleph/plugins/` —— 用户安装的插件
2. `./.aleph/plugins/` —— 项目级插件
3. 与 Aleph 捆绑的内置插件

`PluginWatcher` 监视这些目录的更改并触发热重载。

## 安全属性

- **沙箱 WASM** —— Extism 提供内存隔离和基于能力的安全
- **范围执行** —— 插件只能访问声明的 API
- **清单验证** —— 所有清单在加载前都经过验证
- **无 `static mut`** —— 使用 `OnceLock` 进行延迟初始化

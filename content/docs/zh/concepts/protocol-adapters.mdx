## 概述

Aleph 将**供应商**与**协议**解耦。多个提供商可以共享同一个协议适配器 —— 例如，DeepSeek、Moonshot 和 Groq 都使用 `openai` 协议。添加一个新的 OpenAI 兼容供应商只需要一个预设条目，而不需要新的适配器。

这种设计意味着：

- **一个适配器，多个供应商** —— `openai` 协议处理 OpenAI、DeepSeek、Moonshot 等 20 多个供应商
- **运行时扩展性** —— 通过 YAML 自定义协议，无需重新编译
- **快速迭代** —— YAML 协议在 600ms 内重载

## 三层系统

```
User config.protocol
    │
    ▼
ProtocolRegistry.get(name)
    │
    ├── Dynamic protocols (YAML-loaded) ──▶ ConfigurableProtocol
    │   ├── Minimal mode: base protocol + differences
    │   └── Custom mode: full template rendering
    │
    ├── Built-in protocols ──▶ OpenAI / Anthropic / Gemini / Ollama
    │
    └── Not found ──▶ Error with available protocol list
```

注册表按优先级顺序解析协议名称：动态协议优先，内置协议兜底，若均不存在则报错。

## 内置协议

这些是在启动时注册的编译型 Rust 适配器：

| Protocol | Adapter | Use Case |
|----------|---------|----------|
| `openai` | `OpenAiProtocol` | OpenAI and OpenAI-compatible APIs |
| `anthropic` | `AnthropicProtocol` | Claude API (native Messages API) |
| `gemini` | `GeminiProtocol` | Google Gemini API |
| `openai-responses` | `OpenAiResponsesProtocol` | OpenAI `/v1/responses` API, OpenRouter |
| `codex` / `chatgpt` | `OpenAiResponsesProtocol` (Codex variant) | ChatGPT subscription via OAuth |
| `ollama` | `OllamaProvider` | Local Ollama (native implementation) |

每个内置适配器实现两个方法：

- `build_request(payload, config)` —— 构建 HTTP 请求构造器（流式优先：始终设置 `stream: true`）
- `stream_deltas(response)` —— 将 SSE/流式响应解析为细粒度的 `ProviderDelta` 事件

## 可配置协议

YAML 定义的协议位于 `~/.aleph/protocols/`，支持热重载。系统会监视该目录并自动重新加载定义。

<Callout type="info">
从文件变更到生效的重载延迟约为 600ms（500ms 防抖 + 解析/注册开销）。
</Callout>

### 热重载机制

1. `notify-debouncer-full` 监视 `~/.aleph/protocols/`
2. 检测到文件变更（创建 / 修改 / 删除）
3. YAML 被解析为 `ProtocolDefinition`
4. 创建 `ConfigurableProtocol` 适配器
5. 注册表原子更新
6. 新请求立即使用更新后的协议

## 最小模式与自定义模式

### 最小模式

扩展现有协议，仅覆盖差异部分：

```yaml
# ~/.aleph/protocols/my-proxy.yaml
name: my-proxy
extends: openai
base_url: https://proxy.example.com/v1
differences:
  auth:
    header: X-API-Key
    prefix: "Bearer "
```

最小模式继承基础协议的请求构建和响应解析，仅应用指定的覆盖项。

### 自定义模式

通过模板渲染从头定义完整协议：

```yaml
# ~/.aleph/protocols/exotic-ai.yaml
name: exotic-ai
base_url: https://api.exotic.ai
custom:
  auth:
    type: header
    header: Authorization
    prefix: "Bearer "
  endpoints:
    chat: /v2/completions
    stream: /v2/completions/stream
  request_template: |
    {"model": "{{config.model}}", "messages": [{"role": "user", "content": "{{input}}"}]}
  response_mapping:
    content: "$.choices[0].message.content"
    error: "$.error.message"
```

自定义模式提供对请求模板和 JSONPath 响应映射的完全控制。

## ProtocolRegistry

`ProtocolRegistry` 将协议名称字符串映射到适配器实现：

```rust
pub struct ProtocolRegistry {
    dynamic: RwLock<HashMap<String, Arc<dyn ProtocolAdapter>>>,  // YAML-loaded
    builtin: RwLock<HashMap<String, ProtocolFactory>>,           // Compiled Rust
}
```

查找顺序：

1. **动态协议优先** —— 从 `~/.aleph/protocols/*.yaml` 加载
2. **内置协议兜底** —— 工厂函数按需实例化适配器

注册表访问：

```rust
let registry = ProtocolRegistry::global();
let adapter = registry.get("openai")?;
```

## 添加自定义协议

添加自定义协议只需三步：

**第一步 —— 创建 YAML：**

```yaml
name: my-custom-api
base_protocol: openai
base_url: https://api.example.com/v1
headers:
  Authorization: Bearer ${API_KEY}
```

**第二步 —— 放入协议目录：**

```bash
mv my-custom-api.yaml ~/.aleph/protocols/
```

**第三步 —— 在供应商配置中引用：**

```toml
[providers.my-provider]
protocol = "my-custom-api"
models = ["my-model"]
api_key = "sk-..."
enabled = true
```

## YAML 格式示例

```yaml
name: my-custom-api          # Protocol identifier (referenced in config.protocol)
base_protocol: openai        # Base protocol for minimal mode
base_url: https://api.example.com/v1

# Optional: custom headers applied to every request
headers:
  Authorization: Bearer ${API_KEY}
  X-Custom-Header: my-value

# Minimal mode: override specific fields
differences:
  auth:
    header: X-API-Key
    prefix: "Bearer "

# Or custom mode: full control
custom:
  auth:
    type: header
    header: Authorization
  endpoints:
    chat: /v1/chat/completions
    stream: /v1/chat/completions/stream
  request_template: |
    {"model": "{{config.model}}", "messages": {{messages}}, "stream": true}
  response_mapping:
    content: "$.choices[0].delta.content"
    error: "$.error.message"
```

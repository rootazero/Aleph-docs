---
title: "澄清系统"
description: "用于参数收集的用户澄清流程，支持菜单选择、文本输入和多组问卷。"
---

`clarification` 模块实现 Phantom Flow 交互模式，用于向用户请求缺失的参数。它通过 Halo 覆盖界面支持菜单驱动选择、自由格式文本输入和多组问卷。

## 设计理念

1. **就地交互** —— 澄清出现在 Halo 覆盖层内，不是作为单独的对话框
2. **会话生命周期** —— 创建 → 获取 → 完成/过期 → 清理
3. **超时管理** —— 会话使用 `saturating_sub` 自动过期以防止下溢
4. **并发安全** —— `tokio::sync::RwLock` 保护会话映射

## 澄清类型

### 选择

从预定义选项进行菜单驱动选择：

```rust
let request = ClarificationRequest::select(
    "style-choice",
    "您想要什么风格？",
    vec![
        ClarificationOption::new("professional", "专业"),
        ClarificationOption::new("casual", "休闲"),
    ],
);
```

### 文本

带有可选占位符的自由格式文本输入：

```rust
let request = ClarificationRequest::text(
    "target-language",
    "输入目标语言：",
    Some("例如，西班牙语、法语..."),
);
```

### 多组

多个问题组，每组具有单选选项：

```rust
let request = ClarificationRequest::multi_group(
    "preferences",
    "选择您的偏好",
    vec![
        QuestionGroup {
            label: "主题",
            options: vec!["深色", "浅色"],
        },
        QuestionGroup {
            label: "语言",
            options: vec!["英文", "中文"],
        },
    ],
);
```

## 会话生命周期

```
┌─────────┐   创建    ┌──────────┐   完成   ┌──────────┐
│  开始   │ ───────── │ 待定     │ ──────── │ 完成     │
└─────────┘           └──────────┘          └──────────┘
                            │
                            │ 超时 / 过期
                            ▼
                       ┌──────────┐
                       │ 过期     │ ──► 清理
                       └──────────┘
```

## ClarificationManager

管理所有待处理的澄清会话：

```rust
pub struct ClarificationManager {
    sessions: RwLock<HashMap<String, PendingClarification>>,
    config: SessionConfig,
}
```

**驱逐策略**：当达到 `max_sessions` 时，使用 `min_by_key` 移除最旧的会话（按创建时间）。

## SessionConfig

```rust
pub struct SessionConfig {
    pub max_sessions: usize,      // 默认：100
    pub default_timeout: Duration,// 默认：5 分钟
}
```

## 安全

- **面向用户路径上无 unwrap/expect**
- **超时安全** —— `remaining_time()` 使用 `saturating_sub`
- **并发访问** —— `tokio::sync::RwLock`（无中毒）
- **无 SQL/LanceDB** —— 纯内存会话跟踪

# 工具链

`AgentHarness` 驱动所有网关聊天请求的 **Think → Act** 循环。它由编排器构建，并接收预解析的依赖项（`SessionService`、`ToolService`、`Sandbox`、`AiProvider`）。

**位置**: `src/harness/`

> **迁移完成**: 遗留的 `src/agent_loop/` 目录在第 7 阶段 (2026-04-21) 被删除。所有代理执行——包括 `SubagentTool`——现在通过 编排器 → 工具链 路由。

## 状态机

`Harness` trait 定义了 Think→Act 契约。`TurnState` 是一个简单的二元结果：

| 变体 | 含义 |
|---------|---------|
| `Continue` | 会话应运行另一个 Think→Act 轮次 |
| `Done` | 会话完成；不需要进一步的轮次 |

外层循环 (`Harness::run`) 重复调用 `run_turn`，直到它返回 `Done` 或取消令牌触发。

## 睡眠抑制器

每个 Think→Act 轮次获取类型为 `PreventUserIdleSystemSleep` 的 `IOPMAssertion`，原因字符串为 `"Aleph agent loop"`。断言由 RAII `InhibitorGuard` 持有，其 `Drop` 实现在轮次返回时立即释放它。长时间运行的代理轮次不再会被 macOS 在飞行中让主机进入睡眠而默默截断。

守卫在 `src/harness/agent.rs` 中的 `run_turn` 开始时获取，并在函数返回时自动释放，无论它成功、返回错误还是被取消。

要验证轮次进行中时断言是否处于活动状态：

```bash
pmset -g assertions | grep "Aleph agent loop"
```

断言在轮次完成时从列表中消失。

实现文件：
- `PowerCapability` trait: `desktop/shared/src/traits/power.rs`
- macOS IOPMAssertion FFI: `desktop/macos/src/sleep_inhibitor.rs`
- 代理循环接线: `src/harness/agent.rs::run_turn`

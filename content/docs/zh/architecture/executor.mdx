# 执行器

执行器是工具执行引擎。它接收来自思考器（通过分发器）的工具决策，并将它们路由到适当的处理程序。

**位置**: `src/tools/`

---

## 执行流程

```
Thinker 决策 (tool_use)
     │
     ▼
分发器
     │
     ▼
执行器
     │
     ├───▶ 内置工具 (AlephTool trait)
     │       直接 Rust 执行
     │
     ├───▶ MCP 工具 (Model Context Protocol)
     │       对外部进程的 JSON-RPC
     │
     └───▶ 扩展工具 (WASM / Node.js)
             插件运行时执行
```

Exec 类工具 (`code_exec`, `bash_exec`) 在工具和进程执行之间通过额外的**沙箱层** (`src/sandbox/`)。沙箱拥有每会话工作空间配置、能力执行和通过 `OsSandboxDriver` 的 OS 级 seatbelt 隔离。

## ToolService 外观

消费者仅依赖 `Arc<dyn ToolService>`：

```rust
pub trait ToolService: Send + Sync + 'static {
    async fn execute(&self, name: &str, input: Value) -> Result<ToolOutput, ToolError>;
    async fn list(&self) -> Vec<ToolDefinition>;
    async fn describe(&self, name: &str) -> Option<ToolDefinition>;
}
```

## 中间件栈

```
ExecAuditLayer → PermissionLayer → ContextRuleLayer → TimeoutLayer → CoreDispatch
```

| 层 | 用途 |
|-------|---------|
| `ExecAuditLayer` | 审计日志 |
| `PermissionLayer` | 权限检查 (`SmartFilter`) |
| `ContextRuleLayer` | 基于上下文的规则 |
| `TimeoutLayer` | 超时执行 |
| `CoreDispatch` | 最内层，持有 `ToolRegistry`，包含 `BuiltinHandler`、`McpHandler` 和 `ExtensionHandler` |

`CoreDispatch` 持有由 `ArcSwap` 支持的 `ToolRegistry`，包含三个处理程序源：
- `BuiltinHandler`
- `McpHandler`
- `ExtensionHandler`

## 核心分发

工具作者实现 `AlephTool` —— 外观通过 `BuiltinHandler` 适配它们。作者端 API 无需更改。

会话事件跟踪：调用者通过 `crate::session::invoke_with_session_trace(tool_svc, session_svc, ...)` 捆绑 `ToolService` 分发 + `SessionService` 事件发送。

## AlephTool Trait

### 静态分发（编译时）

```rust
pub trait AlephTool: Clone + Send + Sync + 'static {
    /// 工具名称（在 LLM tool_use 中使用）
    const NAME: &'static str;

    /// LLM 的工具描述
    const DESCRIPTION: &'static str;

    /// 参数类型（通过 schemars 自动生成 JSON Schema）
    type Args: Serialize + DeserializeOwned + JsonSchema + Send;

    /// 返回类型
    type Output: Serialize + Send;

    /// 执行工具
    async fn call(&self, args: Self::Args) -> Result<Self::Output>;
}
```

### 动态分发（运行时）

```rust
pub trait AlephToolDyn: Send + Sync {
    fn name(&self) -> &str;
    fn definition(&self) -> ToolDefinition;
    fn call(&self, args: Value) -> BoxFuture<'_, Result<Value>>;
}

// 空实现：任何 AlephTool 也是 AlephToolDyn
impl<T: AlephTool> AlephToolDyn for T { // 空实现 }
```

## Exec 类工具 & 沙箱

- `code_exec`, `bash_exec` 通过沙箱层路由
- `Sandbox` trait → `WorkspaceSandbox` → `OsSandboxDriver`
- 每会话工作空间位于 `~/.aleph/workspaces/{hash(session_id)}/`
- 工具级权限 (`SmartFilter`) 控制*是否*允许调用
- 沙箱的能力检查控制一旦允许*子进程可以做什么*

# 分发器

分发器管理工具注册、发现和风险评估。它聚合来自多个源的工具，评估其风险级别，并提供语义检索，以便思考器可以超越静态工具列表发现相关工具。

**位置**: `src/dispatcher/`

## 工具注册表

`ToolRegistry` 聚合来自多个源的所有可用工具：

| 源 | 类型 | 示例 |
|--------|------|---------|
| **原生** | 内置 Rust 工具 | `memory_search`、`note_manage` |
| **MCP** | 外部 MCP 服务器 | GitHub、filesystem |
| **技能** | 动态注册的技能 | 自定义技能工具 |
| **自定义** | 用户定义的工具 | 配置的扩展 |

---

## 风险评估

`RiskEvaluator` 为每个工具分配 `RiskLevel`：

```rust
pub enum RiskLevel {
    Safe,       // 只读，无副作用
    Low,        // 轻微副作用（如文件读取）
    Medium,     // 中等风险（如文件写入）
    High,       // 重大风险（如代码执行）
    Critical,   // 破坏性操作
}
```

## 工具索引（语义检索）

`ToolIndex` 启用语义工具发现。系统不仅依赖工具名称和描述，还可以基于用户查询的语义含义检索工具。

```rust
pub struct HydrationResult {
    pub tools: Vec<HydratedTool>,
    pub level: HydrationLevel,
}

pub enum HydrationLevel {
    None,       // 未执行水合
    Partial,    // 部分工具水合
    Full,       // 所有相关工具水合
}
```

`HydrationPipeline` 协调语义检索：
1. **目的推断** — 从查询推断用户意图
2. **工具检索** — 通过语义相似度检索候选工具
3. **工具元丰富** — 用上下文元数据增强工具定义
